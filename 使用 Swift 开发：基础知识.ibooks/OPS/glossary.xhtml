Ternary (三元条件运算符)

三元条件运算符是一种控制流程运算符，可用于根据布尔比较来赋值或返回值。

Gesture recognizer (手势识别器)

手势识别器是一种对象，它会监听用户输入的普通手势，当识别出特定手势后，即会调用相应的操作。

Case

在枚举中，case 关键字声明其中一个枚举选项的名称。在 switch 中，可以使用 case 关键字引入一种式样，用以尝试匹配某个值。

Any

顾名思义，Any 可以代表任何类型的实例：字符串、双精度浮点型、函数等等。

AnyObject

AnyObject 可以代表 Swift 内的任意类，但不能代表结构体。

As!

as! 是一种类型强制转换运算符。此运算符会强制向下转换为指定类型。

As?

as? 是一种条件转换运算符，其会尝试将值向下转换为更具体的类型，并将其储存在新的常量或变量中。

Bool

布尔类型只有两个可能的值：true 和 false。布尔 (Boolean) 是以 George Boole 的名字命名的，这位 19 世纪的数学家当时就意识到问题明确，答案简单有多么重要。在 Swift 中，布尔类型称为 Bool。

Default

当没有其他选项可用时，会选择 default 选项。在 switch 语句中，如果没有其他任何更具体的情况与输入匹配，那么会执行默认代码块。在 if 语句中，default 选项是最终的 else 子句。

Double

Double 是带小数点的数字，例如 7.78 或 1000.0。如果将 Swift 中的常量或变量初始化为带小数点的字面量数字（例如 let distance = 26.2），则推断类型为 Double。

Enum

enum 关键字可用于声明由一组相关选项组成的类型。一个 enum 实例就是 enum 的一个选项。关键字 enum 源于 enumeration 一词，意思是逐一列出不同的事物。

Guard

guard 语句会声明一个条件，该条件必须为 true，才能执行 guard 语句后面的代码。与使用 if 语句相比，使用 guard 语句来检查需求可提高代码的可读性。

Int

Int 会储存一个整数，如 10 或 -42。

Let

let 可用于在具名常量中储存值。在编译时无需知道常量的值，但必须为常量赋值一次（只需一次）。

Nil

nil 指示缺少值。

UIKit

UIKit 框架 (UIKit.framework) 提供了构造和管理 iOS 及 Apple TVOS app 所需的关键基础结构。此框架提供了管理 app 用户界面所需的窗口和视图架构、响应用户输入所需的事件处理基础结构，以及驱动主运行循环和与系统进行交互所需的 app 模型。

Var

var 可用于在具名变量中储存值。变量的值可以转变或更改。

Action (响应动作)

响应动作或 IBAction 用于将源代码中的方法与 Interface Builder 中的控件连接起来。当用户与 app 的控件进行交互时，系统就可以运行特定的代码。例如，某种方法可能与按钮轻点或开关更改等响应动作相关联。

Active scheme

一个 Xcode 项目可以创建多个 app，例如，一个 iOS 应用程序和一个 watchOS 应用程序。active scheme 会告诉 Xcode 哪个是构建目标。

App icon (App 图标)

App 图标是某个 iOS app 的视觉标志或表示形式，用于在主屏幕、App Store、聚焦搜索和“设置”app 中标识该 app。

Argument label (自变量标签)

自变量标签用于引入传递到函数中的自变量，就像你可能会在表单上的名称标签（“Name”）旁边写上你的名字（“Maya”）一样。因此，如果函数声明为 func drawLine(from fromPoint: Point, to toPoint: Point)，那么“from”和“to”两个词就是自变量标签，因为在调用该函数时，它们将标记自变量值，例如：drawLine(from: startPoint, to: endPoint)。

Array (数组)

数组用于按顺序存放同一类型的一系列项目。数组中的项目不必是唯一的；["duck", "duck", "goose"] 是一个包含三个元素的有效数组。

Aspect ratio (宽高比)

宽高比是指图像或屏幕的宽度与高度的关系。

Assistant editor (助理编辑器)

助理编辑器是 Xcode 中的另一个窗口，用于并排显示代码编辑界面。

Auto Layout (自动布局)

使用 Interface Builder 中的“自动布局”来构建自适应界面，可以确保无论屏幕的大小或方向如何，用户界面元素都能保持同样的相对位置。例如，你可以添加一个规则来限制按钮必须始终位于图像视图顶部且与之保持固定距离，还可以再添加一个规则来限制图像视图必须始终在屏幕底部居中。在“自动布局”中定义这些规则后，无论屏幕的大小或方向（竖屏或横屏模式）如何，该按钮和图像视图都将遵循这些规则。

Badge (标记)

标记是标签栏项上的指示符或说明，通常用于传达附加信息。

Bar button (工具栏按钮)

工具栏按钮项目代表 UIToolbar 或 UINavigationItem 对象上的项目。

Base class (基类)

基类位于类层次结构的根部（因此，它没有超类）。

Brand (品牌)

品牌是 app 的视觉标识。

Breakpoint (断点)

断点是程序中有意停止或暂停的位置。程序员使用断点来检查值或执行其他调试操作。

Bug (程序错误)

程序错误是指代码中妨碍应用程序按预期运行的错误。

Button (按钮)

按钮会截获触摸事件，并在被轻点时向目标对象发送操作消息。

Canvas (画布)

画布是 Storyboard 的背景。在画布中，可以添加和排列场景以及用户界面 (UI) 元素。

Case-sensitive、case sensitivity (区分大小写)

如果区分大小写，那么表明字符是大写还是小写很重要。

Class (类)

类通过定义属性来储存值；通过定义方法来提供功能；通过定义构造器来设置它们的初始状态。

Clipping (剪切)

你可能会遇到布局中发生剪切的警告。在大多数情况下，可以使用 Interface Builder 中的“自动布局”来修复剪切问题。

Compiler (编译器)

编译器会将代码中的语句处理成可执行程序。

Compiler error、error (编译器错误、错误)

Xcode 编译器错误通常由导致 app 无法编译和运行的语法错误所引起。Xcode 会在违规代码旁边显示红色错误图标。

Compound assignment (复合赋值)

复合赋值会根据变量的当前值来更新变量的值。使用复合赋值时，无需键入两次变量名。例如，+= 是加法 (+) 复合赋值的简写；它会检查左侧变量的当前值，计算该当前值加上右侧值的和，然后将计算出的和指定为变量的新值。如果不想使用复合赋值，可以将 teamScore += 3 写为 teamScore = teamScore + 3。

Computed property (计算型属性)

Swift 有一个特性叫做计算型属性。使用该特性，属性就可以执行返回计算值的逻辑。

Concatenate、concatenation (串联)

串联是指将两个字符串联接在一起的操作。

Conditional (条件)

条件会运行多次检查并根据检查结果运行代码。这些条件语句是控制流程的一部分。控制流程是更广义的概念。作为开发者，你可以使用控制流程工具来检查某些条件，然后根据这些条件执行不同的代码块。

Conditional cast (条件转换)

在可能的情况下，条件转换会将实例转换为指定的类型。

Console (控制台)

控制台是程序员的信息中心，用于显示有关程序运行的详细信息。请参阅控制台面板。

Console pane、console area (控制台面板、控制台区域)

控制台面板或控制台区域是 Xcode 中的命令行环境，在其中可以调试 app 或与之交互。

Constant (常量)

常量是一种不可变的标识符。为常量赋值后，该值将永远不会改变；尝试将新值赋给现有常量将会产生错误。在 Swift 中，常量使用 let 进行声明。

Constraint (约束)

约束是“自动布局”中用于定义视图布局或大小的规则。

Content mode (内容模式)

通过选择视图的内容模式，以调整其在框架内部适配对应空间的方式。例如，如果某个图像文件大于其图像视图的边框，那么选择内容模式 .scaleToFill 则会挤压（并扭曲）图像，使其大小与边框完全相同；选择内容模式 .top 则会尽可能多地显示图像顶部。

Control (控件)

控件或 UIControl 实例是一种专门的视图类型，用于响应用户输入。

Control event (控件事件)

控件具有简单、直接的特点。用户对于控件都很熟悉，因为控件在大多数 iOS app 中都会用到。轻点按钮或控件的值发生变化时，将触发控件事件。

Date picker (日期选取器)

用户可以使用日期选取器控件来选择特定的日期或时间，也可以两者都有。

Debug area (调试区域)

调试区域可用于检查作用域中常量或变量的当前值。

Deprecated code (已弃用代码)

已弃用代码是指不再受支持或在未来版本的 iOS 中将不受支持的代码。

Developer mode (开发者模式)

在 Xcode 中，使用开发者模式可访问某些调试功能，而且不必每次都输入密码。

Dictionary (字典)

我们常说的字典由一系列单词及其释义组成。Swift 字典与此类似，由一系列键及其关联的值组成。出于优化目的，字典中的值是无序的（这与我们常说的字典不同），但可以通过与其关联的键进行查找。

Document outline (文稿大纲)

文稿大纲是 Interface Builder 中的一个功能，用于显示给定 Storyboard 中所有场景和 UI 元素的列表。

Documentation browser (文档浏览器)

文档浏览器是一种 Xcode 工具，用于查找和阅览开发者文档。

Downcast (向下转换)

使用向下转换可将对象转换为更具体的类型，例如从通用的 Any 转换为 Int，或者从超类转换为子类。

Dynamic data (动态数据)

动态数据是指不断变化的数据。

Enumeration (枚举)

枚举或 enum 是一种数据类型，用于定义一组相关的值，你可以在代码中以类型安全的方式来使用这些值。

Equality (相等性)

使用 == 运算符，可以检查相等性，即两个值是否相等。你可以使用自定类型来检查相等性，方法是采用 Equatable 协议。

Escape character (转义符)

在字符串中使用转义符 (\) 可告知 Swift 应以特殊方式处理接下来的内容。此符号称为转义符是因为它会避开或免除字符串的正常行为。

Exception (异常)

异常也称为错误。请参阅编译器错误。

Executable (可执行)

可执行文件包含 app 的编译代码。App 的可执行文件名称等于 app 的名称减去 .app 扩展名。

Failable initializer (可失败构造器)

可失败构造器是指可失败且将 nil 作为结果返回的构造器。

Flat hierarchy (平面层次结构)

平面层次结构是指导航中的每个场景都位于相同的上下文级别，与标签栏控制器一样。请参阅导航层次结构。

For-in loop (for-in 循环)

for-in 循环会遍历集合，针对集合中的每个项来执行。该循环内的代码块将针对每个属性执行一次。

Force-unwrap (强制解包)

在可选类型值后面使用强制解包运算符 (!)可以访问其基础值。如果强制解包值为 nil 的可选类型，将会导致崩溃。

Frame (边框)

视图的边框是指与父视图的顶端和左侧具有一定距离的矩形。

Function (函数)

函数是指将多行代码合并为一个可再次使用的代码块。有些函数具有输入，并且可根据调用方提供的自变量来改变自身的行为；有些函数具有输出，并且可提供结果；有些函数既没有输入，也没有输出。

Global scope (全局作用域)

全局变量在程序的顶层作用域或全局作用域内定义。请参阅本地作用域和变量隐蔽。

Guard-let

请参阅 guard。

Initializer (构造器)

构造器会设置一个实例以备使用。在声明某个常量或变量的名称后，需要通过首次赋值来初始化该常量或变量。

Integrated Development Environment (IDE，集成开发环境)

集成开发环境 (IDE) 是紧密集成的软件套件，其中整合了用于编写和测试 app 的开发者工具。

If statement (if 语句)

if 语句是一种根据一个或多个条件的值来执行代码的代码结构体。if 语句中的第一个代码块是 if 块。if 语句中也可以包含用于执行附加检查的其他块，如 else 块和 else if 块。

If-else statement (if-else 语句)

if-else 语句是一种代码结构体，当指定条件为 true 时，执行一个代码块；当指定条件为 false 时，执行另一个代码块。

 

Image view (图像视图)

图像视图用于在用户界面中显示图像文件。

Immutable (不可变)

不可变对象不能更新或更改。常量是不可变的；变量是可变的。

Implementation (实现)

实现是指将决定付诸行动。要实现某个方法或函数，可以编写代码来执行函数名称中所描述的操作。

Implicitly unwrapped optional (隐式解包的可选类型)

隐式解包的可选类型是一种也可以像非可选类型值一样使用的可选类型，每次访问此类可选类型时都无需对其解包，因为假定在设置初始值后它始终有值。如果尝试访问隐式解包的可选类型，但其当前为 nil，那么 app 将崩溃。

Index (索引)

索引是指有序集合中项目的编号位置。第一个项目的索引永远都是 0，最后一个项目的索引等于项目总数减 1。

Inheritance (继承)

当某个类是另一个类的子类时，它将从其超类获得所有行为（方法、属性和其他特征）。这称为继承。

Initial view controller (初始视图控制器)

初始视图控制器是 app 启动时用户看到的第一个场景。在 Storyboard 中，会以灰色箭头来标记初始视图控制器，但你也可以在“Attributes”检查器中为单个场景设置初始视图控制器。

Initialization (初始化)

要使用某个类型，必须为该类型创建实例，这个过程称为初始化。

Interval matching (区间匹配)

使用数字时，可以使用区间匹配来检查数字是否在范围内。

Instance method (实例方法)

实例方法是一种在类型（例如 struct、class 或 enum）内声明的函数，其行为取决于个别实体的当前状态。

Label (标签)

标签是可显示一行或多行只读文本的视图。

Landscape (横屏)

横屏是指水平的屏幕方向，像横版绘画或照片一样。

Launch screen (启动屏幕)

启动屏幕会在 app 启动时立即显示，然后会被载入的初始视图控制器替换掉。

Literal (字面量)

字面量值可直接键入或插入到代码中，无需构造器。字面量字符串可以是 "hello"；字面量数组可以是 [1, 2, 3]。非字面量字符串可使用 String() 进行初始化；非字面量整数可使用 homeTeamPoints + awayTeamPoints 进行初始化。

Local scope (本地作用域)

本地变量在特定代码块或程序的本地作用域内定义。请参阅全局作用域和变量隐蔽。

Logical operator (逻辑运算符)

逻辑运算符是一个符号（例如 &&、|| 或 ! ），可用于修改或合并布尔逻辑值 true 和 false。

Memberwise initializer (成员级构造器)

当你创建新的结构体实例时，Swift 会自动创建成员级构造器，然后你可以使用该构造器按名称为新实例设置每个属性。

Method (方法)

方法是在类型内部定义的一种函数。方法可以使用存储在类型的属性中的数据来执行操作。

Minimum viable product (MVP，最小可行性产品)

最小可行性产品 (MVP) 是仅包含发布所需功能的产品。MVP 通常用于快速发布产品和收集用户反馈，以免将时间耗费到构建可能不实用或不需要的功能上。

Modal presentation (模态显示)

模态显示会将一个新的视图控制器放置在前一个视图控制器的上面。在较小的屏幕上，模态显示将始终以全屏显示。要使 UI 适应较大屏幕的设备，可以自定模态显示，使其以弹出窗口、表单或全屏形式显示。

Modal segue (模态过渡)

在模态过渡中，一个视图控制器采用模态显示样式将另一个视图控制器显示为其子项。使用模态过渡可显示用户在继续之前必须完成的任务。

Mutable (可变)

可变对象可以更新或更改。变量是可变的；常量是不可变的。

Navigation bar (导航栏)

导航栏可控制层次结构内容的导航，通常显示在屏幕顶部且与导航控制器配合使用。

Navigation controller (导航控制器)

导航控制器是一种专门的视图控制器子类，其通过一系列视图控制器来管理向后和向前过渡。

Navigation hierarchy (导航层次结构)

导航层次结构是指 app 中的工作流程。平面导航层次结构通常使用标签栏控制器进行构建，而分层式工作流程通常使用导航控制器进行构建。通过将多个工作流程结合在一起，构成更大的工作流程或导航层次结构，可以创建更复杂的 app。

Nested optional (嵌套可选类型)

可选类型值可以具有可选类型属性（例如框中框），这些属性称为嵌套可选类型。

Open source (开源)

开源语言的开发是公开的，社群可以对其提供意见和支持。每个人都可以贡献代码，也可以只沿用已有的代码。

Operator (运算符)

运算符是一个符号（例如 +、- 或 &&），表示对一个或多个值执行的操作。例如，在 1 + 2 中，加法运算符 (+) 表示将数字 1 和 2 加起来。

Optional (可选类型)

可选类型是包含基础值或 nil（指示缺少值）的值。

Optional binding (可选类型绑定)

可选类型绑定是指尝试在条件语句中将可选类型值赋给常量以查看该可选类型是否包含基础值的过程。

Optional chaining (可选类型链)

通过可选类型链，可以使用有条件解包的嵌套可选类型。如果过程中有任何值为 nil，操作将失败。

Outlet (插座变量)

插座变量或 IBOutlet 会将源代码中的变量与 Storyboard 中的对象连接起来，让代码可以从对象中获取信息，以及在 app 运行时执行更改操作。

Override (覆盖)

覆盖是指替换超类上定义的方法的实现。

Parameter (参数)

参数是指函数内的输入值。例如，如果函数声明为 func drawLine(from fromPoint: Point, to toPoint: Point)，那么 fromPoint 和 toPoint 就是参数名称，因为这两个名称将用于函数的实现中。（在非正式情况下，开发者可以将参数与自变量互换使用。）

Persona (用户画像)

用户画像是指一组用户特征或一种用户类型，其会以特定方式使用 app。

Playground

Playground 是 Xcode 中的一种环境，用于试验各种代码并即时查看结果。

Pop (取出)

取出是指移除最近添加到堆栈中的项。例如，将视图控制器从堆栈中取出，表示退出该视图控制器。（将视图控制器压入到导航堆栈上，以显示该视图控制器。）

Portrait (竖屏)

竖屏是指垂直的屏幕方向，像竖版绘画或照片一样。

Project (项目)

项目文件包含用于编译 Swift 程序或 app 的所有代码和资源，可在 Xcode 中打开。

Project navigator (项目导航器)

项目导航器位于 Xcode 用户界面的左侧，其中列出了与项目关联的所有文件，让你可以轻松地管理项目文件。

Project template (项目模板)

Xcode 中包含许多项目模板，以这些模板作为起点，可以开始构建新的 app。

Property (属性)

属性是指结构体、类或枚举所持有的数据片段。例如，每个数组实例都有一个 count 属性。该属性会根据数组的特征不同而变化。

Push (压入)

压入是指向堆栈添加新项。例如，将视图控制器压入到导航堆栈上以显示该视图控制器。（将视图控制器从堆栈中取出，以退出该视图控制器。）

Push notification (推送通知)

推送通知是从 Apple Push Notification 服务 (APN) 发送到未在前台运行的 app 的消息。通知可能显示为用户通知，也可能静默传送。当通知送达时，会暂时唤醒接收通知的 app 来处理新数据。

Quick Help (快速帮助)

Xcode 中的“快速帮助”功能会提供快速的帮助，其会显示一个弹出窗口，其中包含函数的简要说明以及支持该函数的操作系统版本。此外，还会显示该函数所属的框架。

Range (范围)

范围是一组从最小值到最大值的数字。

Results sidebar (结果边栏)

Playground 的结果边栏中会显示有关生成输出值的每行代码的信息。当你添加或更改代码时，playground 会重新运行代码并在边栏中更新结果。

Return type (返回类型)

返回类型用于指示函数或方法将返回的值的类型。

Return value (返回值)

返回值是从函数或方法返回的值或返回给调用方的值。

Scene (场景)

场景是一种 Storyboard 呈现形式，用于呈现 app 中的一屏内容。

Scope (作用域)

类型、变量、常量或函数仅限在指定作用域内可访问。为函数、if 语句或 for 循环添加一对花括号 ({ })，将花括号内的区域指定为本地作用域。

Scope creep (范围蔓延)

向 app 设计中逐渐添加越来越多的功能称为范围蔓延。范围蔓延不但会造成 app 发布延期，而且还可能会增加目标用户的困惑感。

Scroll view (滚动视图)

通过滚动视图，用户可以查看超出视图边界的内容。如果要显示的信息超出设备屏幕边界，请使用滚动视图。

Segmented control (分段控件)

分段控件可将搜索结果范围缩小到特定媒体类型。

Segue (过渡)

过渡是指从 Storyboard 中的一个场景转换到另一个场景。

Self

self 关键字是指对象的特定实例，可在调用方法，或者在类型定义内获取或设置属性时使用。

Show segue (Show 过渡)

Show 过渡会根据目标视图控制器来改变显示新内容的方式。对于导航控制器， Show 过渡会将新内容压入到当前视图控制器堆栈的顶部。通过导航控制器和 Show 过渡，可以检索层次结构数据。

Sibling (同级)

在具有父对象和子对象的情景中，两个子对象具有同级关系。在讨论视图时，同级视图共享一个父视图，或者位于视图层次结构的同一级别。

Slider (滑块)

滑块控件是一种界面功能，用户可以使用它在最小数值与最大数值之间选择一个值。

Stack view (堆栈视图)

使用堆栈视图可在用户界面中以从上到下的列形式或从左到右的行形式设置元素。

Standard editor (标准编辑器)

标准编辑器会显示当前选择的文件项目导航器。

State (状态)

状态是指在给定时间储存的变量、程序或系统的信息。

Static data (静态数据)

静态数据是没有变化的数据。

Step control button (分步控制按钮)

分步控制按钮位于调试区域顶部，适用于慢速逐步执行代码。

Storyboard

Storyboard 是一个用户界面文件，用来告诉 app 在屏幕的什么位置显示信息。Xcode 会在 Interface Builder 中打开 Storyboard。要编辑 Storyboard，可以使用检查器区域中的点按、拖移和选择选项。

String interpolation (字符串插值)

使用字符串插值可将常量或变量的值放置在字符串字面量的中间。要添加常量或变量的占位符，可以使用转义序列，即在要替换的代码周围添加括号。例如，如果 friendName 设置为 "Lee"，那么表达式 "Have a good one, \(friendName)" 将自动读取为 "Have a good one, Lee"。

Structure (结构体)

结构体是一种与类相似的数据类型，但它不支持继承，而且只能通过值传递，而不能通过引用传递。

Subclass (子类)

子类是另一个类（称为超类）的子项。

Substring (子字符串)

子字符串是字符串的一部分。

Superclass (超类)

超类是另一个类（称为子类）的父类。

Switch

switch 关键字会根据值的特征在不同的操作路线之间进行选择。值对应的选项用 case 语句进行编写。一旦 switch 语句在值和 case 之间找到第一个匹配项，即会执行相应 case 语句的代码块。当该代码块执行完毕后，程序会继续执行整个 switch 语句后面的代码。

Symbol (符号)

符号是给定类型的方法或属性。

System item (系统项)

iOS 包含许多系统项，方便用户向标签栏添加熟悉的标签栏项，例如“更多”项和“个人收藏”项。

Tab bar (标签栏)

标签栏包含多个标签栏项，通过这些项可以轻松地访问 app 中的不同视图。

Tab bar controller (标签栏控制器)

通过标签栏控制器，用户可以在其工作流程的任意一点快速在不同上下文或任务之间进行切换。

Tab bar item (标签栏项)

标签栏项是标签栏内的一个元素或特定功能，由一个图标图像和文本组成。

Table view (表格视图)

表格视图用户界面会以列表形式显示多个项目或记录。

Target (目标)

目标是有关如何基于一组文件来构建单个成品（例如框架、库、应用程序或命令行工具）的指示说明。

Text field (文本字段)

文本字段或 UITextField 会在 app 界面中显示可编辑文本区域。iOS 键盘可配置为多种类型的输入。

Toolbar (工具栏)

Xcode 窗口顶部的工具栏会显示当前项目的状态，以及用于隐藏/显示其他窗口区域、选择运行 app 的设备以及构建/运行 app 的选项。

Type (类型)

类型是一类数据的一组具名属性（功能）和方法（行为）。Swift 中的类型名称始终都使用驼峰式命名法，例如 Double、String 或 BicycleRoute。

Type casting (类型转换)

类型转换可用于检查某个对象的类型，也可用于将该对象视为来自其自己的类层次结构中其他位置的不同超类或子类来处理。

Type inference (类型推断)

无论代码中是否明确地表述出类型的名称，Swift 都可以使用类型推断根据可用的信息推断出某个对象的类型。例如，如果函数声明为返回 String（如 func answer() -> String），并且常量初始化为 answer 函数的结果（如 let nextIdea = answer()），那么将推断出 nextIdea 的类型为 String。

Type inspection (类型检查)

类型检查用于确定或检查数据的特定类型。

Type safety (类型安全)

Swift 具有类型安全特性，可防止编写出类型不正确或异常的代码。例如，如果某个函数预期 Fruit 类型，而你使用 Vegetable 来调用它，那么只有在修正该错误后，你的 app 或 playground 才能正常运行。

Unicode

Unicode 标准定义了如何以一致的方式呈现几乎所有语言中的所有字符。向 Unicode 标准中添加哪些新字符（包括表情符号）以及如何为这些字符提供支持，由 Unicode 委员会成员来决定。

Unwind segue (Unwind 过渡)

当关闭场景或视图控制器时，将调用 Unwind 过渡，它可用于向导航堆栈中的其他视图传递信息。

Inspector area (检查器区域)

检查器区域用于查找和编辑在编辑器中选择的某个项目的详细信息。在 Interface Builder 中编辑用户界面文件时，检查器区域特别有用。

Variable (变量)

变量是初始化后可以改变的值，在 Swift 中由 var 关键字来指示。

Variable shadowing (变量隐蔽)

如果在本地作用域内声明的变量与在全局作用域内声明的变量同名，将发生变量隐蔽。引用该变量的代码会引用最近可访问作用域内的实例。

Variables view (变量视图)

变量视图会在主动调试期间列出每个变量的值。

Version editor (版本编辑器)

当使用源代码控制系统（如 Git）管理 Xcode 文件时，可使用版本编辑器并排显示当前版本和先前版本的视图。

View controller (视图控制器)

视图控制器用于管理一组视图，以及协调 app 的数据模型与显示数据的视图之间的信息流。

Warning (警告)

编译器警告的直观表现形式就是代码中的错误。此类警告不会阻止编译 app，但要安全编译和运行程序，应该将其解决掉。

While loop (while 循环)

while 循环是只要给定条件为 true 就一直运行的代码块。当条件变为 false 时，循环就会停止运行。

Workflow (工作流程)

工作流程是一种导航层次结构，通过它，用户可以在你的 app 中完成给定任务或访问某项功能。

XIB

XIB 文件代表 iOS 中的单个场景或视图，可在 Interface Builder 中打开和修改。在 iOS 5 中引入 Storyboard 之前，XIB 文件更为流行。

Comment (注释)

可读文本，用于说明周围的代码。系统不会将注释视为程序的一部分来执行。在 Swift 中，可以通过在注释文本前面添加“//”来创建注释。

Size class (尺寸类)

尺寸类代表可用于调整界面的不同尺寸，其中高度和宽度为小尺寸或常规尺寸。

Root view controller (根视图控制器)

要显示的第一个视图控制器或场景。导航堆栈的底部。

Human Interface Guidelines

由 Apple 维护的一些指导原则，用于说明如何构建简单直观且具有吸引力的用户界面。 

Template image (模板图像)

一种没有颜色但具有 app 或封闭视图 (enclosing view) 色调的图像。

Color set (颜色集)

将颜色集添加到素材目录后，就可以不必对自定颜色值进行硬编码，还可以轻松构建和修改 app 的自定配色方案。颜色集内的各种颜色可以按名称进行引用，引用方法与在代码中使用 UIColor 常量创建系统定义的颜色相似。

Comparison operator (比较运算符)

比较运算符用于确定两个值的关系。一个值可以等于、大于或小于另一个值。有六种比较运算符，涵盖了这三种条件及其逆条件（不等于、不大于、不小于）。

Named color (具名颜色)

具名颜色是自定颜色，属于素材目录中的颜色集。

Named image (具名图像)

具名图像是素材目录中的图像，可通过 UIImage.init(named:) 构造器按名称进行实例化。

Repeat-while loop (repeat-while 循环)

repeat-while 循环类似于 while 循环，不同之处在于前者会在循环体末尾执行其条件检查。repeat-while 循环总是至少运行一次其主体，而 while 循环可能永远不会运行其主体。

Iteration (迭代)

迭代过程就是将同一响应动作重复多次。Swift 中的迭代是使用循环来完成的。Swift 有三种类型的循环：for-in、while 和 repeat-while。

Half-open range operator (半开区间运算符)

半开区间运算符用于表示下限和上限（不含最终数字）之间的连续整数集。例如，表达式 8..<13 代表从 8 到 12 的数字集。半开区间运算符常用于引用集合的数字索引范围，因为第一个索引是 0。0..<myCollection.count 代表 myCollection 的索引。

Closed range operator (闭区间运算符)

闭区间运算符 ... 用于表示下限和上限（均含）之间的连续整数集。例如，表达式 8...13 代表从 8 到 13 的数字集。

Git

Git 是一种版本控制系统，用于跟踪代码的不同分支以及对代码的所有修改。

Switch (开关)

一种提供二元选择的控件，如“开/关”。
