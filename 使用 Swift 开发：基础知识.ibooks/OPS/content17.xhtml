你现在已经学习了结构体是将数据和功能编译成类型的一种方式。许多编程语言还支持另一种执行类似功能的特性，称为“类”。在特殊情况下，可以（且应该）使用类来替代结构体。

类与结构体十分类似，两者都可以用作程序或 app 的构建代码块。在本节课中，你将学习类与结构体的不同之处，以及何时应使用类而不是结构体。你还将学习继承、超类和子类。

类与结构体十分类似。除了其他相似之处，二者都可以定义属性来储存值、定义方法来实现功能，以及定义构造器来设置自己的初始状态。用于执行这些操作的语法几乎都是相同的。

你可以使用 class 关键字和唯一名称来定义一个类。然后通过列出带有正确类型标注的常量或变量声明，将属性定义为 class 的一部分。与结构体一样，最佳做法是将类型名称首字母大写，并将属性名称小写：

 

class Person {

let name: String

 

init(name: String) {

self.name = name

}

}

 

可以通过在类定义中添加函数，来向类中添加功能。

 

class Person {

let name: String

 

init(name: String) {

self.name = name

}

x

func sayHello() {

print(”Hello, there!”)

}

}

 

let person = Person(name: “Jasmine”)

print(person.name)

person.sayHello()

控制台输出：

Jasmine

Hello, there!

继承

类与结构体之间的最大区别是类可以有继承关系。任何类都可以有父类和子级类。父类称为超类，子级类则称为子类。

与家庭关系一样，子类会从超类继承属性和方法，但是子类可以增加或替换超类属性和方法的实现。

在下面的小节中，你将看到如何定义基类这种没有父类的类，如何定义子类、以及如何覆盖来自基类的属性或方法。

定义基类

不从超类进行继承的类称为“基类”。到目前为止，你见过的所有类都是基类。

下面是对象 Vehicle 的基类示例：

 

class Vehicle {

var currentSpeed = 0.0

 

var description: String {

“traveling at \(currentSpeed) miles per hour”

}

 

func makeNoise() {

// 什么也不做 - 车辆不一定会发出噪音

}

}

 

let someVehicle = Vehicle()

print(”Vehicle: \(someVehicle.description)”)

控制台输出：

Vehicle: traveling at 0.0 miles per hour

Vehicle 类具有名为 currentSpeed 的属性，其默认值为 0.0。计算型变量 description 会使用 currentSpeed 属性，并返回可阅读的交通工具描述。这个类还包含名为 makeNoise() 的方法。此方法实际不会对基类 Vehicle 实例执行任何操作，而是稍后由子类进行自定。

Vehicle 类可以为任意类型的交通工具（如汽车、自行车或飞机等）定义属性和方法等通用特征。单独使用这个类并不是非常实用，但是它有助于轻松定义其他更具体的类型。

创建子类

子类生成是指在现有类的基础上创建新类的行为。你可以对子类从超类继承的属性和方法进行细化，使其更加具体。还可以向子类添加新的属性或方法。

要定义从超类继承的新类型，请将类型名称写在超类名称前面，并以冒号分隔：

 

class SomeSubclass: SomeSuperclass {

// 子类定义在这里

}

 

以下示例定义了名为 Bicycle 的子类，其超类为 Vehicle：

 

class Bicycle: Vehicle {

var hasBasket = false

}

 

新的 Bicycle 类自动继承了 Vehicle 的所有属性和方法，如 currentSpeed 和 description 属性以及 makeNoise() 方法。

除了继承的属性和方法之外，Bicycle 类还定义了新的布尔属性 hasBasket。默认情况下，Bicycle 的新实例将没有车篮。你可以在创建特定 Bicycle 实例之后，或者在该类型的自定构造器中，将其 hasBasket 属性设置为 true。

 

let bicycle = Bicycle()

bicycle.hasBasket = true

 

正如你所预期的那样，你还可以更新 currentSpeed 属性，该属性将会影响实例的描述。

 

bicycle.currentSpeed = 15.0

print(”Bicycle: \(bicycle.description)”)

控制台输出：

Bicycle: traveling at 15.0 miles per hour

 

子类自身也可以具有子类。例如，你可以创建一表示双人自行车 TandemBike 的类。

 

class Tandem: Bicycle {

var currentNumberOfPassengers = 0

}

 

Tandem 会从 Bicycle 继承所有属性和方法，后者又从 Vehicle 继承所有属性和方法。Tandom 子类还添加了一个名为 currentNumberOfPassengers 的新属性，其默认值为 0。

如果你创建了 Tandem 的新实例，那么就可以访问所有继承的属性和方法。

 

let tandem = Tandem()

tandem.hasBasket = true

tandem.currentNumberOfPassengers = 2

tandem.currentSpeed = 22.0

print(”Tandem: \(tandem.description)”)

控制台输出：

Tandem: traveling at 22.0 miles per hour

 

覆盖方法和属性

创建子类的优点之一是，每个子类都能自定属性或方法的实现。要覆盖本来会被继承的特征（例如为函数编写新的实现)，请在新定义的前面添加 override 关键字。

以下示例定义了新的 Train 类，它覆盖了 Train 从 Vehicle 继承的 makeNoise() 属性。

 

class Train: Vehicle {

override func makeNoise() {

print(”Choo Choo!”)

}

}

 

let train = Train()

train.makeNoise()

控制台输出：

Choo Choo!

 

你还可以通过提供 getter 或一段可返回值的代码（如计算型属性）来覆盖属性。以下示例定义了名为 Car 的新类，该类是 Vehicle 的子类。这个新类具有新的 gear 属性，其默认值为 1。Car 类还覆盖了 description 属性，以提供包含当前档位的自定描述：

 

class Car: Vehicle {

var gear = 1

override var description: String {

super.description + “ in gear \(gear)”

}

}

 

请注意，该新实现通过调用 super.description 来访问超类的描述，然后在描述的末尾添加一些额外的文本，以提供关于档位的信息。

 

let car = Car()

car.currentSpeed = 25.0

car.gear = 3

print(”Car: \(car.description)”)

控制台输出：

Car: traveling at 25.0 miles per hour in gear 3

 

覆盖构造器

假设你有一个具有 name 属性的 Person 类。构造器将 name 设置为指定参数。

 

class Person {

let name: String

 

init(name: String) {

self.name = name

}

}

 

现在假设要创建 Person 的子类 Student。每个 Student 都包含一个附加属性 favoriteSubject。

 

class Student: Person {

var favoriteSubject: String

}

 

如果尝试编译此代码，则 Student 将会失败，因为你还没有提供用于将 favoriteSubject 属性设置为初始值的构造器。由于 Person 超类已经进行了初始化 name 属性的工作，所以 Student 构造器可以通过 super.init() 来调用超类的构造器。在调用此构造器前，需要先为在子类中添加的所有属性赋值。

 

class Student: Person {

var favoriteSubject: String

 

init(name: String, favoriteSubject: String) {

self.favoriteSubject = favoriteSubject

super.init(name: name)

}

}

 

通过调用超类的构造器，Student 类就无需重复在 Person 构造器中已经写过的操作。

引用

类的一个特性就是能够引用赋给常量或变量的值。创建类的实例时，Swift 会在设备内存中挑选一个区域来储存该实例。该内存区域具有地址。将该实例赋给常量或变量后，其中会储存用于引用该实例的地址。

因此，常量或变量不包含值本身，而是指向内存中的值。

在将类赋给多个变量时，每个变量都会引用或指向内存中的同一个地址。所以，如果你更新了其中一个变量，两个变量都会更新。

 

class Person {

let name: String

var age: Int

 

init(name: String, age: Int) {

self.name = name

self.age = age

}

}

 

var jack = Person(name: “Jack”, age: 24)

var myFriend = jack

 

jack.age += 1

 

print(jack.age)

print(myFriend.age)

控制台输出：

25

25

 

相比之下，如果创建了结构体的实例，那么就是将字面量值赋给该变量。如果创建了与另一个结构体变量相等的变量，那么会复制该值。对任一变量执行的任何操作都将只反映在被修改的变量上。

 

struct Person {

var name: String

var age: Int

}

 

var jack = Person(name: “Jack”, age: 24)

var myFriend = jack

 

jack.age += 1

 

print(jack.age)

print(myFriend.age)

控制台输出：

25

24

 

成员级构造器

你已经了解到在创建类的实例时，必须先设置所有属性值。与对结构体的操作不同，Swift 不会为类创建成员级构造器。不过，开发者的常见做法是为自己定义的类创建自己的成员级构造器，以确保为所有对象的所有实例都设置初始值。

类还是结构体？

由于类和结构体十分类似，所以在构建代码块时可能难以判断何时使用类、何时实用结构体。

一项基本规则是，应先将新类型设置为结构体，直到需要由类提供的特性时，再将类型设置为类。

如果正在使用的框架用到了类，或者如果希望在多个位置引用某类型的同一实例，请先使用类。

使用具有类的框架

你会经常使用一些其他开发人员编写的资源，如 Foundation 或 UIKit 之类的框架。在这些情况下，可以先使用基类，然后再创建子类来添加自己的特定功能。例如，如果在使用 UIKit 时希望创建自定视图，可以创建 UIView 子类。

当使用框架时，通常需要传递类实例。许多框架的方法调用都需要某些内容是类。因此在这些情况下，应始终选择使用类而不是结构体。

稳定标识

有时你会希望在多个位置使用同一个对象实例，但是拷贝该实例却没有意义。因为每个类的常量或变量都是指向同一数据的地址，而该数据具有稳定标识。

例如 UITableViewCell 的子类 MessageCell 表示表格视图中的某一行。该单元格专门用于显示关于电子邮件消息的信息。你将看到，MessageCell 的每个实例都会被代码中的多个位置访问。

 

class MessageCell: UITableViewCell {

 

func update(message: Message) {

// 使用消息信息来更新 UITableViewCell 属性

textLabel.text = message.subject

detailTextLabel.text = message.previewText

}

}

 

当表格视图准备显示单元格时，它会调用函数 cellForRow(at: IndexPath)，请求在列表的每个位置显示单元格。在调用函数的过程中，会对表格视图的所有单元格都进行初始化（并放入内存中）。

 

override func tableView(_ tableView: UITableView, cellForRowAt

indexPath: IndexPath) -> UITableViewCell {

 

let cell = MessageCell(style: .default, reuseIdentifier:

“MessageCell”)

cell.update(message: message)

 

// 将单元格返回给调用此方法来请求该单元格的表格视图

return cell

}

当用户在表格视图中滚动邮件列表时，会调用一个方法，允许程序在即将显示的单元格中进行设置、更新或修改。函数会将引用作为参数递给单元格。

 

override func tableView(_ tableView: UITableView, willDisplay

cell: UITableViewCell, forRowAt indexPath: IndexPath) {

// 在此处对单元格执行你想要执行的任何操作

}

 

稍后你将进一步了解关于此处出现的一些陌生语法。现在重点关注即将在表格视图中显示的每个单元格，都会调用函数。此外请注意，函数还传递了三个参数：对已经存在的 tableView 的引用、已经存在但即将显示的 cell，以及名为 indexPath 的参数。

因为 tableView 和 cell 都属于类，所以参数是对内存中的 tableView 和 cell 的引用。如果更新 cell 参数，就是更新在 cellForRow 函数中初始化的那个单元格，也是即将显示的那个单元格。

思考一下类与结构体的工作方式有何区别。如果 cell 为结构体或值类型，则参数将是即将显示的单元格的新副本。因此，即使更新了 cell，也不会看到所作的更改，因为更新的是副本而不是即将显示的单元格本身。
