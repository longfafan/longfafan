Swift 简史

在 2014 年度 Apple 全球开发者大会上，Apple 推出了现代编程语言 Swift，可为 iOS 和 macOS 编写 app。Apple 现在有了包括 watchOS 和 Apple TVOS 在内的新平台，这些平台也使用 Swift 作为主要编程语言。

自 20 世纪 90 年代以来，大多数开发者都使用 Objective-C 为 Apple 平台编写 app，这是一种建立在 C 编程语言基础之上的语言。Objective-C 的使用已经超过 30 年，而 C 语言的使用更是超过 40 年。这两种语言都为软件开发者社群提供了良好的服务。所以，在可预见的未来，它们还将继续存在。

然而，Objective-C 在学习上比较有挑战性。近年来科技发展突飞猛进，Apple 看到了机会，创建了一种更易于学习、阅读、编写和维护的更加现代的语言。

在学习 Swift 时，可能会发现其中有 C 和 Objective-C 的影子。

现代语言

什么是现代语言？就是安全、快速、逻辑清晰的语言。Apple 在设计 Swift 时，将这三大主要特点融入到了每一个决策核心之中。随着你不断深入学习 Swift 编程概念，你将会逐渐理解每一个决策能够如此安全、快速且逻辑清晰的原因。

Swift 之所以成为现代语言，是因为它的一些特性，其中包括：

  • 简洁的语法，让代码更具可读性且更易于处理
  • 可选类型，当某个值可能不存在时的一种新的表示方式
  • 类型推断，可加速开发过程，并允许编译器帮助识别常见问题
  • 类型安全，强制实施更不容易让程序崩溃的代码
  • 用于内存管理的自动引用计数 (ARC)，可自动处理原生编程的一些更深层次的技术难题
  • 元组和多个返回值，用于让较小代码单元执行更多操作
  • 泛型，有助于开发者编写可在多场景中使用的代码
  • 快速简洁地对集合进行迭代，使 Swift 成为一种快速的语言
  • 支持方法、扩展和协议的结构体，用于让 Swift 对内存使用和速度进行优化，同时为开发者提供了灵活性
  • map、filter、reduce 及其他函数编程模式简化了代码，也优化了以前需要多行代码的常见操作
  • 强大的错误处理机制，有助于降低开发者编写代码时的程序错误数量，并更好地处理可能导致 app 崩溃或未按预期执行的场景

此刻，你可能还不太熟悉列表中的大多数概念。没关系，现在只要明白是这些特性让 Swift 成为适合用来构建 app 的语言就够了。随着对这门课程了解的加深，你将学习每种特性的含义，以及这些特性是如何与编写安全、快速和富有表现力的代码相关联的。

安全的语言

以上提到的许多特性都有助于编写让 app 更不容易崩溃的代码，从而让 Swift 成为了一门安全的语言。计算机程序只能按照代码进行处理，因此为处理一件事情而编写的代码可能无法处理另一件事情。类型安全会强制你明确说明所创建、操作和赋值的所有对象的“类型”，并且仅允许编写给定对象能够处理的代码。如果代码不是专门用于处理所引用对象的“类型”，那么类型安全可防止编写可能会崩溃的代码。同样，类型推断也允许编译器去推断对象的类型，从而节省时间，并再次确保编译器能够强制执行适当的规则，来限制每种类型能够执行哪些操作和函数。可选类型是一种特殊的 Swift 语法，用于更好地表示值有可能缺失的情况。这有助于确保代码能够处理存在值和不存在值的场景。Swift 还提供成熟的错误处理机制，顾名思义，即可用于编写得体、简单明了地处理错误的代码。

开源

2015 年 12 月，Apple 在 GitHub 上以开源项目的形式发布了 Swift 语言和支持资源。开源具有重大意义。关于开源最重要的一点是，开源语言是公开开发的，社群可以对其提供反馈和支持。每个人都可以贡献代码，也可以只沿用已有的代码。

开源开发则意味着 Swift 在不断发展和改进。随着时间的推移，语法可能会发生变化，并且将会添加对更多平台的支持，包括向 Linux 提供更多 Swift 技术。既然你正在学习 Swift，那么随着语言的不断改进，以及其在 Apple 平台和更多平台的采用率的提高，你可以确信自己的知识将会变得越来越有价值。

有关如何沿用或参与构建 Swift 语言的更多信息，请访问项目主页 Swift.org。

Hello World

每当开始学习一门新的编程语言时，有一个由来已久的传统就是构建“Hello, world!”app，这是你可以在任何编程语言中编写的最简单程序之一。这个程序所做的就是将“Hello, world!”打印到屏幕上。

Swift 代码是在纯文本文件中编写的，其文件扩展名为 .swift。文件中的每一行都表示一条语句，一个程序是由一条或多条语句组成的。这些语句都是希望 app 运行的指令。通常执行代码时会从文件的顶部开始，一直执行到文件的底部为止。

正如接下来将要学习的那样，可以使用控制流程语句 (if, else) 来控制是否执行代码的特定部分，使用循环 (for-in, while) 控制这些部分执行多少次，以及控制如何使用能够传递给不同语句的数据。

有些程序是由成千上万个文件中的无数个语句组成的。编译器会将所有文件合并到一个程序中，使代码可以执行。

但现在，让我们简单一点。

在 Swift 中，默认文件是 main.swift。该 main.swift 文件中包含的所有 Swift 代码都将从上到下执行。

那么如何使用 Swift 编写“Hello, world!”app 呢？

 

print(”Hello, world!”)

 

正如你可能从上述代码猜测到的那样，print() 是一个函数（更小的代码单元，用于执行特定任务），它接受一点文本，称为 String，并将其打印到控制台。

控制台是用于系统管理的文本输入和显示工具。几十年前，所有计算机交互都是通过控制台或命令行界面进行的。尽管如今大多数任务都有图形用户界面，但是开发者还是会使用控制台来执行许多编程或管理任务。实际上，你使用的几乎所有的系统都有幕后控制台，即使正在使用图形界面，也可以访问控制台。

终端

如何访问控制台? macOS 附带一个称为“终端”的控制台 app，而 Swift 则附带一个称为“REPL”的工具，代表“读入、求值、输出”循环 (Read, Eval, Print Loop)。“REPL”可用于输入简单的命令，对其进行求值，并输出结果。

使用“Swift REPL”在控制台中编写你的第一个“Hello, world!”程序。

  1. 在 Mac 上打开“终端”应用程序。可以在“聚焦”中搜索“终端”，也可以在系统的“应用程序/实用工具”文件夹中查找该应用程序。
  2. 键入 swift，再按 return 键，进入 Swift REPL。
  3. 键入命令 print("Hello, world!")，然后按 return 键执行。

请注意，“Hello, world!”就打印在屏幕上 print 命令的下方。如果你在编程方面完全是新手，那么现在可能是你第一次为计算机编写指令并让其执行。恭喜！

现在，可以通过执行以下步骤退出 Swift REPL 和“终端”：

  1. 键入 :quit，然后按 return 键，退出 Swift REPL。
  2. 退出“终端”。

Playground

Playground 是随 Swift 一起发布的最重要的内容之一，这是一种特殊的 Xcode 文稿类型，以简单格式运行 Swift 代码而且结果轻松可见。

出于多种原因，相对于使用“REPL”来说，使用 playground 更为可取。对于初学者来说，使用 playground 编写 Swift 代码简单有趣。通过使用更熟悉的界面，可以键入一行代码，然后结果会立即出现在侧栏中。如果代码需要运行一段时间，可以在时间轴上看到进度。

Playground 对于 Swift 开发者也非常有用。使用这些 playground 可以轻松在一个位置管理更多代码，并在结果边栏查看多个结果。开发者经常使用 playground 对 Swift 代码进行原型设计，然后在测试之后，再将其移动到 Xcode 项目中。

从技术层面上来说，playground 是 main.swift 文件的文件包装器。每次只要编辑代码，playground 就会运行结果。你可以包含其他文件和资源，这也是在将来的活动中要进行的操作。

 

 

要查看 Swift 代码的打印结果，可以在 playground 中打开一个调试控制台区域。

请记住，每次编辑 Swift 代码时，代码都是从上到下运行的。每个表达式的结果都显示在右侧。

  2. 将 playground 命名为“Hello, world!”，并将其存储到课程资源文件夹。

 

你是否注意到默认的 playground 附带一个 import 语句和变量 var str = “Hello, playground”？打开结果边栏（如果尚未显示的话），并注意字符串值已在边栏中打印。

import UIKit 语句允许你使用 UIKit 框架中的任意内容。当你完成这门课程中的练习时，这条语句将会变得更加重要。通常，应该在所有的 playground 中都保留该 import 语句。

  1. 刚才使用 Swift REPL 编写了 print(”Hello, world!”)，向控制台打印“Hello, world!”。现在，在 playground 中试一下。另起一行，键入 print(”Hello, world!”)。

现在，应该能在结果边栏和调试控制台中看到“Hello, world!”了。

这个 playground 中发生了什么？该 playground 文件模板中的代码 var str = “Hello, playground” 创建了名为 str 的变量，并将其设置为等于包含“Hello, playground”的文本字符串。与处理“Hello, world!”的方式一样，由于 str 值是一个 String，因此可以将该值传递给 print 函数。

  1. 更新代码以打印 str 变量，而不是“Hello, world!”文本，方法是替换“Hello, world!”为 str。

 

现在先暂停一下，思考一下为何会这样。print 函数接受了 String 参数，并将其打印到控制台。这两个示例都能正常运行，是因为“Hello, world!”是一个 String，而 str 是一个容纳 String 的变量。
