无论何时处理数据，类型都扮演最重要的角色。例如，如果函数返回 Int，你就知道可以在数学表达式中使用其值。但如果类型信息不够具体，你需要仔细检查数据才能确定如何使用数据，该怎么做？

在本节课中，你将学习为何有些数据只能用更广义的类型来表示，以及如何在使用之前测试数据的具体种类。

内容提要

  • 如何将不同类型的值混合到同一个集合中
  • 如何在异构集合中检查值的具体类型
  • 如何先将对象向下转换为特定类型然后再访问其属性和方法

词汇

  • as!
  • as?
  • Any
  • AnyObject
  • Conditional cast (条件转换)
  • Downcast (向下转换)
  • Type casting (类型转换)
  • Type inspection (类型检查)

类型转换

假定李明的工作是去客户家里照顾宠物。在他到达每位客户的家里时，会根据动物的不同种类完成不同的任务。如果宠物是狗，那么他要遛狗。如果是猫，那么他要换猫砂。如果是鸟，那么他要清洁鸟笼。

在 Swift 中，函数的声明决定了要返回的数据类型。因为函数不能返回 Dog、Cat、Bird，所以最好的方法是返回这三种宠物的父类型 Animal。

 

func getClientPet() -> Animal {

// 返回宠物值

}

 

let pet = getClientPet() // pet 属于 Animal 类型

 

可惜这个类型太宽泛了，对李明来说没什么用。因为不知道具体的动物类型，李明的工作就可能会意外变成遛鸟、换狗砂、清洁猫笼。可以使用下列带有 Dog、Cat、Bird 参数的有效函数。

 

func walk(dog: Dog) {

print(”Walking \(dog.name)”)

}

 

func cleanLitterBox(cat: Cat) {

print(”Cleaning the \(cat.boxSize) litter box’”)

}

func cleanCage(bird: Bird) {

print(”Removing the \(bird.featherColor) feathers at the

bottom of the cage’”)

 

李明需要了解 Animal 的具体 pet 子类。你可以使用 as? 运算符来尝试将值向下转换为更具体的类型，并将其储存在新的常量中。此运算叫做条件转换，这是因为只要可能，它就会将实例转换为指定的类型。在转换类型之前，应该使用 if-let 语法来检查条件：

 

let pets = allClientAnimals()

 

for pet in pets {

if let dog = pet as? Dog {

walk(dog: dog)

} else if let cat = pet as? Cat {

cleanLitterBox(cat: cat)

} else if let bird = pet as? Bird {

cleanCage(bird: bird)

}

}

 

现在李明可以确定他要遛狗、换猫砂、清洁鸟笼了。

还有一种强制形式的类型转换运算符：as!。此运算符会强制向下转换为指定类型。但如果指定的类型不正确，就会导致程序崩溃，结果跟强制解包可选类型一样。

假设阿兰要去宠物店接自己的宠物狗回家。fetchPet(for customer: String) 函数可能会返回 Animal 类型。

 

let alansDog = fetchPet(for: “Alan”)

// alansDog 被推断为 Animal 类型

 

如果你确定返回的对象类型会更具体，就可以使用 as! 运算符立即转换值。

 

let alansDog = fetchPet(for: “Alan”) as! Dog

// alansDog 被推断为 Dog 类型

 

开始构建 app 后，你会发现在使用 UIKit 时，API 会返回非常通用的对象，如 UIViewController。但作为应用程序的开发者，你知道应该是哪种具体类型。例如，如果按下 FirstViewController 视图中的按钮始终显示 SecondViewController，那么就可以在 prepare(for segue:) 函数中强制将 destination 向下转换为 SecondViewController。每当使用 Storyboard 过渡加载新的视图控制器，都就会调用此函数。

 

class SecondViewController: UIViewController {

var names: [String]?

}

 

func prepare(for segue: UIStoryboardSegue, sender: Any?) {

let secondVC = segue.destination as! SecondViewController

secondVC.names = [”Peter”, “Jamie”, “Tricia”]

}

 

只有在确定具体类型正确时，才能使用 as!。

 

Any

你已经学过，数组在默认情况下是设置用于处理具体类型的，如 Int、String 或 Animal。因为你知道异构集合中每个实例的类型，所以使用异构集合相对比较简单。

但是如果希望使用非具体类型，该怎么做？Swift 提供两种特殊类型：Any 和 AnyObject。顾名思义，Any 可以代表任何类型的实例：字符串、双精度浮点型、函数等等。AnyObject 可以代表 Swift 内任意类的实例，但不能代表结构体。

以下数组示例可包含任何类型（或 [Any]）的实例：

 

var items: [Any] = [5, “Bill”, 6.7, Dog()]

 

因为上面这个数组可以包含任何类型的实例，所以无法保证任何项目的类型。例如，如果你使用 items[0] 来访问以下数组中的第一项，那么它将返回值 5，其类型为非具体类型 Any。要继续并使用 firstItem 的值作为 Int，应该使用 as? 运算符：

 

var items: [Any] = [5, “Bill”, 6.7, true]

if let firstItem = items[0] as? Int {

print(firstItem + 4) //9

}

 

虽然 Any 可以跟其他类型一样使用，但最好还是使用你预期要处理的具体类型。在代码中使用 Any 或 AnyObject 之前，应确定需要这些特殊类型所提供的行为和功能。
