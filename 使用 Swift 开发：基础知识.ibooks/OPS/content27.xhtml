Swift 最强大的功能之一就是阅读代码和迅速理解数据的能力。当不确定某个函数是否能返回数据时，Swift 会强制对两种可能的场景做出适当处理。

Swift 使用称为“可选类型”的独特语法来处理此类情况。在本节课中，你将学习使用可选类型来正确处理不确定数据是否存在的情况。

Nil

可选类型适用于不确定值是否存在的情况。可选类型表示两种可能性：有值并且可以使用该值；根本没有值。

假设你要为书店构建一个列出销售书目的 app。你有一个结构体对象 Book，它具有 name 和 publicationYear 的属性。

 

struct Book {

var name: String

var publicationYear: Int

}

 

let firstHarryPotter = Book(name: “Harry Potter and the

Sorcerer’s Stone”, publicationYear: 1997)

let secondHarryPotter = Book(name: “Harry Potter and the

Chamber of Secrets”, publicationYear: 1998)

let thirdHarryPotter = Book(name: “Harry Potter and the

Prisoner of Azkaban”, publicationYear: 1999)

 

let books = [firstHarryPotter, secondHarryPotter,

thirdHarryPotter]

 

 

到目前为止，一切正常。现在假设你要构建一个显示书目列表的屏幕，其中的书目已发布但尚未出版。

没有出版日期，要如何初始化这些书目呢？如何为 publicationYear 赋值呢？

 

let unannouncedBook = Book(name: “Rebels and Lions”,

publicationYear: 0)

 

“零”值不准确，因为那意味着这本书已经有 2,000 多年的历史了。

let unannouncedBook = Book(name: “Rebels and Lions”,

publicationYear: 2019)

 

本年度或者甚至下一年度也不准确，因为这本书可能在两年后才会发布。发布日期未知。

nil 表示缺少值，或者没有值。因为还没有 publicationYear，所以 publicationYear 应为 nil。

 

let unannouncedBook = Book(name: “Rebels and Lions”,

publicationYear: nil)

 

现在代码看起来比较正确了，但编译器却抛出错误。初始化期间必须设置所有实例属性，不能将 nil 传递给 publicationYear 参数，因为该参数需要 Int 值。

可选类型通过为可能存在的值提供包装器解决了此问题。你可以把可选类型想象成一个箱子，箱子打开时，里面或者包含期望类型的实例，或者什么都没有 (nil)。

每个类型都有一个与之匹配的“可选类型”类型，通过在原始类型名称之后添加一个 ? 即可声明相匹配的可选类型。

在本例中，需要将 publicationYear 属性的类型标注更新为 Int? ，即一个 Int 的可选类型。

 

struct Book {

var name: String

var publicationYear: Int?

}

 

let firstHarryPotter = Book(name: “Harry Potter and the

Sorcerer’s Stone”, publicationYear: 1997)

let secondHarryPotter = Book(name: “Harry Potter and the

Chamber of Secrets”, publicationYear: 1998)

let thirdHarryPotter = Book(name: “Harry Potter and the

Prisoner of Azkaban”, publicationYear: 1999)

 

let books = [firstHarryPotter, secondHarryPotter,

thirdHarryPotter]

 

let unannouncedBook = Book(name: “Rebels and Lions”,

publicationYear: nil)

 

指定可选类型的类型

请注意，创建可选类型时必须指定类型。请思考一下，如果试着让 Swift 推断类型，那么会产生什么结果。

在此示例中，类型推断将假定变量是非可选类型：

 

var serverResponseCode = 404 // Int 整型，不是可选类型 Int？

 

在此示例中，如果数据不是 nil，那么类型推断就没有任何信息来确定数据的类型：

 

var serverResponseCode = nil // 错误，当不是 nil 时没有指定类型

 

考虑到这些原因，在大多数情况下，你将需要在创建可选类型变量或常量时使用类型标注来指定类型。看看以下代码如何正确设置 Int? 可选类型：

 

var serverResponseCode: Int? = 404 // 设置为 404，但稍后可能为 nil

 

var serverResponseCode: Int? = nil // 设置为 nil，但稍后可能会拥有一个 Int 值

使用可选类型值

如何确定可选类型是否包含值？如何访问该值？可以先使用 if 语句来比较可选类型与 nil。如果值不是 nil，再使用强制解包运算符 ! 来解包该值。

 

if publicationYear != nil {

let actualYear = publicationYear!

print(actualYear)

}

 

如果跳过可选类型与 nil 的比较环节，而对一个不包含值的可选类型进行强制解包，那么代码会产生错误，并会在尝试运行时崩溃。

 

let unwrappedYear = publicationYear! // 运行错误

print(unwrappedNumber)

 

看起来在使用包含的值之前，最好先比较可选类型是否为 nil，但这样做也感觉有点多余。回想一下，Swift 的主要设计目标是安全性和清晰度，它所提供的简洁语法，有助于安全地使用可选类型包含的值，并在可选类型不包含值时避免错误。

可选类型绑定会对可选类型进行解包，如果其中包含值，就将该值赋给常量作为非可选类型，这是安全的做法。如果使用此方法，就无须烦恼你所使用的是值还是 nil。

可选类型绑定的语法如下所示：

 

if let constantName = someOptional {

// constantName 在括号内将被安全的解包

}

 

如果 someOptional 具有值，那么将会把该值赋给 constantName，并且使用时必须以花括号括住。

看看在前面的 Book 示例中，可选类型绑定如何发挥作用：

 

if let unwrappedPublicationYear = book.publicationYear {

print(”The book was published in \

(unwrappedPublicationYear)”)

}

 

就像其他 if 语句一样，此处也可以添加 else 子句。

 

if let unwrappedPublicationYear = book.publicationYear {

print(”The book was published in \(unwrappedPublicationYear)”)

}

else {

print(”The book does not have an official publication date.”)

}

 

函数和可选类型

Swift 提供了许多能够返回可选类型值的函数。

例如，将 String 的值设为“123”。你会看到此处 string 可以转换为 Int。

 

let string = “123”

let possibleNumber = Int(string)

 

但是，如果无法转换该字符串怎么办？

 

let string = “Cynthia”

let possibleNumber = Int(string)

 

Swift 会将 possibleNumber 推断为 Int? 类型，因为 Int 的构造器接受 String 作为参数，但不一定能够成功将 String 成功转换为 Int。如果可以将 string 转换为 Int，那么 possibleNumber 将具有该值。如果无法转换，那么 possibleNumber 将为 nil。

如果要编写一个接受可选类型作为自变量的函数，只需在参数列表中更新该类型即可。例如下面这个打印函数，它接受 firstName、middleName 和 lastName，但允许 middleName 为 nil，因为不是每个人都有中间名。

 

func printFullName(firstName: String, middleName: String?,

lastName: String)

 

对返回可选类型的函数来说也一样：只需更新返回类型即可。例如，一个网站 URL 会返回该网页上的文本。返回的文本是可选类型，因为该 URL 可能无效，也可能不返回任何文本。

 

func textFromURL(url: URL) -> String?

 

可失败构造器

任何可能返回 nil 的构造器都称为可失败构造器。在本节课前面的部分中，你已经看到 Int 构造器如何尝试从 String 创建 Int，并在无法转换 String 的情况下返回 nil。

为了提高可控性和安全性，最好创建自己的可失败构造器，并定义逻辑以返回实例或 nil。

例如，Toddler 的定义如下所示：

 

struct Toddler {

var name: String

var monthsOld: Int

}

 

在此示例中，每个 Toddler 都必须有名字，还必须有月龄。但你可能不想为月龄小于 12 个月或大于 36 个月的幼儿创建实例。为了具有这样的灵活性，你可以使用 init? 来定义可失败构造器。问号 (?) 告诉 Swift 此构造器可能会返回 nil， 或者返回 Toddler? 类型的实例。

在构造器的主体中，你可以检查提供的月龄是否小于 12 或大于 36。如果任一情况成立，构造器就会返回 nil，而不是向 monthsOld 赋值。

 

struct Toddler {

var name: String

var monthsOld: Int

 

init?(name: String, monthsOld: Int) {

if monthsOld < 12 || monthsOld > 36 {

return nil

} else {

self.name = name

self.monthsOld = monthsOld

}

}

}

 

在使用可失败构造器创建 Toddler 实例时，会始终返回可选类型。要安全地对值进行解包以供后续使用，你可以使用可选类型绑定：

 

let toddler = Toddler(name: “Joanna”, monthsOld: 14)

 

if let myToddler = toddler {

print(”\(myToddler.name) is \(myToddler.monthsOld) months old”)

} else {

print(”The age you specified for the toddler is not between 1

and 3 yrs of age”)

}

可选类型链

可选类型值也可以具有可选类型属性，可以将其想象成箱子套箱子。这称为“嵌套可选类型”。

在以下示例中，注意每个 Person 都有 age，也都可能有 residence。Residence 可能有 address，但不是每个 Address 都有 apartmentNumber。

 

struct Person {

var age: Int

var residence: Residence?

}

 

struct Residence {

var address: Address?

}

 

struct Address {

var buildingNumber: String

var streetName: String

var apartmentNumber: String?

}

 

对嵌套可选类型进行解包可能需要很多代码。以下示例会检查个人的地址，看看他/她是否住公寓。要针对某个给定的 Person 对象执行此操作，必须对 residence 可选类型、address 可选类型和 apartmentNumber 可选类型进行解包。如果使用 if-let 语法，那么必须执行很多条件解包：

 

if let theResidence = person.residence {

if let theAddress = theResidence.address {

if let theApartmentNumber = theAddress.apartmentNumber {

print(”He/she lives in apartment number \

(theApartmentNumber).”)

}

}

}

 

还有一种更好的方法可以执行此操作。你无需将名称赋给每个可选类型，而是可以使用一种称为可选类型链的构造来根据条件对每个属性进行解包。如果某人有住所，其地址中有公寓门牌号，而该公寓门牌号可以转换为整数，那么就可以使用 theApartmentNumber 来引用该门牌号，如下所示：

 

if let theApartmentNumber =

person.residence?.address?.apartmentNumber {

print(”He/she lives in apartment number \(theApartmentNumber).”)

}

 

在将可选类型组合成链时，? 会显示在链中的每个可选类型之前。

如果 nil 值在任意位置中断了该链，那么 if let 语句会失败。这样就不会将任何值赋给常量，而花括号中的代码也永远不会执行。如果任何值都不为 nil，那么会执行花括号内的代码，并且常量会具有值。

 

隐式解包的可选类型

除非你为对象的所有非可选类型属性指定了值，否则不能进行对象初始化。但在有些情况下，尤其是在进行 iOS 开发时，有些属性只是暂时为 nil，等到初始化之后就会具有指定值。例如，你已经使用 Interface Builder 创建了插座变量，以便访问代码中的特定接口片段。

 

class ViewController: UIViewController {

@IBOutlet var label: UILabel!

}

 

如果你是这个类的开发者，你会知道只要创建并向用户显示 ViewController，屏幕上始终会显示 label，这是因为你在 Storyboard 中添加了标签。但在 iOS 开发中，Storyboard 元素仅在初始化之后才会与对应的插座变量相连接。因此，必须允许 label 有一小会儿时间为 nil。

那么将常规可选类型 UILabel 用作类型呢？标准可选类型需要 if-let 语法来持续对值进行解包，这样就为数据可能不存在的情况提供了一种安全机制。但你知道当 Storyboard 连接到插座变量之后，label 会具有一个值，所以对非真正“可选类型”进行解包有点麻烦。

为了变通解决此问题，Swift 提供了用于隐式解包的可选类型的语法，使用感叹号 ! 而不是问号 ?。正如名称所暗示的那样，无论何时在代码中使用此类型的可选类型，都会自动解包。这样你可以在使用 label 时将其视为非可选类型，而且允许 ViewController 不使用它即可进行初始化。

隐式解包的可选类型仅应在一种特殊情况下使用：需要进行对象初始化，却又不能提供值，但确定会在其他代码尝试访问该对象之前为其提供值。借助隐式解包的可选类型来避免使用 if let 语法看似很方便，但如果过度使用此方法，就等于放弃了该编程语言中一项重要的安全功能。因此，许多 Swift 开发者将代码中过多使用 ! 视为危险信号。如果尝试访问隐式解包的可选类型的值，而该值为 nil，程序将崩溃。
