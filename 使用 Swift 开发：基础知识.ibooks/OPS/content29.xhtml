大部分程序错误源于代码过于复杂。代码越容易阅读，就越容易找到潜在的程序错误。

在本节课中，你将学习如何使用 guard 更好地管理控制流程。你将学习如何预先处理无效值和特例值，这样就不用将它们交织在整个编程逻辑中。

内容提要

  • 如何使用 guard 来编写可读性更好的代码
  • 如何构建用于防范无效自变量的函数

词汇

  • guard
  • guard-let

相关资源

  • Swift 编程语言指南：提前退出

在开始处理更复杂的 app 时，你会需要编写一些取决于一系列 true 条件然后才能执行的函数。但是代码中的条件越多，阅读代码和调试就越难，尤其是当 if 语句是控制流程的唯一形式时。

你已经学过，每个 if 语句都会对布尔值表达式求值：如果结果为 true，那么将执行在该语句中定义的代码；否则，将跳过。但是如果你有多个 if 语句彼此嵌套，该怎么做？那样你的代码就会形成程序员所说的“金字塔厄运”：

func singHappyBirthday() {

if birthdayIsToday {

if !invitedGuests.isEmpty {

if cakeCandlesLit {

print(”Happy Birthday to you!”)

} else {

print(”The cake’s candles haven’t been lit.”)

}

} else {

print(”It’s just a family party.”)

}

} else {

print(”No one has a birthday today.”)

}

}

 

这个示例的问题出在哪呢？代码随着每个 if 语句从每行的开始就越走越远，这样的代码很难阅读。而每个 else 语句都从对应的 if 开始越走越远，真的很难判断它们到底跟谁匹配。可以改写此逻辑以减轻金字塔效应，同时仍使用 if 语句：

 

func singHappyBirthday() {

if !birthdayIsToday {

print("No one has a birthday today.")

return

}

 

if invitedGuests.isEmpty {

print("It's just a family party.")

return

}

 

if cakeCandlesLit == false {

print("The cake's candles haven't been lit.")

return

}

 

print("Happy Birthday to you!")

}

 

但你可以更进一步，使用 guard 语句向阅读此代码的人清楚表达：必须满足特定条件后才能继续。

 

func singHappyBirthday() {

guard birthdayIsToday else {

print(”No one has a birthday today.”)

return

}

 

guard !invitedGuests.isEmpty else {

print(”It’s just a family party.”)

return

}

 

guard cakeCandlesLit else {

print(”The cake’s candles haven’t been lit.”)

return

}

 

print(”Happy Birthday to you!”)

}

 

guard 语句的 else 代码块只有在表达式返回 false 时才执行，与 if 语句正相反，if 语句是在返回 true 时，才执行代码块。

 

guard condition else {

// false 时执行的代码

}

 

// true 时执行的代码

 

利用此设计，可以编写函数在无法完成任务时提前返回。guard 语句要求你在 else 情况下使用 return 关键字退出函数的作用域。通过使用 guard 语句删除所有不想要的条件并调用 return，就可以将条件检查移至函数顶端，并将期待运行的代码放在底端。想要执行的代码不再受到意外条件的限制。这也向读者清楚传达了作者针对条件检查的意图和推理，这与使用 if 语句时无需 return 不同。

这里是 divide 函数的两个版本，其中一个使用 if 语句，另一个使用 guard 语句。因为零不能做除数，所以该函数的每个版本都会在除数不为零时打印结果。

 

func divide(_ number: Double, by divisor: Double) {

if divisor != 0.0 {

let result = number / divisor

print(result)

}

}

 

func divide(_ number: Double, by divisor: Double) {

guard divisor != 0.0 else {

return

}

let result = number / divisor

print(result)

}

 

如果传入的除数为 0，则使用 guard 编写的代码会提前 return。在到达实际执行除法计算的行时，早已删除了所有错误的参数。

上述 if 示例还可以采用类似于使用 guard 语法的示例的方式编写，方法是对检查中的条件取反。但是在这些情况下，使用 guard 效果更好，可读性也更高。

 

func divide(_ number: Double, by divisor: Double) {

if divisor == 0.0 { return }

let result = number / divisor

print(result)

}

结合使用 guard 和可选类型

在前面的课程中，已经学习了关于可选类型的内容以及当可选类型包含值时函数应执行任务。在使用 if-let 语法对可选类型进行解包以将其绑定到常量时，常量必须位于花括号内才有效。

 

if let eggs = goose.eggs {

print(”The goose laid \(eggs.count) eggs.”)

}

// eggs 在此处不可用

 

你可以改为使用 guard let 将可选类型中的值绑定到位于花括号外的常量。

 

guard let eggs = goose.eggs else { return }

// eggs 在此之后可用

print(”The goose laid \(eggs.count) eggs.”)

 

请注意将 else 语句放在条件之后。这种放置方式意味着 guard 会检查是否存在 true 或非可选条件。如果你试图解包的值为 nil，那么会执行 else 代码块中的代码，否则会继续执行右花括号之后的代码行，并且解包的值可供使用。

if let 和 guard let 都可以在一个语句中解包多个可选类型。但是，使用 guard let 这样做会使所有解包的值在函数其余部分中均可用，而不是仅在控制流程花括号范围内可用。

 

func processBook(title: String?, price: Double?, pages: Int?) {

if let theTitle = title,

let thePrice = price,

let thePages = pages {

print(”\(theTitle) costs $\(thePrice) and has \(thePages)

pages.”)

}

}

 

func processBook(title: String?, price: Double?, pages: Int?) {

guard let theTitle = title,

let thePrice = price,

let thePages = pages else {

return

}

print(”\(theTitle) costs $\(thePrice) and has \(thePages) pages.”)

}

 

使用 guard 语句来移动条件代码，可以提高程序的可读性。在整个课程中，你已经看到了别人编写的大量代码。你可能已经发现，如果能快速找到核心功能，而不必在验证代码中大海捞针，那理解代码就会变得容易得多。
