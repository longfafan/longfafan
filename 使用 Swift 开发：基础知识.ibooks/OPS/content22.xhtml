当你第一次学习 Interface Builder 时，你向屏幕中添加了一个按钮，并在轻点该按钮时进行打印。在本节课中，你将运用新学的视图和控件知识来回顾这个练习，并使用开关和滑块来重复这个练习。

按钮

正如你自己使用 app 的直接体验一样，按钮是最常见的输入控件类型。向主场景添加一个按钮，并让该按钮在被轻点时向控制台打印一个说明。

  1. 使用 iOS“App”模板创建一个名为“CommonInputControls”的新项目。​

 

  1. 打开 Main.storyboard。然后在“Object”库中找到一个“Button”对象。


  1. 将该按钮拖移到场景中，并将其放置在距离屏幕顶部约三分之一的位置。打开助理编辑器来查看对应的 ViewController.swift 文件。

 

  1. 按住 control 键从按钮拖移到视图控制器代码，以添加 IBAction。一定要在花括号内添加响应动作来定义 ViewController 类。在此类全新的视图控制器文件中，大多数开发者都会在 viewDidLoad() 代码块下添加新的响应动作。

 

请为响应动作使用描述性名称，如 buttonTapped。这对于你以后查看 ViewController.swift 文件中的代码非常有帮助，因为从响应动作的名称即可看出谁将触发它。

 

当用户轻点按钮并在按钮的边界之内松开时，该按钮自然会执行代码。这就是为什么当你从按钮创建 IBAction 时，默认控件事件为“Touch Up Inside”的原因。你还可以使用将与“Touch Up Inside”同时被触发的“Primary Action Triggered”控件事件。如果希望执行代码以响应不同控件事件，可以从弹出菜单的选项中进行选择。

 

请记住，当你创建 IBAction 时，就会将该函数作为参数传递给 sender。sender 指触发响应动作的特定控件。在本例中，sender 是受到轻点的按钮。

 

@IBAction func buttonTapped(_ sender: Any) {

 

}

 

如果你知道 sender 将始终为同一类型的对象（如此示例中的按钮），则可以将 Any 更改为具体的控件类型（此示例中是 UIButton）。要执行此更改，请访问 IBAction 内的 sender 的属性。

  1. 当用户轻点按钮时，使用 print() 函数可在控制台中打印一行文本。

 

print(”Button was tapped!”)

 

你可以使用同样的工作流程将任意代码或逻辑放置在控件的响应动作之内。

 

  1. 在“Simulator”中运行 app，并尝试点按你刚刚创建的按钮。你应该可以在控制台区域看到打印的文本。

 

现在，按钮上的文本与其将会触发的响应动作并不相关。使用 Interface Builder 更新按钮的文本，使其对即将执行的代码更具描述性。

你还可以设置包括其字体、对齐方式和阴影在内的其他按钮属性。要感受哪些内容可以调整，请花些时间体验一下“Attributes”检查器中的各种选项。

开关

开关用于针对单个选项进行切换。当开关以某种方式切换时，可以使用 IBAction 来执行代码。或者，可以通过访问 sender 参数或 IBOutlet 的 isOn 值来查看目前开关是切换打开还是关闭的位置。

向“CommonInputControls”项目添加一个开关，并在用户切换控件时打印开关是处于打开还是关闭状态。

 

  1. 在“Object”库中找一个“Switch”对象并将其拖移到场景中。

 

  1. 使用助理编辑器，通过开关向 ViewController.swift 文件添加 IBAction。与按钮响应动作一样，请使用如 switchToggled 之类的描述性名称，并将 sender 类型设置为 UISwitch。添加将在控制台中打印开关是处于打开还是关闭状态的代码。

@IBAction func switchToggled(_ sender: UISwitch) {

if sender.isOn {

print(”The switch is on!”)

} else {

print(”The switch is off.”)

}

}

 

  1. 在“Simulator”中运行 app，并切换开关。你应该可以在控制台区域看到打印的文本。

 

 

滑块

使用滑块，用户可以轻松地控制值，如调节音量或在可能有大量连续值的情况下设置某个数值。例如，在“显示与亮度”设置中，使用滑块来调节设备显示屏的亮度。

向“CommonInputControls”项目添加一个滑块，并在滑块的值发生更改时打印其值。

  1. 在“Object”库中找一个“Slider”对象并将其拖移到场景中。

 

  1. 使用助理编辑器，通过滑块向 ViewController.swift 添加 IBAction。请使用如 sliderValueChanged 之类的描述性名称，并将 sender 类型设置为 UISlider。添加将在控制台中打印滑块值的代码。

@IBAction func sliderValueChanged(_ sender: UISlider) {

print(sender.value)

}

 

  1. 在“Simulator”中运行 app，并移动滑块。你应该可以在控制台中看到打印的值。

 


文本字段

文本字段用于让用户输入如用户名或密码之类的少量文本。例如，当你在新的电子邮件草稿中输入主题时，使用的就是文本字段。

向“CommonInputControls”项目中添加标签和文本字段，并在用户轻点键盘上的“Enter”或“Done”键时，将标签的文本设置为文本字段中的当前文本。

  1. 在“Object”库中找一个“Text Field”对象并将其拖移到场景中。

 

 

  1. 在助理编辑器中，使用文本字段的“Primary Action Triggered”事件向 ViewController.swift 添加 IBAction。请使用如 keyboardReturnKeyTapped 之类的描述性名称，并将 sender 类型设置为 UITextField。使用响应动作打印该字段的当前文本。

@IBAction func keyboardReturnKeyTapped(_ sender: UITextField) {

if let text = sender.text {

print(text)

}

}

 

  1. 请记住，当用户编辑字段的内容时，文本字段也可以触发代码。在助理编辑器中，使用文本字段的“Editing Changed”控件事件向 ViewController.swift 添加 IBAction。请使用如 textChanged 之类的描述性名称，并将 sender 类型设置为 UITextField。使用响应动作打印该字段的当前文本。

@IBAction func textChanged(_ sender: UITextField) {

 

if let text = sender.text {

print(text)

}

}

 

文本字段非常灵活，应根据其最适合使用的情况进行配置。例如，可以根据用户输入电子邮件地址还是 URL 来对键盘进行设置，让其显示不同的按键。你还可以打开或关闭自动更正，或将文本字段设置为安全输入字段以在用户键入时隐藏字符。请花一些时间来探索这些选项，并在构建 app 时正确使用它们。

响应动作和插座变量

即使视图或控件并非 sender，你也可以创建插座变量，以允许响应动作访问视图和控件中的属性。

例如，你可以为某个标签设置一个插座变量，以根据按钮的响应动作来更新该标签。

在下面的步骤中，你将更改由轻点按钮所触发的 buttonTapped 函数，以访问开关的当前状态 (isOn) 和滑块的当前值 (value)。

  1. 通过按住 control 键并拖移来为开关和滑块添加 IBOutlet 引用（此方法你在助理编辑器中用过）。在视图控制器文件中，开发者通常将 IBOutlet 引用放置在 viewDidLoad() 函数上方。与为 IBAction 命名时一样，请为 IBOutlet 选择描述性的名称。对于开关和滑块的 IBOutlet，你可能想选择 switch 和 slider，但 switch 是不行的。“switch”是 Swift 中的保留关键字，因此最好使用 toggle 来为开关的 IBOutlet 命名。

@IBOutlet var toggle: UISwitch!

@IBOutlet var slider: UISlider!

 

  1. 从按钮更新 IBAction 中的 buttonTapped 函数，以打印当前的 isOn 状态和滑块的值。

@IBAction func buttonTapped(_ sender: UIButton) {

print(”Button was tapped!”)

 

if toggle.isOn {

print(”The switch is on!”)

} else {

print(”The switch is off.”)

}

 

print(”The slider is set to \(slider.value)”)

}

  1. 在“Simulator”中运行 app，并点按按钮。你应该可以在控制台中看到打印的文本。

手势识别器

每个控件都会与手势进行交互：按钮和开关配置为与轻点搭配使用，滑块配置为与轻扫搭配使用等等。一个手势只能关联一个视图，但一个视图可以有多个手势。例如，开关响应轻点，不过用户也可以通过向右或向左轻扫开关来将其打开或关闭。你可以根据需要在任意视图顶部添加自己的手势识别器，从简单的轻点到复杂的双指开合、长按和旋转都可以。

在本示例中，你将在视图控制器的视图顶部添加一个轻点手势识别器。首先，在“Object”库中搜索“gesture”来查看可用的手势识别器的完整列表。


找到“Tap Gesture Recognizer”，并拖移到应该响应轻点事件的视图顶部。在本例中，将其拖移到视图控制器的视图顶部。

 

你将看到识别器添加到“文稿大纲”中。从“文稿大纲”中选择识别器，然后打开“Attributes”检查器。不同的手势将包含不同的属性列表。你可以调整识别器所需的轻点次数（如轻点一下或轻点两下），以及触发与识别器关联的任意响应动作所需的触摸次数或手指根数。在此示例中，你希望识别用一根手指轻点一下，因此将两个值都保留为 1。

 

 

使用助理编辑器，通过识别器向 ViewController.swift 添加 IBAction。请使用如 respondToTapGesture 之类的描述性名称，并将 sender 类型设置为 UITapGestureRecognizer。下面实现的 respondToTapGesture 将确定在识别器的响应动作被调用时用户所轻点的位置，然后将在控制台中打印 x/y 值。

 

@IBAction func respondToTapGesture(_ sender:

UITapGestureRecognizer) {

let location = sender.location(in: view)

print(location)

}

 

构建并运行 app，然后尝试轻点屏幕。控制台会打印你所轻点的位置的坐标。请注意，当你与屏幕上的其他控件进行交互时，并没有触发响应动作，这是因为该手势只与视图控制器的视图相关联。

 


通过编程关联响应动作

Interface Builder 如何将控件连接到响应动作？理解如何通过代码将控件连接到响应动作会很有帮助。这至少将帮助你理解 Interface Builder 代你执行的工作。

要演示以编程方式将控件连接到方法，需要从 buttonTapped(_:) 响应动作断开现有按钮，然后使用代码将它们重新连接起来。高亮显示 Storyboard 中的按钮，然后打开“Connections”检查器。在控件事件列表中，你可以看到，当你在控件内按下手指然后抬起时，按钮执行的是 buttonTapped(_:) 方法。

 

 

轻点方法名称旁的“x”，断开按钮与响应动作之间的关联。“Touch Up Inside”将不再连接到任何方法。

 

 

要以编程方式将按钮连接到方法，你将需要在代码中引用该按钮。使用 Interface Builder 为按钮创建一个 IBOutlet。

 

@IBOutlet var button: UIButton!

 

将按钮连接到响应动作的最佳位置是在视图载入完成之后，向 viewDidLoad() 底部添加以下代码：

 

button.addTarget(self, action: #selector(buttonTapped(_:)),

for: .touchUpInside)

 

addTarget(_:action:for:) 方法会将控件连接到特定响应动作，并且需要使用三个自变量。第一个自变量为你希望执行的函数的所有者。buttonTapped(_:) 方法的所有者为 ViewController 或 self。第二个自变量为“selector”，该名称用于为对象选择要执行的方法。Swift 使用 #selector 作为找到特定方法的语法。最后一个自变量为应触发响应动作的事件。就像先前在“Connections”检查器中所看到的那样，你应该将响应动作与“Touch Up Inside”事件相关联。(其他如开关和滑块之类的控件应利用“Value Changed”事件。)

构建并运行 app，并验证轻点按钮时是否仍会执行该方法。将“CommonInputControls”项目存储到你的项目文件夹中。

 

练习：基本交互

本练习的目标是，使用两个在轻点时会更改标签内容的按钮来创建一个 app 并进行编译。

使用“Single View App”模板创建一个名为“Two Buttons”的新项目。

 

第 1 步

在 Interface Builder 中创建视图

  • 在 Storyboard 中，从“Object”库中拖移一个文本字段，并将其放置在视图的顶部。使用布局参考线来放置标签，以便其从视图的左侧页边空白延伸至右侧页边空白。将占位符文本设置为“Enter text to display in the label below”。
  • 从“Object”库中拖移两个按钮。将一个按钮放在文本字段正下方，另一个按钮放在前一个按钮正下方。它们都应在屏幕上居中显示。将顶部按钮的标题更改为“Set Text”，底部按钮的标题更改为“Clear Text”。
  • 从“Object”库中拖移一个标签，并将其放在最后一个按钮下面。使用布局参考线来放置标签，以便其从视图的左侧页边空白延伸至右侧页边空白。
  • 在“Attributes”检查器中将标签的文本设置为居中对齐。连按两次标签并删除现有文本。将新文本设置为“Placeholder”。新文本会在用户轻点某个按钮时被动态设置。你很快就会学到这一内容。

如果运行 app，你会看到这一文本字段和两个按钮。不过如果你点按按钮，也不会有什么响应。这种情况将很快得到更改。

第 2 步

创建插座变量和响应动作

  • 仍然选定 Main.storyboard，然后打开助理编辑器。视图控制器的实现文件应出现在 Storyboard 画布的旁边。
  • 在 ViewController.swift 文件中为标签创建一个 IBOutlet。
  • 此时将出现一个弹出窗口。确保连接设置为“插座变量”后，将插座变量命名为 label，类型设置为 UILabel。然后点按“Connect”。
  • 在 ViewController.swift 文件中为文本字段创建一个 IBOutlet。将插座变量命名为 textField。
  • 从“Set Text”按钮创建名为 setTextButtonTapped 的 IBAction，并从“Clear Text”按钮创建名为 clearTextButtonTapped 的 IBAction。

第 3 步

为响应动作添加代码

你已为按钮添加了响应动作，但响应动作还没有可执行的代码。在此步骤的最后，你会使响应动作能够执行某些操作。

  • 在项目导航器中选择 ViewController.swift。你将在此文件中实现响应动作。
  • 实现 setTextButtonTapped 响应动作，以将 label.text 设置为文本字段中的当前文本。

你刚才执行了哪些操作？你将新的字符串赋给了标签的文本属性。现在，当你轻点或点按“Set Text”按钮时，标签将会显示该字符串。

  • 实现 clearTextButtonTapped 响应动作，以将 textField.text 和 label.text 设置为空字符串。

现在，“Clear Text”按钮将清除文本标签和文本字段中的文本。

  • 运行 app，并通过在文本字段中添加文本和轻点“Set Text”按钮来测试 app。标签的文本应反映文本字段中的文本。现在轻点“Clear Text”按钮。文本字段和标签中的文本都应该被清除。

做得非常好！现在，你已经构建了一个使用多个控件的 app，这些控件可执行代码来更新另一个视图上的属性。接下来，你可以继续学习下一课了。请务必将你的作品存储到你的项目文件夹中。
