你已经学过视图控制器管理 app 中的不同场景。但随着 app 变得越来越复杂，你会发现需要使用不同视图控制器的不同场景来显示信息。此外还需要在不同场景之间转换，让用户可以在 app 中导航。

在本节课中，你将学习如何使用过渡从一个视图控制器转换到另一个，如何定义视图控制器之间的关系，导航控制器如何帮助你管理显示相关或层次结构内容的场景。

内容提要

  • 如何从一个视图控制器移动到另一个
  • 如何添加和自定导航控制器
  • 如何将信息从一个视图控制器传递到另一个

看一下你手机上最常用的 app。有哪个 app 只有一个屏幕而且外观从不改变吗？可能没有吧。大多数 app 都有很多场景，用于显示不同类型的信息。每个场景都由单独的视图控制器实例或类提供支持。

作为开发者，你的工作是让用户可以轻松从一个场景移动到另一个场景。可以使用 Interface Builder 来添加不同场景之间的过渡或转换。还可以通过将具有相关内容的场景包含在一个导航控制器中来创建这些场景之间的特殊关系。

在学习了不同过渡类型之后，还会学习如何创建导航控制器中不同场景之间的过渡。在掌握了如何使用过渡和导航控制器之后，你将能够构建更复杂的界面和导航层次结构，这样也就能帮助你构建功能更强大的 app 了。

 

过渡

过渡是指从一个视图控制器到另一个的转换。过渡通常在用户轻点某个按钮或表行时开始，在加载新视图控制器时结束。在 Interface Builder 中定义过渡的方法跟创建插座变量和响应动作类似，方法都是连接起始点和结束点，点按并从一个场景拖移到另一个场景。此外还可以使用编程的方式来实现过渡。

除了转换之外，过渡还可以定义视图控制器的显示方法。有一种常见方法是模态显示，这种方法会将一个新的视图控制器放置在前一个视图控制器的上面。在较小的屏幕上，模态显示将始终以全屏显示。要使 UI 适应较大屏幕的设备，可以自定模态显示，使其以弹出窗口、表单或全屏形式显示。

在模态显示新的视图控制器时，可以使用过渡让用户关闭新的视图控制器，并返回前一个视图控制器。

创建触发的过渡

为了实践视图控制器之间的转换，你要创建一个简单的 app，它可以在交通信号灯的不同颜色之间循环。开始时，使用 iOS“App”模板创建新的 Xcode 项目。将该项目命名为“TrafficSegues”。创建项目时，请确保将界面选项设置为“Storyboard”。在项目导航器中选择 Main.storyboard 以在 Interface Builder 中打开项目。

 

Unwind 过渡

你刚刚创建了一个简短的过渡序列。尽管用户可以向下轻扫以关闭这些视图，但最佳做法是始终包含一个用于关闭模态视图的按钮。为此，你需要创建一个 Unwind 过渡。过渡是转换到另一个场景，而 Unwind 过渡则是从当前场景转换以返回到前一个显示的场景。

Unwind 过渡乍看起来好像不容易理解。通过将以 UIStoryboardSegue 作为参数的函数添加到任何场景的视图控制器定义中，就可以告知 Interface Builder 该场景是 Unwind 过渡的有效目标。

在本节课中，你刚刚添加的方法不包含任何代码，但可用于将信息从过渡的终点传回源视图控制器。

导航控制器

模态过渡是从 app 中的一个上下文转换到另一个上下文时首选的方法。例如，在 iOS 的“邮件” app 中，轻点“编写”按钮 从阅读消息转换为编写消息。如果用户选择返回前一个上下文，则“Cancel”按钮始终可用。

但在某些情况下，要求过渡从一个视图控制器转换到相关的视图控制器。例如，当用户轻点“Settings”中的单元格时，新的视图控制器会以动画效果从右到左覆盖屏幕，看起来就是添加到了显示的视图控制器的堆栈上。在堆栈顶层添加一个新的视图控制器的操作称为压入堆栈。以下视频演示了推送转换的默认动画，并与“设置”、“通讯录”和“健康”app 中的模态转换进行比较。

轻点左上角的“Back”按钮或往回轻扫，以关闭顶层视图并返回层级第二高的视图控制器，动画效果是从左到右。关闭堆栈顶层的视图控制器的操作称为从堆栈取出。

导航控制器管理视图控制器的堆栈，并提供在相关视图之间导航时的动画效果。

 

 

压入和取出结构就像洗一摞脏盘子一样。洗完并擦干一个盘子之后，就要把它放在橱柜里。你洗的第一个盘子是一摞盘子中最下边的那个，洗的最后一个盘子是最上边的那个。然后，当你从橱柜里拿干净盘子的时候，最先拿到的是你洗的最后一个盘子。

现在假设每个盘子都是一个被压入屏幕的视图控制器。随着你继续压入新的视图控制器，第一个（称为根）视图控制器在堆栈中的位置越来越往下。轻点“Back”按钮很多下最终将返回到根视图控制器，此时“Back”按钮将随之消失。每个导航控制器都有一个根视图控制器。

还可以将导航控制器想象成对分层数据结构进行镜像。对于“邮件”的情况，账户列表（根）允许你轻点以显示账户的文件夹。点击每个文件夹将显示其消息。“设置”与之类似。有一个设置类别列表（根），可以选择其中某个类别以显示其中的设置或子类别，每次都进入层次结构的更深一层。对于上述示例，将从根选择“通用”，然后是其中的“辅助功能”。将视图控制器压入堆栈会进入层次结构中更深的层次，而从堆栈取出视图控制器会沿层次结构向上返回到根。

回到“TrafficSegues”项目，可以使用导航控制器来管理红色、黄色、绿色屏幕。红色将压入到黄色，黄色将压入到绿色。

如果要将导航控制器添加到场景中，请选择红色视图控制器。接下来，点按底部工具栏中的“Embed In”按钮并选择导航控制器。或者，转至 Xcode 菜单栏，并选择“Editor”>“Embed In”>“Navigation Controller”。

 

其中任一方法都会将导航控制器放在场景的开头，并将红色视图控制器设置为根视图。

构建并运行 app 以查看有哪些改变。你可能会注意到几个重要的不同之处：

  • 红色、黄色、绿色视图控制器之间的 Show 过渡采用了“Show (Push)”，而不是“Present Modally”。（这是 Show 过渡的重要功能：它采用的显示方法取决于是用在导航控制器中还是单独使用。）
  • “Dismiss”按钮仍将展开回到红色视图控制器，但执行此操作的方法是取出视图控制器而不是关闭视图控制器。
  • 在每个视图的顶端都有一个半透明的导航栏，栏中为“Back”按钮以及标题和其他按钮提供了空间。
  • 文稿大纲中现在包括“Navigation Controller Scene”，其中又包括“Navigation Bar”。

 

导航栏

导航控制器最明显的功能就是导航栏，它显示在屏幕顶端。导航栏可能会显示标题或按钮项，有时它们也会一起显示。

在文稿大纲中选择导航栏，查看“Attributes”检查器以了解可以自定哪些属性，如栏色调、标题颜色、标题字体。（还可以选择在代码中修改这些属性。）查看 UINavigationBar 文档以获得可自定属性的完整列表。

导航项

每个 UIViewController 都有一个 navigationItem，可用于自定其导航栏。当你在较早的步骤中添加了导航控制器时，Interface Builder 自动将导航项添加到了根（红色）视图控制器。

 

 

在文稿大纲中，选择红色视图控制器的导航项，并打开“Attributes”检查器。在“Title”属性中输入“Red”。

 

 

构建并运行 app。在压入黄色屏幕时，你会注意到“Back”按钮现在显示“Red”。怎么会这样？“Back”按钮使用前一个视图控制器的标题作为其文本。但是，如果前一个视图控制器没有标题，“Back”按钮只会显示“Back”。如果要让“Back”按钮使用其他文本，如“Go To Red”，可以在红色视图控制器导航项的“Attributes”检查器的“Back Button”字段中输入该文本。

 

 

在特定情况下，Interface Builder 将向视图控制器添加导航项。如果没有添加导航项，你可以通过在“Object”库中找到导航项并将其拖移到视图控制器之上来自行添加导航项。

通过选择黄色和绿色视图控制器的导航项并使用“Attributes”检查器来创建“Yellow”和“Green”标题，从而设置这些视图控制器的标题。

除了标题和“Back”按钮之外，导航项还可以包含特殊类型的按钮（称为栏按钮），此按钮可以显示在导航栏或工具栏上。在“Object”库中找到“Bar Button Item”，在绿色视图控制器的导航栏的右上角放置一个。

 

 

点按你刚刚添加的栏按钮，然后打开“Attributes”检查器。在“System Item”弹出菜单中，你会看到常用的按钮选项，如“Add”、“Save”和“Cancel”。选择其中一个或两个以查看按钮文本更改。再尝试一下“Style”和“Tint”属性。使用“Bar Item”属性可以进一步自定按钮。例如，可以使用“Image”字段将文本标题替换为图像或图标。

现在，继续操作，将栏项的“Title”属性更新为“Pop”。（请注意，此更新会将“System Item”选项更改为“Custom”。）

接下来，可以将此新栏按钮连接到 Unwind 过渡。请参阅用于连接“Dismiss”按钮的相同步骤。在完成这些操作之后，就可以删除“Dismiss”按钮了。

Storyboard 现在包含所有过渡了。构建并运行 app 以查看刚添加的标题和按钮。请注意，点按“Pop”按钮会返回到红色视图控制器。

 

 

大标题

你可能已注意到，在“设置”等系统 app 中，主要屏幕上的导航栏标题显示的比后续场景中大得多。通过在 Interface Builder 中选择导航栏，然后在“Attributes”检查器中选择“Prefers Large Titles”框，可以将此大标题添加到你自己的 app 中。

 

 

请注意，所有标题，而不仅仅是第一个视图控制器的标题，现在都采用了大标题。通过选择相关导航项并从“Attributes”检查器中的“Large Title”下拉列表中选择一个选项，可以调整哪些视图控制器使用大标题选项。

 

“Always”选项将确保该导航项的标题始终为大标题，“Never”选项将确保该导航项的标题绝不会是大标题，而“Automatic”选项将采用导航堆栈中先前视图控制器的行为。除非你有明确的理由不这样做，否则最好让根视图控制器采用大标题，后续视图控制器采用较小的标题。

传递信息

在很多 app 中，都需要在发生过渡之前将信息从一个视图控制器传递到另一个视图控制器。例如，在“通讯录”app 中轻点姓名时，需要将关于该联系人的详细信息先呈现到“信息”屏幕，然后才能显示。

 

 

如果要了解此工作原理，请更新红色视图控制器，使其不仅包含按钮还包含文本字段。按下该按钮时，会触发过渡，Interface Builder 将使用文本字段中的任何文本作为黄色视图控制器导航项的标题。

开始时，将文本字段从“Object”库拖移到红色视图控制器上，将其至于按钮正上方。对于此示例，添加约束时不必担心。

 

 

为文本字段创建插座变量，并命名为“textField”。你需要在代码中引用该文本字段才能访问文本并相应地更新目标位置的标题。

 

现在你要设置过渡以将文本字段中的文本传递到黄色视图控制器。

每个 UIViewController 都有一个方法 prepare(for:sender:)，在发生从视图控制器的过渡之前会调用该方法。在靠近 ViewController 定义底部的位置、右括号的前边开始输入“prepare”。Xcode 将为你提供帮助以完成该方法的命名。选取 prepare(for segue: UIStoryboardSegue, sender: Any?) 并按下 return 键以添加该方法。

 

此方法的第一个自变量就是过渡本身。一个过渡包含几个属性，用于帮助在过渡中传递信息：

  • identifier：过渡的名称，可以与其他过渡区分开来。你可以使用“Attributes”检查器在 Interface Builder 中设置此属性。
  • destination：将在过渡完成时显示的视图控制器。当值是 UIViewController 时，可能需要向下转换为特定的 UIViewController 子类，以便访问只有在该子类上才可访问的属性。

因为红色视图控制器上只有一个过渡，所以不需要 identifier。另外，因为你的目标是更新导航项的 title 属性，而每个 UIViewController 都有此属性，所以无需向下转换。因此，用于设置目标位置导航项标题的代码相当直接：

 

override func prepare(for segue: UIStoryboardSegue, sender:

Any?) {

segue.destination.navigationItem.title = textField.text

}

 

构建并运行 app。然后在文本字段中输入短字符串。在按下该按钮时，你刚刚添加的方法将更新黄色屏幕上的标题。

编写用于在场景之间传递数据的代码一开始可能有点复杂。但你会发现在屏幕之间传递数据特别有用，这样你编写的代码就可以非常灵活。在“测试”项目中，你会继续使用过渡来处理信息传递。

创建编程过渡

有时你需要使用一些逻辑来决定是否执行过渡。此时你创建的每个过渡都涉及到从按钮拖移到视图控制器。在以此方式创建过渡时，始终会在按下按钮时执行该过渡。在此部分中，你会定义几个视图控制器之间的通用过渡，并通过编程方式确定是否应执行这些过渡。

开始之前，要将应用程序恢复到初始状态。删除 Storyboard 中的所有现有过渡、控件、标签。还需要删除已创建的所有插座变量，以及 prepare(for:sender:) 和 unwindToRed(segue:) 方法。就绪时，你的 app 应该如下所示：

 

在此示例中，你会在红色视图控制器中放置两个按钮。一个按钮会过渡到黄色屏幕，另一个会过渡到绿色。该 UI 还会包含 UISwitch，它将确定是否执行这些过渡。如果启用此开关，就会执行过渡；否则，就不会发生过渡。

为了在代码中执行过渡，你需要给每个过渡一个有效的标识符字符串。通过在过渡行上轻点来选择该过渡本身。在“Attributes”检查器中，将一个标识符命名为“Yellow”，另一个命名为“Green”。

 

 

你需要检查代码中 UISwitch 的状态，以确定是否执行过渡。为了执行此操作，需要创建该开关的插座变量。打开助理编辑器，按住 control 键并从该开关拖移到 ViewController 定义中的一个有效位置。

 

@IBOutlet var segueSwitch: UISwitch!

 

按住 control 键并从该按钮拖移到 ViewController 定义中的一个有效位置，以便为每个按钮创建响应动作。

 

@IBAction func yellowButtonTapped(_ sender: Any) {

 

}

 

@IBAction func greenButtonTapped(_ sender: Any) {

 

}

 

要以编程方式执行过渡，视图控制器上存在一个名为 performSegue(withIdentifier sender:) 的方法。此方法的第一个参数接受 String，它对应于你在“Attributes”检查器中为过渡指定的标识符。sender 参数是可以提供给过渡的附加信息，它显示哪个控件触发了过渡，但在此示例中不需要此参数，可将其设置为 nil。

只有当开关设置在“打开”位置时，才在每个方法中调用 performSegue(withIdentifier:, sender:)。

 

@IBAction func yellowButtonTapped(_ sender: Any) {

if segueSwitch.isOn {

performSegue(withIdentifier: “Yellow”, sender: nil)

}

}

 

@IBAction func greenButtonTapped(_ sender: Any) {

if segueSwitch.isOn {

performSegue(withIdentifier: “Green”, sender: nil)

}

}

 

 

构建并运行应用程序。在按下每个按钮时，将触发对应的响应动作。每个响应动作都会检查开关的状态，并在启用开关的情况下执行相应的过渡。

回想一下，传递到 prepare(for:sender:) 的 UIStoryboardSegue 的 identifier 属性告诉你即将执行的过渡的名称。检查 shouldPerformSegue(withIdentifier:sender:) 的 API 文档。如何将 segueSwitch.isOn 的检查移入 shouldPerformSegue(withIdentifier:sender:) 而不是每个按钮的响应动作之内？

练习：登录

本练习的目标是创建登录屏幕，以在视图控制器之间传递用户名。你将使用视图控制器、导航控制器和过渡来创建登录屏幕和简单的登录后屏幕，使其在标题中显示用户名或与忘记用户名或密码相关的文本。

此练习中的指示说明不会涵盖约束和自动布局。但在你执行这些步骤的过程中，应该花些时间来确保视图适应不同的屏幕大小和方向。

使用 iOS“App”模板创建一个名为“Login”的新项目。

第 1 步

使用简单过渡创建 Storyboard 场景

 

  • 使用上一页中的图像配置 Storyboard 就可以有两个视图控制器，第一个针对登录屏幕，第二个针对登录后屏幕。
  • 为登录屏幕添加两个文本字段和三个按钮，还要在图像之后。将第一个按钮的文本改为“Log In”，第二个改为“Forgot Username”，第三个改为“Forgot Password”。
  • 对于这两个文本字段，要使用“Attributes”检查器来将占位符文本设置为“Username”和“Password”。如果要隐藏在密码文本字段中输入的文本，请选择“Attributes”检查器底部的“Secure Text Entry”复选框。
  • 从登录按钮到登录后屏幕，创建 Show 过渡。
  • 使用登录后屏幕视图控制器的“Attributes”检查器，找到“Presentation”选项并选择“Full Screen”。这会以全屏方式展示视图，而不是作为可通过向下轻扫来关闭的卡片。你希望用户感觉到已进入 app，登录屏幕不应在背景逗留。
  • 运行 app。检查是否能从登录屏幕过渡到登录后屏幕。由于你并未设置从登录屏幕传递信息的方式，因此登录后屏幕会是空白的。

第 2 步

添加导航控制器并准备过渡

  • 将登录视图控制器嵌入导航控制器中，并将标题添加到导航项。将其“Large Title”选项更改为“Always”。选择导航控制器的导航栏并选中“Prefers Large Titles”。
  • 选择登录后屏幕的导航项。将其“Large Title”选项更改为“Always”。

第 3 步

添加编程过渡

  • 创建从登录视图控制器（不是登录按钮）到登录后视图控制器的过渡。请务必赋予该过渡一个描述性的标识符。
  • 为其余两个按钮（“忘记用户名”按钮和“忘记密码”按钮）各自创建一个从其传出的插座变量，并为其指定描述性的名称，如“forgotUserNameButton”。
  • 创建来自每个按钮的响应动作。
  • 在每个响应动作内，调用 performSegue(withIdentifier:sender:)，传递最新创建的过渡的标识符。不必将 sender 设置为 nil，只需要将 sender 设置为受到轻点的按钮。例如，如果过渡标识符是 ForgottenUsernameOrPassword，那么按钮的响应动作内部可能类似于以下内容：

performSegue(withIdentifier: "ForgottenUsernameOrPassword",

sender: sender)

 

  • 在本节课前面的部分中，你已经学过利用 prepare(for:sender:) 可以访问调用的过渡的标识符。既然有两个可能的标识符，就需要使用控制流程语句根据调用的是哪个过渡将不同信息传递到登录后屏幕。如果调用的是登录过渡，那么登录后屏幕的标题应该是用户名。但如果轻点的是“忘记密码”按钮，那么你会希望标题为“忘记密码”。类似地，点击“忘记用户名”​
    按钮会将标题更改为“忘记用户名”。在继续阅读之前，花一分钟时间来查看是否可以使用 segue.identifier、sender、向下转换和 if 语句来独立执行此操作。
  • 即使你无法独立操作也没有关系。这是新材料。如果要根据轻点的按钮来传递不同的标题，prepare(for:sender:) 方法的主体可能看似以下内容：

 

guard let sender = sender as? UIButton else {return}

 

if sender == forgotPasswordButton {

segue.destination.navigationItem.title = “Forgot Password”

} else if sender == forgotUsernameButton {

segue.destination.navigationItem.title = “Forgot Username”

} else {

segue.destination.navigationItem.title = usernameTextField.text

}

 

  • 上面的代码将发送方转换为 UIButton，在本例中，始终会成功完成此转换。为什么呢？登录过渡是专门由一个按钮触发的，而 ForgottenUsernameOrPassword 过渡是由 performSegue(withIdentifier:sender:) 方法触发的，你将相应按钮作为发送方传入其中。随后，if 语句会检查 sender 是不是 forgotPasswordButton，并相应地设置标题。如果 sender 不是 forgotPasswordButton，那么另一个 if 语句会检查 sender 是不是 forgotUserNameButton，并相应地设置标题。如果 sender 不是 forgotPasswordButton，那就只剩下一种情况：在轻点登录按钮时，会将标题设置为用户名。
