当你构建 app 时，你希望确保它在每台 iOS 设备上都有不错的显示效果。Xcode 包括称为“自动布局”的强大系统，有助于轻松构建适合于不同屏幕大小的复杂界面。

“自动布局”依靠约束或规则来动态计算视图层次结构中所有视图的大小和位置。因此，无论用户手中拿着哪种设备或是如何放置该设备，界面的外观和工作方式都将相同。

在本节课中，你将学习“自动布局”的基本原理来构建精确设计的用户界面。

内容提要

  • 如何使用自动布局来构建精确视图

  • 如何利用堆栈视图来简化自动布局

当你开始开发适用于 iOS 的 app 时，必须考虑 app 将在哪些不同的设备上运行。你已经了解每个用户界面元素都有一定的大小和屏幕位置。这些元素的大小和位置相对于设备的大小和方向如何发生变化？

使用 iOS“App”模板创建新的 Xcode 项目。创建项目时，请确保将界面选项设置为“Storyboard”。将项目命名为“AutoLayoutPractice”。打开 Main.storyboard，使用布局参考线将“Object”库中的一个“Button”添加到屏幕左上角。

Interface Builder 提供了一种快速的方式来查看界面在不同屏幕大小上的显示效果。点按“View as”按钮（位于底部按钮栏中的“Show/Hide Document Outline”按钮旁），可显示一个列出多种 iOS 设备及其界面样式和方向的菜单。

 

请继续体验该列表中的各个选项。当你选择不同的屏幕大小、样式和方向时，画布会相应地重绘用户界面。如果已经将按钮放置在画布的左上方，则无论你选择哪种设备或方向，该按钮还将保持在同一位置。

为何使用“自动布局”？

现在让画布返回 iPhone 12/iPhone 12 Pro 的竖屏模式，然后将按钮移动到屏幕中心。利用蓝色对齐参考线来确保按钮位于视图正中心。

 

 

现在将方向切换为横屏模式。你将看到按钮不再居中显示。事实上，按钮甚至不在屏幕上！

 

这是怎么回事？按钮仍然位于你在竖屏模式中创建的 X/Y 坐标位置。如果希望按钮保持在正中心，而不受屏幕大小或方向的影响，则需要创建一组约束（或规则），用于确定按钮大小和位置。这一使用约束来设置自适应界面的系统称为“自动布局”。

创建对齐约束

底部按钮栏右侧有一组用于创建和管理约束的工具。要将按钮锁定在屏幕的中心，需要创建两个约束来定义按钮的位置：

  • 按钮的水平中心等于视图的水平中心。
  • 按钮的垂直中心等于视图的垂直中心。

首先返回竖屏模式并保证按钮在视图内居中对齐。在选定该按钮的情况下，点按“Align”按钮 ，也就是底部约束工具栏中的第二个按钮。

如此会显示一个弹出窗口，其中提供一系列对齐约束用于定义所选对象与父视图之间的关系。选中底部的两个选项“Horizontally in Container”和“Vertically in Container”，然后点按“Add 2 Constraints”。在创建了这些约束之后，你应该看到按钮顶上出现交叉的蓝色参考线，指示与父视图的水平和垂直对齐方式。

 

 

使用“View as”按钮选择不同的设备和方向。你将看到刚刚添加的对齐约束让按钮在所有视图中都保持居中对齐。

创建大小约束

那么按钮的宽度和高度呢？由于目前没有适用于按钮的大小约束，因此按钮大小是由按钮文本和字体来决定的。但是 Interface Builder 提供了多种方式来设置界面元素的宽度和高度。

首先使用“Attributes”检查器将按钮的背景颜色更改为非透明。此操作将使按钮的大小更容易看清。

 

 

现在假设你希望无论设备的屏幕大小或方向如何变化，按钮的高度始终为 60。点按“Add New Constraints”按钮 ，就是紧挨着“Align”工具右侧的按钮。

如此会显示一个弹出窗口，其中提供一系列文本字段和复选框来帮助你添加约束。选中“Height”复选框，并将值调整为 60。点按“Add 1 Constraint”来创建高度约束。

 

 

再次注意，如果使用“View as”按钮选择不同的设备和方向，你将看到所添加的大小约束使按钮大小在所有视图中保持相同。

相对于屏幕的约束

你已经固定了按钮的高度，但是如果你希望按钮的宽度根据屏幕的大小变化，该怎么做？假定你希望按钮的左右边缘分别与视图的左右边缘始终相距 20 像素。

再次点按“Add New Constraints”按钮，并注意弹出窗口顶部的四个字段。这些值定义了按钮上边缘、前边缘、下边缘和后边缘到最近的视图的距离。在本例中，由于屏幕中没有其他视图，因此这些值规定的是按钮边缘到视图边缘的距离。

“前边缘”指的是屏幕的左边缘，而“后边缘”指的是右边缘。将其称之为“前边缘”和“后边缘”，而不是“左边缘”和“右边缘”，是因为并非所有语言的阅读方向都相同。某些用户可能需要以相反方向设计 app。利用前边缘和后边缘约束，更容易实现此过程。

将左右边距都调整为 20。红色指示线将变为实线，以表示哪些边距已设置了约束。点按“Add 2 Constraints”。

 

请注意，按钮现在已扩展为与屏幕的每个边缘相距 20 像素。使用“View as”按钮选择不同的设备和方向。你将看到按钮的宽度会变化，以使按钮始终与屏幕边缘保持相同的距离。如果按钮出于某种原因而未更新，请使用 Update Frames 工具 更新按钮的边框。

安全区域布局参考线

通过选择按钮并按 Delete 键，从屏幕删除按钮。将“Object”库中的标签添加到屏幕左上角。使用“Add New Constraints”工具，在标签的上边缘、左边缘和右边缘与视图之间创建约束，将约束值均设为 0。

 

现在选择该标签，并打开“Size”检查器。可以查看刚才添加的三个约束。

 

请注意，约束关系的双方是你所选中的标签与超级视图。也就是说，标签边缘受到主视图控制器的视图边缘约束。你可以在 Interface Builder 中看到，如此设置会将标签放置在状态栏下方，遮挡住了标签的部分文本。你可能想要直接更改上边缘约束的值，让上边距等于状态栏的高度。但是，如果以后该视图控制器嵌入导航控制器中，该怎么办呢？到时候，标签不再被状态栏遮挡，但又会被导航栏遮挡。

要解决这个问题，你需要相对于视图控制器的安全区域来创建约束。安全区域在 Interface Builder 中显示为视图控制器主视图的子视图。

 

如果选择安全区域，所选中的视图控制器区域将不包含系统视图占据的部分（如状态栏和主页指示符）。虽然 Interface Builder 能够考虑到这些常用系统视图的影响，但是在运行时可能会显示其他系统栏。安全区域将根据需要针对这些系统栏进行调整。这样一来，你的内容就可以适应系统覆盖层，而避免被遮挡。

删除当前已设置的约束，方法是在“Size”检查器中一次选择一个约束并按 Delete 键。

 

在屏幕上将标签往下拖动一点，使其低于状态栏，并使用“Add New Constraints”工具在标签的上边缘和安全区域的上边缘之间创建约束。还要为标签的前边缘和后边缘添加约束。由于“Add New Constraints”工具会让视图受到最近视图的约束，因此使用该工具创建这些约束会相对于安全区域来约束标签的上边缘、前边缘和后边缘。

 

 

解决约束问题

现在删除你刚才添加的标签。在视图控制器的中心添加一个按钮，并为按钮设置两个对齐约束：一个用于使按钮在屏幕上垂直居中，另一个用于使按钮在屏幕上水平居中。现在使用“Add New Constraints”工具将其左边缘约束为与屏幕左边缘相距 20 像素，并将其右边缘约束为与屏幕右边缘相距 30 像素。

画布上出现红线指示设置有问题。你现在创建了两个互相冲突的约束，每个约束都在尝试指示按钮的 X 轴位置。要查看约束冲突列表，请点按文稿大纲右上角的红色指示器。

 

 

其中一个约束要求：“将按钮的水平中心设置为视图的水平中心”，而另一个约束要求：“按钮左边缘到视图左边缘的距离为 20 像素”。由于两个约束不能并存，因此你将需要移除一个约束。

在“Conflicting Constraints”列表中，点按右上角的红色错误指示器。选择使按钮居中的约束，然后点按“Delete Constraints”。

 

你有时会遇到难以解决的约束问题。又或者你正在向场景添加界面元素，并需要重新配置视图的布局。在任意情况下，你都可以点按“Add New Constraints”工具旁的“Resolve Auto Layout Issues”按钮 ，使用其中提供的选项来尝试自动解决约束问题。

“Update Constraint Constants”将尝试更新规则以匹配 Storyboard 场景中当前显示的内容。

“Add Missing Constraints”将尝试添加与 Storyboard 场景中当前显示的内容相匹配的新约束。

“Reset to Suggested Constraints”将清除所有约束并尝试准确地分配与 Storyboard 场景中当前显示的内容匹配的新约束。

“Clear Constraints”将移除与所选视图或所有视图的位置和大小相关联的所有规则。

 

解决约束警告

将你刚刚在设置的按钮移至场景中的不同位置。你应该会看到一个“自动布局”警告。

 

要了解警告详情，请点按文稿大纲右上角的黄色指示符。该警告通知你，按钮的位置与约束定义的位置不再一致。

 

要调整按钮的位置以使其符合约束，请选中画布中的按钮，然后点按“Update Frames”工具 。

同级间的约束

本节课到目前为止，你已经学习了如何创建约束来定义父子视图之间的关系。但是有时候你也会希望创建同级视图之间的约束。在以下练习中，你将设置多个标签来显示关于你自己的文本信息。

删除你刚刚设置的按钮，并向场景添加一个标签。连按两次标签，并键入你的姓名。将标签拖移到视图顶部附近，使用参考线将其水平居中。

 

选中标签，使用“Add New Constraints”工具将其约束为与安全区域上边缘相距 0 像素。要启用约束，你可能需要选中弹出窗口顶部字段下的红色指示线。

 

 

使用“Align”工具将标签的水平中心与父视图的中心对齐。

 

设置完一个标签之后，可以添加第二个标签，并根据同级的位置来设置新标签的位置。将另一个标签从“Object”库中拖移到视图中，并将其放置在现有标签的正下方。在新标签的文本中键入关于你自己的一些信息（比如你的一个爱好）。

现在假设你希望新标签与第一个标签具有相同的水平中心，并位于距离第一个标签下方 20 像素的位置。选中新标签，点按“Add New Constraints”按钮。在弹出窗口的顶部文本字段中输入 20，并点按“Add 1 Constraint”。

此时，Interface Builder 将显示错误，指示新标签具有垂直位置的约束，但是没有关于水平位置的信息。Interface Builder 能帮你解决此错误。

 

按住 command 键并逐个点按场景中的这两个标签来将其选中，并点按“Align”按钮。在弹出窗口中选中“Horizontal Centers”复选框，将其值设置为 0，然后点按“Add 1 Constraint”。此操作会告诉 Interface Builder 你希望所选视图具有相同的中心位置，即偏移量为 0 像素。

如果 Interface Builder 仍然显示警告，请点按“Update Frames”按钮来更新标签的边框，以匹配你刚刚指定的约束。

 

 

要练习你刚刚学习的内容，请添加三个新标签，再重复三次上述步骤来将标签放置在场景中。完成后，每一个标签应位于距离前一个标签下方 20 像素的位置，并且都应为水平居中。

 

 

堆栈视图

你是否发现上一个练习的操作有一点重复？管理多个对象的约束可能显得有些枯燥，尤其是当每个元素与前一个元素的距离都相同时。如果稍后你决定将标签之间的距离由 20 像素变为 30 像素，该怎么做？或者如果你希望将所有标签都放置在父视图的左侧而不是中心，该怎么做？你将必须花费大量时间才能逐个更新每个约束。

UIKit 提供了更智能的方法。无需创建并更新大量的单个约束，你可以使用堆栈视图来自动管理其子视图的约束。

一个堆栈视图可管理一行或一列界面元素，并按照堆栈视图中设置的属性来排列这些元素。首先，选中你刚刚创建的所有标签，然后点按“Resolve Auto Layout Issues”工具右侧的“Embed In Stack”按钮 ，并从显示的列表中选择“Stack View”。此操作会将选中的多个视图组织成一个堆栈，并会移除单个标签中的所有约束。

 

目前，你的新堆栈视图还没有任何定义其大小或方向的约束。要更改这种情况，请选中堆栈视图，然后点按“Add New Constraints”按钮。将堆栈视图的上边缘、前边缘和后边缘设置为距离父视图边缘 0 像素的位置。（你将需要选中与每个文本字段关联的红色指示器，才能启用约束。）将堆栈视图的高度约束为 300。如果需要，请点按“Add 4 Constraints”，并点按“Update Frames”按钮。

 

堆栈视图属性

 

 

既然已经将所有标签放置在一个堆栈中，那么接下来可以学习如何管理堆栈视图并定义其子视图的位置了。但是在开始之前，请将每个标签的背景设置为不同的颜色，以便清楚地显示对界面的调整效果。

 

在文稿大纲中选择堆栈视图，并打开“Attributes”检查器以显示四个主要属性。继续了解并调整这些属性。

  1. “Axis”确定视图中的元素是以垂直方式还是水平方式进行堆栈。
  2. “Alignment”描述如何放置视图中的元素。你可以选择以下任一选项：
  • Fill：每个元素都会填满堆栈的大小。例如，如果堆栈视图的宽度为 320 像素，则垂直堆栈中每个元素的宽度也将为 320 像素。
  • Leading：每个元素的前边缘与堆栈视图的前边缘对齐。
  • Center：每个元素的中心与堆栈视图的中心对齐。
  • Trailing：每个元素的后边缘与堆栈视图的后边缘对齐。
  1. “Distribution”定义堆栈视图中元素的分布方式。你可以选择以下任一选项：
  • Fill：堆栈视图会调整已经排列好的子视图的大小，以便其沿着指定的轴填满所有可用空间。堆栈视图会尽可能将第一个标签的大小设置为最大以完全填满堆栈。
  • Fill Equally：堆栈视图会调整已经排列好的子视图的大小，以便其沿着轴填满所有可用空间。系统会调整视图的大小，以便其沿着堆栈视图的轴的大小都相同。
  • Fill Proportionally：堆栈视图会调整已经排列好的子视图的大小，以便其沿着轴填满所有可用空间。如果更改堆栈视图的大小，则会按比例调整这些子视图。例如，如果两个标签的高度分别为 50 和 100，此时你将堆栈视图的高度增加 50%，则这两个标签的高度随之变为 75 和 150。
  • Equal Spacing：堆栈视图不会调整子视图的大小，而是按照同一间距放置这些子视图。
  • Equal Centering：堆栈视图不会调整子视图的大小，而是会确保每个子视图的中心到其他子视图的中心的距离相等。
  1. “Spacing”描述堆栈视图中每个元素之间的空间大小。例如，你可以将该值从 0 更改为 20，使标签之间的间距增加 20 点。

使用“Size”检查器可移除堆栈视图中的高度约束。如果堆栈视图没有指定高度，则会根据其子视图来调整大小。

 

既然你已经定义了一些属性，那么接下来请尝试向堆栈中添加新标签。或者重新排列现有标签，并调整堆栈的间距和对齐约束。假设你还未使用堆栈视图，但是却希望调整界面。那么你将需要添加、移除或调整多少个约束？

最好尽可能尝试使用堆栈视图来管理约束而非单独管理约束。堆栈视图用于快速创建精美的界面，并让你将来能够轻松修改或自定这些界面。

尺寸类

由于屏幕大小和方向的组合如此之多，因此必须构建完美适合所有 iOS 设备的界面。以 iPad 设备为例，使用分屏功能会让其变得更加复杂，因为 app 的运行空间可能会比最初预期的小得多。为了简化众多不同的情况下的用户界面开发，iOS 包含了尺寸类。

每台 iOS 设备都有一组默认的尺寸类，可在设计界面时用作指导。宽度和高度这两种尺寸不是“Compact”尺寸类就是“Regular”尺寸类，它们共同构成了一个特征集合。“View as”按钮指示当前设备的特征集合和所选的方向。例如，如果使用“View as”按钮选择“iPhone 12”/“iPhone 12 Pro”的“Portrait”模式，你将看到按钮标题中包含“(wC hR)”，表示此设备的特征集合为“Compact”宽度和“Regular”高度。

 

 

尽管以竖屏和横屏模式查看 iPad 时，它都具有“Regular”宽度和“Regular”高度尺寸类，但是也不会始终全屏运行 app。通过使用分屏功能，一款 app 只需利用三分之一、二分之一或三分之二的屏幕实际使用面积。不同大小的 iPad 在进行多任务处理时将具有不同的特征集合。听起来似乎要处理很多事情，但是此时无需考虑去适配各种各样的设备。而你只需将重点放在支持四种不同的宽度/高度特征集合，并保证界面正确即可。

特征变化

可以根据特征集合将约束和属性设置为不同的值。假设你希望堆栈视图的“Spacing”属性在大部分时间都为 0。但是，在具有“Regular”宽度和“Regular”高度的设备上运行时，该属性应该设置为 20。使用 Interface Builder，可以针对特定属性或约束轻松添加不同的特征。使用先前创建的包含彩色标签的堆栈视图，选择堆栈并打开“Attributes”检查器。将其“Spacing”设置为 0，然后点按“+”按钮，这将会打开一个能更改变化的弹出窗口。

 

 

如果 你希望为“Regular”宽度和“Regular”高度特征集合添加一个新变体，请将相应的控件设置为希望的值，然后再选择“Add Variation”。这样将会为此特定的特征集合新增一个独一无二的“Spacing”属性。将此新“Spacing”更改为 20。

 

要验证此特征变化能否正常运行，请点按“View as”按钮，并将设备更改为“iPad”（该设备具有“Regular”宽度和“Regular”高度）。你应该看到每个标签之间的间距增加了。如果返回到 iPhone 12 / iPhone 12 Pro，标签之间的间距就消失了。

Installed 属性

有时，你将希望根据特征集合来禁用特定视图或约束。例如，可能出于节省空间的考虑，如果特征集合的高度设置为“Compact”，则 app 界面不应该包含红色标签。要开始实现此设计决策，请选中红色标签，并打开“Attributes”检查器。你将在底部附近发现一个“Installed”复选框。此复选框用于确定视图层次结构中是否存在特定视图或约束。点按该复选框旁边的“+”按钮，同样会显示用来加入变体的弹出窗口。将“Width”设置为“Any”，将“Height”设置为“Compact”，然后点按“Add Variation”。此操作将会添加一个新的“Installed”属性，此属性会覆盖所有包括“Compact”高度的特征集合。

 

现在取消选中此新属性的复选框，并使用 iPhone 12 Pro Simulator 来构建并运行应用程序。当设备处于“Portrait”模式（“Compact”宽度，“Regular”高度）时，将在堆栈视图中显示红色标签。如果将设备使旋转为横屏模式，则特征集合会更新为“Regular”宽度和“Compact”高度，红色标签将消失以节省空间。

调试视图层次结构

视图层次结构中可能会包含大量堆栈视图和各种各样的按钮、标签和图像，因而变得极其复杂。如果发现运行时问题，例如一个视图遮挡了另一个视图或者约束未按预期生效，Xcode 提供了一种工具来帮助你解决问题。

构建并运行你刚刚构建的包含彩色标签的 app。在载入视图之后，按下“Debug View Hierarchy”​
 按钮（位于 Xcode 底部的“Debug”区域顶端）。该视图将出现在 3D 环境中，你可以旋转该视图来查看视图层次结构是如何构造的。

 

你在编辑器区域会看到以下项目。当你向顶层移动时，下一视图会位于前一视图的上一层。

  • 黑色视图为应用程序的 UIWindow
  • 白色视图为视图控制器的视图
  • 透明的小矩形为包含彩色标签的 UIStackView
  • 红色、绿色、蓝色、橙色和紫色标签是之后添加的

使用左侧的调试导航器 可以看到层次结构的树状图，以及每个约束的值。此操作将快速获取每个约束的值而无需将其打印到控制台。

在“Debug”区域的顶部还有各种各样的“Debug View Hierarchy”工具按钮，可用于启用/禁用每个视图的内容、隐藏/显示约束等等。如果你遇到了难以解决的视图问题，则应该考虑使用此工具。

 

练习：计算器

本练习的目标是使用“自动布局”来创建将会根据任意屏幕大小和布局进行缩放的视图。你将使用视图对象、约束和堆栈视图，来创建在所有尺寸的设备中都可以维持其布局的简单计算器。

使用 iOS“App”模板创建一个名为“Calculator”的新项目。

第 1 步

创建外部约束


  • 在 Interface Builder 中，将“View as”设备设置为 iPhone 12 / iPhone 12 Pro。
  • 现在请忽略按钮的复杂性，将计算器分解为两个不同的部分，让顶部三分之一的部分显示输入的数字，底部三分之二的部分用于输入，这一操作再简单不过了。
  • 将一个垂直堆栈视图从“Object”库中拖移到场景中。使用“Add New Constraints”工具添加将堆栈视图的上边缘、前边缘、下边缘、后边缘与外部视图的相应边缘对齐的四个约束，并将间距设置为 0。点按“Update Frames”工具，堆栈视图应覆盖整个屏幕。

 

 

  • 将两个 UIView 对象从“Object”库中拖移到堆栈视图中。按住 control 键将对象从底层视图拖移到外层视图，并在弹出式菜单中选择“Equal Heights”。此操作会将内层视图的高度设置为与外层视图相同，这是你将在下一步中需要解决的问题。

 

 

  • 选中底层视图并打开“Size”检查器。找到高度约束并连按两次。将“Multiplier”值从 1 更改为 0.66，这会将内层视图的高度设置为外层视图高度乘以 0.66。底层视图的高度将始终为视图控制器视图大小的三分之二。

 

 

将一个 UILabel 从“Object”库中添加到顶部容器视图中。将文本设置为 "0"，将字体大小设置为 30.0，然后打开“Add New Constraints”工具。添加下边缘和后边缘约束，二者的间距为 8 像素。

 

 

第 2 步

添加按钮并设置其大小

  • 完成的计算器具有五行按钮，并且每行的高度都相同。你可以使用另一个垂直堆栈视图来实现 该目标。将一个垂直堆栈视图拖移到底层容器视图中，然后使用“Add New Constraints”工具，将堆栈视图的上边缘、前边缘、下边缘、后边缘约束为与底层容器视图的对应边缘间距 0。点按“Update Frames”工具，堆栈视图应覆盖整个底层容器。
  • 选择新添加的垂直堆栈视图，并打开“Attributes”检查器。将“Distribution”属性设置为“Fill Equally”，以便堆栈视图中的所有子视图的高度都相同。将“Spacing”设置为 1，此操作将在各个行之间创建水平分隔线。
  • 你可以将水平堆栈视图用于每行按钮。在垂直堆栈视图中添加一个水平堆栈视图，并将其“Distribution”属性设置为“Fill Equally”，以便堆栈视图中的所有子视图的宽度都相同。将“Spacing”设置为 1，此操作将在各个子视图之间创建水平分隔线。
  • 向水平堆栈视图中添加一个 UIButton。将色调设置为“深色文本”，背景颜色设置为“浅灰色”。
  • 将按钮拷贝 (command-C) 到剪贴板中，然后将三个其他按钮粘贴 (command-V) 到水平堆栈视图中。将最后一个按钮的背景颜色更改为“系统红色”。

 

 

  • 选择文稿大纲中的水平堆栈视图，并将其拷贝 (command-C) 到剪贴板中。然后再将四个其他水平堆栈视图粘贴 (command-V) 到垂直堆栈视图中。

 

 

  • 底层水平堆栈视图只使用了三个按钮而不是四个。由于某些按钮的大小与其他按钮不同，因此需要将此堆栈视图的“Distribution”更新为“Fill Proportionally”。然后，从底层视图中删除一个浅灰色按钮，以便堆栈视图包含正确数量的控件。

 

  • 按住 control 键，从左下角的浅灰色按钮拖移到旁边的按钮，然后选择“Equal Widths”。此操作将确保两个按钮的大小始终相同。为中间的浅灰色按钮和红色按钮重复此过程。然后选中最左侧的灰色按钮，并打开“Size”检查器。连按两次其宽度约束并将“Multiplier”更改为 2。此操作将确保左侧灰色按钮的宽度为另一个灰色按钮的二倍。

 

  • 最后，更新顶层的三个按钮，使用深灰色，并更新按钮的标题以匹配传统计算器上的标题。

 

恭喜！你已经使用约束和堆栈视图创建了一个简单的计算器。请务必在多个 iOS 设备中构建并运行你的应用程序，以验证定义的约束是否在所有大小的屏幕中都有效。请将你的作品存储到项目文件夹中。
