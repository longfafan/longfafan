Swift 自带许多实用类型，用来表示如数字、文本、集合、true 或 false 值之类的数据。但是，随着对构建 app 的不断深入，你会发现你想使用自己设计的属性和函数来创建自己的数据类型。

通过声明结构体来创建自定数据类型，结构体会将一个或多个变量合并到一个类型中。你可通过向结构体中添加类型和实例方法来定义功能。

在本节课中，你将学习如何创建结构体，以及如何使用结构体来构建代码块。

你可能还没有意识到，从本课程的一开始，你就已经在使用结构体了。Swift 自带预定义结构体，用来表示常见类型的数据。如果你希望类型满足你程序或 app 的特定需求，可以定义自己的结构体。

结构体最简单的形式就是具名组，其中包含构成类型的一个或多个属性。属性代表关于结构体的实例的信息。

使用 struct 关键字和唯一名称定义一个结构体。然后可以通过列出具有正确类型标注的常量或变量声明来定义属性，将其作为 struct 的一部分。类型名称的拼写惯例是首字母大写，而属性名称则使用驼峰式命名法。

下面是具有 name 属性的 Person 结构体的简单声明：

 

struct Person {

var name: String

}

 

上述声明只是定义了 Person 类型的属性。该类型本身并没有值。要使 Person 类型具有值，必须为其创建实例。然后，就可以使用点语法访问其属性中存储的数据，如人员的 name。

 

let firstPerson = Person(name: “Jasmine”)

print(firstPerson.name)

控制台输出：

Jasmine

 

从示例中的 name 不难看出，在程序的整个生命周期中可能会创建不止一个 Person。你创建的下一个实例可能名为 secondPerson，或者可以创建名为 people 的集合，其中包含多个 Person 对象。在后面的课程中你会了解关于集合的知识。

可以通过添加方法向结构体中添加功能。方法是具有特定类型的函数。在这一示例中，我们的 Person 实例现在可以 sayHello 了。

 

struct Person {

var name: String

func sayHello() {

print(”Hello, there! My name is \(name)!”)

}

}

 

现在可以使用点语法直接调用实例方法。

 

let person = Person(name: “Jasmine”)

person.sayHello()

控制台输出：

Hello, there! My name is Jasmine!

 

在后面的部分中，你将进一步了解关于实例和实例方法的更多内容。

 

实例

你已经了解结构体可以定义一个新的类型。要使用该类型，必须为其创建实例，这个过程被称为初始化。初始化完成后，每个实例都继承了该结构体的所有属性和功能。

在以下示例中，myShirt 和 yourShirt 都是具有 size 和 color 属性的 Shirt 对象。但是对于每个属性来说，每个对象都是具有不同值的不同 Shirt，因此每个对象都是类型为 Shirt 的不同实例。Size 和 Color 类型定义了一组可用选项，这称为“枚举”，你将在后面的课程中学习这方面的内容。由于目前尚未定义 Size 和 Color，因此将此示例拷贝到 playground 后将无法编译。请查看一下代码，尝试从概念上理解代码的含义。

 

struct Shirt {

var size: Size

var color: Color

}

// 定义衬衫的属性。

 

let myShirt = Shirt(size: .xl, color: .blue)

// 创建单个衬衫的实例。

let yourShirt = Shirt(size: .m, color: .red)

// 创建单个衬衫的另外一个实例。

 

下面是添加了功能的结构体定义示例。该结构体定义了 Car 对象的属性和功能，并将 firstCar 和 secondCar 描述为两个实例。

 

struct Car {

var make: String

var model: String

var year: Int

var topSpeed: Int

 

func startEngine() {

print(”The \(year) \(make) \(model)’s engine has started.”)

}

 

func drive() {

print(”The \(year) \(make) \(model) is moving.”)

}

 

func park() {

print(”The \(year) \(make) \(model) is parked.”)

}

}

 

let firstCar = Car(make: “Honda”, model: “Civic”, year: 2010,

topSpeed: 120)

let secondCar = Car(make: “Ford”, model: “Fusion”, year: 2013,

topSpeed: 125)

 

firstCar.startEngine()

firstCar.drive()

控制台输出：

The 2010 Honda Civic’s engine has started.

The 2010 Honda Civic is moving.

 

本代码运行后汽车会怎样移动？假设 firstCar 和 secondCar 面前的车道是一样的，那么在执行这段代码后，只有 firstCar 会移动，而 secondCar 甚至连引擎都不会启动。

构造器

所有 Swift 类型都自带构造器，它们与返回该类型新实例的函数类似。默认构造器为 init()。

默认构造器所创建的实例具有默认值。默认 String 为 “”，默认 Int 为 0，默认 Bool 为 false：

 

var string = String.init() // “”

var integer = Int.init() // 0

var bool = Bool.init() // false

 

但是默认构造器的简写版更为常见。与上面的示例相比，以下代码片段更为简洁，并且具备相同的功能：

 

var string = String() // “”

var integer = Int() // 0

var bool = Bool() // false

 

无论何时定义新类型，都必须考虑如何创建新实例。本节课程涵盖了不同的初始化属性值方法。

默认值

Swift 要求在初始化新实例的过程中为所有实例属性设置值。

其中一种方法是在类型定义中提供默认属性值。每个实例都将使用该值进行初始化。这对于定义默认状态一致（比如里程表读数都为 0）的对象非常有用。

在前面的小节中，你已经了解了 String、Int 和 Bool 的默认值。通过使用默认属性值可以为自定类型的每个新实例创建默认状态。

 

struct Odometer {

var count: Int = 0

}

 

let odometer = Odometer()

print(odometer.count)

控制台输出：

0

 

请注意，在声明属性时，count 设置为默认值 0。所有新创建的 Odometer 实例都将具有该默认值。

成员级构造器

有时，你会希望创建一个不使用默认值的新实例。例如，你可能需要使用先前旅行或前任车主的行驶里程对 Odometer 进行初始化。在定义新结构体时，Swift 会创建名为“成员级构造器”的特殊构造器，其中包含结构体的所有属性。成员级构造器可用于为新实例的每个属性设置初始值。

 

let odometer = Odometer(count: 27000)

print(odometer.count)

控制台输出：

27000

 

如果类型的新实例没有默认状态，正确的做法是使用成员级构造器。

例如以下具有 name 属性的 Person 结构体。你会将哪个默认值赋给 name？

 

struct Person {

var name: String

}

 

乍一看，你可能会说 name 的默认值可能是 “”。但空 String 不是一个 name，而且你也不想使用空名称对 Person 进行初始化。

要调用成员级构造器，请使用类型名称，后接圆括号，圆括号内包含与每个属性匹配的参数。实际上，你已经在本课程的多个代码片段中见过成员级构造器。

 

struct Person {

var name: String

 

func sayHello() {

print(”Hello, there!”)

}

}

 

let person = Person(name: “Jasmine”) // 成员初始值设定

 

struct Shirt {

var size: Size

var color: Color

}

 

let myShirt = Shirt(size: .xl, color: .blue) // 成员初始值设定

 

struct Car {

var make: String

var model: String

var year: Int

var topSpeed: Int

}

 

let firstCar = Car(make: “Honda”, model: “Civic”, year: 2010,

topSpeed: 120) // 成员初始值设定

成员级构造器是创建自定结构体的新实例的最常用的方法。但是有时候会希望在对所有属性进行赋值之前，先定义一个构造器来完成某些自定逻辑。在这些情况下，可以定义一个自定构造器。

自定构造器

可以通过定义自己的构造器来自定初始化过程。自定构造器的要求与默认值和成员级构造器相同：完成初始化之前，所有属性都必须设置为初始值。

例如以下具有 celsius 属性的 Temperature 结构体。如果可以访问以“摄氏度”表示的温度，那么可以使用成员级构造器对其进行初始化。

 

struct Temperature {

var celsius: Double

}

 

let temperature = Temperature(celsius: 30.0)

 

但是如果可以访问以“华氏度”表示的温度，那么在使用成员级构造器之前需要将该值转换为“摄氏度”。

 

let fahrenheitValue = 98.6

let celsiusValue = (fahrenheitValue - 32) / 1.8

 

let temperature = Temperature(celsius: celsiusValue)

 

在初始化新的 Temperature 对象之前，成员级构造器要求你先计算“摄氏度”值。而你可以创建以“华氏度”作为参数的自定构造器，执行计算，并将该值赋给 celsius 属性。

 

struct Temperature {

var celsius: Double

 

init(celsius: Double) {

self.celsius = celsius

}

 

init(fahrenheit: Double) {

celsius = (fahrenheit - 32) / 1.8

}

}

 

let currentTemperature = Temperature(celsius: 18.5)

let boiling = Temperature(fahrenheit: 212.0)

 

print(currentTemperature.celsius)

print(boiling.celsius)

控制台输出：

18.5

100.0

 

你可能注意到了，在前一示例中有一个成员级构造器。在向类型定义中添加自定构造器时，你必须定义自己的成员级构造器；Swift 将不再提供成员级构造器。

你可以添加多个自定构造器。下面的代码重新定义了 Temperature 以添加“开氏度”构造器。

 

struct Temperature {

var celsius: Double

 

init(celsius: Double) {

self.celsius = celsius

}

 

init(fahrenheit: Double) {

celsius = (fahrenheit - 32) / 1.8

}

 

init(kelvin: Double) {

celsius = kelvin - 273.15

}

}

 

let currentTemperature = Temperature(celsius: 18.5)

let boiling = Temperature(fahrenheit: 212.0)

let freezing = Temperature(kelvin: 273.15)

 

print(currentTemperature.celsius)

print(boiling.celsius)

print(freezing.celsius)

控制台输出：

18.5

100.0

0

 

每个 Temperature 实例都是使用不同构造器和值创建的，但每个实例结尾都是带有所需的 celsius 属性的 Temperature 对象。

实例方法

实例方法是可以针对某个类型的特定实例进行调用的函数。使用实例方法，可以访问和修改结构体的属性，以及添加与实例用途相关的功能。

正如前面所述，你通过在类型定义中添加函数来添加实例方法。然后就可以针对该类型的实例来调用该函数了。你可能在之前的 Person 或 Car 结构体中注意到了这个语法。

例如以下 Size 结构体，它带有 area() 实例方法，该方法将宽度与高度相乘来计算特定实例的面积：

 

struct Size {

var width: Double

var height: Double

 

func area() -> Double {

width * height

}

}

 

let someSize = Size(width: 10.0, height: 5.5)

let area = someSize.area() // area 的值是 55.0

 

someSize 实例属于 Size 类型，width 和 height 是其属性。area() 是实例方法，可以针对所有 Size 类型的实例进行调用。

 

更改方法

有时，你会希望在实例方法内更新结构体的属性值。为此，需要在函数之前添加 mutating 关键字。

在以下示例中，简单的结构体中储存了关于特定 car 对象的行驶里程数据。在查看代码之前，先思考一下行驶里程计数器需要储存哪些数据以及执行哪些响应动作。

  • 储存里程表中显示的行驶里程计数
  • 当汽车行驶时，通过增加行驶里程计数来更新行驶里程
  • 如果汽车行驶的英里数超过了里程表可显示的英里数上限，那么可能需要重置行驶里程计数。

 

struct Odometer {

var count: Int = 0 // count 属性的默认值为 0。

 

mutating func increment() {

count += 1

}

 

mutating func increment(by amount: Int) {

count += amount

}

 

mutating func reset() {

count = 0

}

}

 

var odometer = Odometer() // odometer.count 属性的默认值为 0

odometer.increment() // odometer.count 属性的值递增为 1

odometer.increment(by: 15) // odometer.count 属性的值递增为 16

odometer.reset() // odometer.count 属性的值重置为 0

odometer 实例属于 Odometer 类型，increment() 和 increment(by:) 是向实例添加英里数的实例方法。reset() 实例方法会将行驶里程计数重置为 0。

 

计算型属性

Swift 有一个特性，允许属性执行返回计算值的逻辑。

例如 Temperature 示例。世界上大部分国家和地区使用“摄氏度”作为衡量温度的标准，一些国家和地区 (如美国) 会使用“华氏度”，而某些专业领域则使用“开氏度”。因此，如果 Temperture 结构体能够支持这三种度量系统，那么可能会非常有用。

 

struct Temperature {

var celsius: Double

var fahrenheit: Double

var kelvin: Double

}

 

假设你已对此结构体使用了成员级构造器：

 

let temperature = Temperature(celsius: 0, fahrenheit: 32.0,

kelvin: 273.15)

 

如果你希望编写代码来初始化 Temperature 对象，都需要计算每种温度，并将以上所有值作为参数进行传递。

另一种可选方法是添加多个可处理上述计算的构造器。

 

struct Temperature {

var celsius: Double

var fahrenheit: Double

var kelvin: Double

 

init(celsius: Double) {

self.celsius = celsius

fahrenheit = celsius * 1.8 + 32

kelvin = celsius + 273.15

}

 

init(fahrenheit: Double) {

self.fahrenheit = fahrenheit

celsius = (fahrenheit - 32) / 1.8

kelvin = celsius + 273.15

}

 

init(kelvin: Double) {

self.kelvin = kelvin

celsius = kelvin - 273.15

fahrenheit = celsius * 1.8 + 32

}

}

 

let currentTemperature = Temperature(celsius: 18.5)

let boiling = Temperature(fahrenheit: 212.0)

let freezing = Temperature(kelvin: 273.15)

 

 

 

上面的方法使用了多个构造器，涉及管理大量状态或信息。无论何时温度发生变化，这三种属性都需要更新。这种方法极易出错，这将会令任何编程人员都感到沮丧。

Swift 提供了一种更安全的方法。通过计算型属性，你可以创建基于其他实例属性或逻辑来计算自身值的属性。

 

struct Temperature {

var celsius: Double

 

var fahrenheit: Double {

celsius * 1.8 + 32

}

 

var kelvin: Double {

celsius + 273.15

}

}

 

要添加计算型属性，请将属性声明为变量 (因为它的值可以更改)。此外，你还必须明确声明类型。然后再使用左花括号 ({) 和右花括号 (}) 来定义计算返回值的逻辑。

与对其他任何属性的处理一样，你可用点语法来访问计算型属性。

 

let currentTemperature = Temperature(celsius: 0.0)

print(currentTemperature.fahrenheit)

print(currentTemperature.kelvin)

控制台输出：

32.0

273.15

 

每次访问计算型属性时都会执行该属性所包含的逻辑，因此返回值将始终是最新的。

属性观察器

在 Swift 中，可以观察任意属性，并根据属性值的更改进行响应。每次为属性设置值时，都会调用属性观察器，即使新值与属性的当前值相同也不例外。可以针对任意给定属性定义的观察器闭包或代码块有两种：willSet 和 didSet。

在以下示例中，已定义带有 totalSteps 属性的 StepCounter。willSet 和 didSet 观察器也都已定义。无论何时修改 totalSteps，都会先调用 willSet，然后你可以访问新值，以将其设置为 newValue 常量中的属性值。在更新属性值之后，会调用 didSet，你可以使用 oldValue 访问先前的属性值。

 

struct StepCounter {

var totalSteps: Int = 0 {

willSet {

print(”About to set totalSteps to \(newValue)”)

}

        didSet {

            if totalSteps > oldValue  {

                print(”Added \(totalSteps - oldValue) steps”)

            }

        }

    }

}

 

下面是为新的 StepCounter 修改 totalSteps 属性时的输出：

var stepCounter = StepCounter()

stepCounter.totalSteps = 40

stepCounter.totalSteps = 100

控制台输出：

About to set totalSteps to 40

Added 40 steps

About to set totalSteps to 100

Added 60 steps

类型属性和方法

你已经学习了实例属性是关于某种类型的单个实例的数据，而实例方法则是可以针对某种类型的单个实例进行调用的函数。

Swift 还支持添加可以针对类型本身进行访问或调用的类型属性和方法。使用 static 关键字可以向类型中添加属性或方法。

如果属性与类型相关，但和实例本身的属性无关，那么类型属性会非常有用。

以下示例定义了一个 Temperature 结构体，其具有名为 boilingPoint 的静态属性，该属性是一个常量值，对于所有 Temperature 实例都是一样的。

 

struct Temperature {

static var boilingPoint = 100

}

 

针对类型名称使用点语法来访问类型属性。

 

let boilingPoint = Temperature.boilingPoint

 

类型方法与类型属性类似。如果响应动作与类型相关却不是该类型的特定实例应执行的响应动作，那么可以使用类型方法。

Swift 标准库中定义的 Double 结构体包含一个名为 minimum 的静态方法，其会返回两个参数中的较小者。

 

let smallerNumber = Double.minimum(100.0, -1000.0)

 

拷贝

如果将结构体赋给变量，或者将实例作为参数传递给函数，那么你就对值进行了拷贝。不同的变量因而会成为值的不同实例，也就是更改一个值不会更改另一个值。

 

var someSize = Size(width: 250, height: 1000)

var anotherSize = someSize

 

someSize.width = 500

 

print(someSize.width)

print(anotherSize.width)

控制台输出：

500

250

 

请注意，someSize 的 width 属性已更改为 500，但 anotherSize 的 width 属性却没有更改，这是因为在将 anotherSize 设置为 someSize 时，只创建了 someSize 的副本，而副本的宽度不会随着原始宽度的更改而发生改变。

 

Self

你可能已经注意到了本小节中的 self 一词。Swift 中的 self 指的是类型的当前实例。也可以在实例方法或计算型属性中使用 self 来指代其自己的实例。

 

struct Car {

var color: Color

 

var description: String {

return “This is a \(self.color) car.”

}

}

许多语言都要求使用 self 引用类型定义内的实例属性或方法。如果当前对象存在属性或方法名称，那么 Swift 编译器会识别并选择是否使用 self。

下面这个示例与前一示例的功能相同。

 

struct Car {

var color: Color

 

var description: String {

return “This is a \(color) car.”

}

}

 

如果构造器中的参数名称与属性名称相匹配，那么必须在其中使用 self。

 

struct Temperature {

var celsius: Double

 

init(celsius: Double) {

self.celsius = celsius

}

}

 

这一概念被称为“隐蔽”，你将在后面的课程中进一步了解关于“隐蔽”的更多内容。

变量属性

你是否注意到本课程中的所有结构体示例都采用了变量属性而非常量？例如课程开始时使用的 Car 自定结构体。

 

struct Car {

var make: String

var year: Int

var color: Color

var topSpeed: Int

}

 

汽车的颜色可通过重新喷漆轻松改变，如果车主升级了引擎，最高时速也可能会更新。但是汽车的品牌和出厂日期始终都不会更改，因此为什么不使用 let 来定义这些属性呢？

变量属性提供了便捷方式来基于旧数据创建新数据。在以下代码中，构建一辆 2010 款蓝色福特极其简单，只需拷贝 2010 款本田的代码，再修改 make 属性即可。如果 make 属性为常量，那么第二辆汽车则需要使用成员级构造器来创建。

 

var firstCar = Car(make: “Honda”, year: 2010, color: .blue,

topSpeed: 120)

var secondCar = firstCar

secondCar.make = “Ford”

 

在本节课的先前部分，你了解了无论何时将结构体的实例赋给新变量，都会拷贝值。在前一示例中，firstCar 仍为 Honda。如果想要明确防止 firstCar 属性改变，只需使用 let 声明实例即可。

 

let firstCar = Car(make: “Honda”, year: 2010, color: .blue,

topSpeed: 120)

firstCar.color = .red // Compiler error!

 

即使使用 var 来声明属性，常量的值也不可更改。作为一项通用规则，应尽可能使用 let 来定义结构体的实例，如果实例需要改变，那么请使用 var，并在定义结构体的属性时使用 var。
