Xcode playground 是非常适合学习和试验 Swift 编程语言的工具，而 Xcode 项目则适用于创建应用程序。在编写代码时，你会想看看代码对 app 的影响。为此，Xcode 提供了“Simulator”。在这个应用程序中，你可以在各种设备和不同大小的屏幕上测试 app，包括型号较旧的设备，这些设备可能缺少 Touch ID 或 Face ID 等先进的最新硬件技术，或者具有已在 iPhone X 上移除的 Home 键。

在本节课中，你将完成一些必要的步骤来配置 Mac 以构建和调试 app，以及在 Xcode 中使用“Simulator”来测试 app。你还将学习如何在物理设备上运行 app，以及如何使用断点找到代码中的程序错误。

内容提要

  • 如何使用“Simulator”在 Xcode 环境中运行 app
  • 如何在物理设备上运行 app
  • 如何使用断点执行基本调试

构建并运行

首先，在 Xcode 中创建一个新项目。你将使用在前一课中用过的 iOS“App”模板。创建项目时，请确保将界面选项设置为“Storyboard”。将项目命名为“GettingStarted”，并选择保存位置。

接下来，你将看到 Xcode 工作区。在 Xcode 工具栏中找到“Scheme”菜单，然后从列表中选择你要模拟的设备。

 

点按“Run”按钮 ，或者使用键盘快捷键 (command-R) 开始在“Simulator”中启动 app。

如果是第一次运行 app，Xcode 会询问你是否愿意在 Mac 上启用开发者模式。启用开发者模式后，Xcode 无需在每次访问特定的调试功能时都要求你输入密码。本课程的每一节课都假设你已启用开发者模式。

 

 

启动“Simulator”之后，你应该看到一个具有白色背景的设备图像。要将设备图像从竖版旋转为横版，可使用键盘快捷键 command-左箭头和 command-右箭头。

设备图像在“Simulator”中看起来可能会很大，这取决于你的 Mac 以及你选择模拟的设备的屏幕分辨率。在“Simulator”应用程序内，可以使用从 command-1 到 command-3 的键盘快捷键来放大或缩小设备图像。如果使用的是旧款 Mac，可以选择分辨率较低的旧款设备（如 iPhone SE）来减少“Simulator”对系统的总体影响。

要退出“Simulator”，可使用键盘快捷键 command-Q，或者直接返回 Xcode。

“Simulator”并不适用于 app 的所有部分。某些交互取决于实际运行的物理设备。例如，如果尝试使用“Simulator”来测试与“相机”app 的交互，则程序将会崩溃。如果你的代码取决于 Mac 上所没有的其他硬件组件（可能是加速感应器、陀螺仪或近距离传感器），那么你需要使用实际设备进行测试。

“Simulator”还存在一些软件限制。例如，推送通知只能发送到物理设备上，而且用于协助撰写电子邮件和短信的 MessageUI 框架与“Simulator”不兼容。如果在“Simulator”中遇到的问题较多，可以尝试在 iPhone 或 iPad 中测试代码。你的问题可能会得到解决。

“Simulator”在 Mac 上运行，Mac 是在 Intel“x86-64” 体系结构上构建的，但 iOS 设备是在 ARM (Advanced RISC Machine) 体系结构上构建的。这意味着 app 运行方式存在一些底层差异，有时这种差异很细微。

总的来说，“Simulator”在开发和调试 app 时表现出色，但最好还是在实际硬件中测试代码。

使用个人设备

移动 app 的美妙之处在于，你可以把它们带到任何地方。你可能会想和朋友、同事分享你的编程进度。但是，你必须有一个 Apple Developer 网站的账户才能开始在物理设备上运行代码。账户是免费的，所以不用担心。

使用 web 浏览器前往 developer.apple.com/cn/，并点按“帐户”。可以使用现有 Apple ID 注册。如果没有 Apple ID，请继续并创建一个 Apple ID，这也是免费的。如果输入 Apple ID 之后，你的开发者账户一切正常，则可以返回到 Xcode。

（使用一个免费账户只能在一台物理设备上运行 iOS app。如果希望将 app 分配给多台设备，或是希望将其发布到 App Store，则需要在 Apple Developer Program 中注册。）

现在你将需要告诉 Xcode 你的新开发者账户。使用键盘快捷键 command-逗号打开 Xcode 的“Preferences”（或从 Mac 菜单栏选择“Xcode”>“Preferences”），然后选择左上角附近的“Accounts”按钮。点按左下角的“+”按钮并从菜单中选择“Add Apple ID”。在输入你的凭据之后，就可以在物理 iOS 设备上运行并调试 app 了。

使用合适的 USB 线缆将 iOS 设备连接到 Mac。Xcode 将自动从设备中下载必要的信息，并在“Scheme”菜单中显示其名称。选择你的物理设备的名称，它通常位于列表的顶部，设备模拟器之前。

再次构建并运行 app。你可能会收到以下询问是否信任开发者证书的提示。请遵循提醒中的指示以便设备运行 app。

 


再次构建并运行，你应该可以在 iOS 设备上看到相同的简单白色屏幕了。要阻止 app 运行，请点按 Xcode 工具栏左端附近的“Stop”按钮（或使用 command-句号键盘快捷键）。

以无线方式构建和运行

Xcode 还提供了通过网络将 app 部署到设备的选项。为此，使用合适的 USB 线缆将 iOS 设备连接到 Mac，然后从“Window”下拉菜单中选择“Devices and Simulators”来打开“Devices and Simulators”窗口。

确保在“Devices and Simulators”窗口中最左侧的列表内选中了你的设备。选中“Connect via network”复选框。

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

如果设备与 Mac 位于同一网络上，片刻之后，设备名称旁边会显示地球形状。这表明设备已通过无线方式连接。

现在可以拔下将设备连接到 Mac 的 USB 线缆，然后以无线方式构建和运行 app。

上述步骤适用于大多数需要无线配对的情况。但如果你仍未能完成无线配对，你的企业或机构网络的系统管理员可能设置了某些网络限制。在本例中，打开“Devices and Simulators”窗口，按住 control 键并点按你的设备，然后在显示的菜单中点按“Connect via IP Address”。接着需要从设备的“设置”查找设备的 IP 地址，在提示位置输入该地址，然后点按“Connect”。这样应该就能成功配对你的设备了。如果在无线调试时遇到问题，可以随时通过 USB 线缆进行配对。

调试应用程序

即使是最好的开发者，想要编写完美的代码也比较困难。调试是识别和消除 app 中可能出现的问题的过程。Xcode 提供了一些可帮助你完成此过程的工具。

前面已介绍过，在“Simulator”上或在设备上运行 app 时，Xcode 会将 app 连接到其调试器。这样，你可以实时观察代码的执行情况，使用断点阻止代码执行，将代码中的信息打印到控制台，等等。

在学习本课程的过程中，你会遇到三种类型的问题：警告、编译器错误和程序错误。

警告

警告是最容易修复的问题。无论何时构建代码，都会出现警告，不过这并不会妨碍程序的成功编译和运行。如果满足以下条件，则会抛出警告：

  • 编写永远不会执行的代码
  • 创建永远不变的变量
  • 使用过时的代码（也称为“已弃用代码”）

请看一下这条警告，然后回到 Xcode，在项目导航器中选择 ViewController.swift，并在编辑器区域的 super.viewDidLoad() 的正下方添加以下代码行：

 

let x = 4

 

此代码将 4 这个值赋给 x 常量。使用键盘快捷键 command-B 构建应用程序。你应该可以在刚刚添加的代码行上看到黄色警告标志。

 

Xcode 将尽其所能以一种直接的方式来解释这个警告：

控制台输出：

Initialization of immutable value ‘x’ was never used; consider

replacing with assignment to ‘_’ or removing it.

 

编译器的意思是，它创建了 x，但你没有以有意义的方式使用 x，所以可将其移除。删除这一行代码并重新构建以移除警告。

编译器错误

错误表示存在更严重的问题。例如，代码无效（如拼写错误)、未正确声明变量，或者未正确调用函数。与警告不同，错误可阻止代码的执行。如果代码存在错误，“Simulator”甚至都不会启动。

下面来看看两种不同的错误是如何产生的。在 ViewController.swift 中，移除 super.viewDidLoad() 末尾的左括号和右括号，仅保留 super.viewDidLoad。在这行代码下方，编写 navigationController.title = “Debugging”。

在构建 app 之后，你将会看到两个红色错误符号。

 

 

 

 

其中一个错误符号在中心有一个点 。点按此错误，Xcode 将显示关于修复代码的建议。点按“Fix”按钮，以应用 Xcode 提供的建议。在本例中，Xcode 提供了有效的修复。但你不能依赖编译器来修复所有错误，你需要能够自行解决错误。

另一个错误符号的中心有一个感叹号 。对于这种类型的错误，Xcode 不会提供修复建议，而是会提供有用的消息来帮助你识别问题。编译器认为，使用左括号和右括号来调用函数是正确的语法。在 super.viewDidLoad 末尾添加这两个字符以移除最后一个错误。

程序错误

第三类问题称为“程序错误”，这是最难追查的问题。程序错误指在运行程序时发生的错误，可导致程序崩溃或输出不正确。可能需要花费一些时间进行实际查找才能发现程序错误。

在 ViewController.swift 中，在 super.viewDidLoad() 下方添加以下几行代码：

 

var names = [”Tammy”, “Cole”]

names.removeFirst()

names.removeFirst()

names.removeFirst()

此时，你可能不熟悉 Swift，这没有关系。这几行代码很容易理解。names 是指包含“Tammy”和“Cole”这两个文本片段的列表。每个后续行都会从列表中移除第一项。鉴于调用了三次方法来删除第一项，但是列表中却只有两个项目，你认为将会发生什么情况？

构建你的应用程序。由于语法是有效的，你将收到“构建成功”的消息。现在尝试运行这个 app。过一会儿，程序将崩溃，并且在上述最后一行代码中将以红色显示以下消息：

 

Thread 1: Fatal error: Can't remove first element from an empty collection

 

当程序试图删除剩余的第一个元素却无法找到这个元素时，它就会崩溃。你已经猜到了这一点。但是假设你不确定如何修复这个问题，Xcode 可以通过一次检查一行的方式来排查你的代码问题。

在开始查找程序错误之前，你将需要向代码中添加一个断点。断点可以让程序在指定点上暂停执行。要创建断点，可以通过点按要暂停执行的代码行左侧的装订线区域来完成。在本例中，向 var names = [”Tammy”, “Cole”] 代码行添加了断点。

 

 

构建并运行 app，你将看到程序在断点处暂停。这就对了。在调试区域中，显示变量视图以检查当前值（按钮位于右下角）。由于断点行尚未执行，所以 names 不包含任何值。

你可以从此处开始使用位于调试区域顶部的分步控制按钮继续慢速执行代码：

  • Continue ：恢复代码执行，直到到达下一个断点为止。如果你现在选择这个按钮，代码就会崩溃，因为在第三个 names.removeFirst() 之前没有其他断点。
  • Step over ：执行选定代码行，并暂停执行下一行。
  • Step into ：如果在调用函数的代码行中点按此按钮，则会前进到函数的第一行，然后再次暂停执行。
  • Step out ：执行调用函数的代码中的所有剩余行，并暂停执行函数之后的行。

 

点按“Step over”按钮继续执行一行代码。你可以在变量视图中看到已经将正确的值赋给 names。到目前为止，一切正常。

 

点按“Step over”按钮执行对 names.removeFirst() 的第一次调用。names 列表中不再包含“Tammy”，所以这行代码可以正常工作。再次点按“Step over”执行对 names.removeFirst() 的第二次调用，让 names 成为没有任何值的空白列表。这也没有问题。现在，你应该清楚第三次调用 names.removeFirst() 是产生程序错误的原因。

移除第三次调用 names.removeFirst() 并再次运行程序，以验证是否已修复错误。

调试技能对于开发者构建应用程序至关重要。在调试时，请采用以下方法：

  1. 尝试理解问题
  2. 进行头脑风暴，列出可能的解决方案
  3. 尝试该解决方案
  4. 验证解决方案是否有效，如有必要请重复使用

请一步一个脚印去解决每一个程序错误。在构建 app 时遇到程序错误会很令人沮丧，但如果能够修复程序错误，你会感觉非常棒。

练习：调试你的第一个 app

本练习的目标是找到并解决编译器错误、运行时错误和编译器警告。

第 1 步

查找并修复编译器错误

  • 打开 Xcode 项目“FirstTimeDebugging”。
  • 尝试运行 app。请注意，app 不会运行，因为存在一些编译器错误。本节课已介绍过，编译器错误由红色符号来指示，该符号会显示在有错误的代码行上，或编译器猜测可能有错误的代码行上。所有编译器错误也将列在问题导航器中。
  • 修复编译器错误，以便运行 app。有两个较为常见的错误可能会导致此 app 出现编译器错误：
  • 缺失或多余圆括号或花括号（无论是左括号还是右括号）。
  • 引用函数或属性时拼写不正确（编译器对此非常严格，要求必须按照实际提供的名称一字不差地引用函数或属性）。​
    提示：有时，一个错误可能会导致编译器标记多个错误。修复一个错误可能会消除所有错误符号。

第 2 步

查找并修复运行时错误

  • 你是否能够移除项目中的所有红色错误标志？如果可以，请尝试再次运行 app。
  • 这一次请注意，app 在“Simulator”中打开后立即停止执行，而且屏幕中有一行代码上出现了一条红线。事实上红线指示出现问题。要了解可能是什么问题，可以查看控制台区域中的文本。尝试解决此运行时错误。建议你在受影响的代码行及其之前添加断点，然后再运行 app。

第 3 步

查找并修复编译器警告

  • 既然 app 可以运行，请特别关注另外几个问题。打开项目的问题导航器。你会注意到几个用黄色三角形指示的警告。请处理所有这些警告。

恭喜！现在你的项目应该没有编译器错误、运行时程序错误和警告了。
