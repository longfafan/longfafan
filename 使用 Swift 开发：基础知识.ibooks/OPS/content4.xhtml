一般来说，构建 app 和编程主要就是处理数据。作为开发者，你需要了解如何使用明确定义的类型来处理和储存数据。

在本节课中，你将学习如何使用常量和变量来命名数据片段，以便稍后在程序中使用。你将学习为不会改变的值定义常量，为可能会改变的值定义变量。

你还将学习类型，类型就像是用于规划如何处理数据的蓝图。不同类型可以以常量或变量的形式使用和储存。你将学习 Swift 中包含哪些类型，以及 Swift 类型如何帮助你编写更好的代码。

常量和变量会将名称（比如 age 或 name）与特定类型的值（比如数字 29 或字符串 “John”）相关联。然后就可以在程序中多次使用该名称来引用该值了。常量的值是不可变的，一旦设置即不可更改。而变量的值则是可变的，可随时更改。

如果定义了常量或变量，那么就是要求编译器在将要运行该程序的设备内存中为该值分配存储空间。你也可以要求编译器将常量或变量名称与已赋值相关联，以便稍后访问该值。

常量

如果想要为程序生命周期内不会改变的值命名，可以使用常量。

在 Swift 中，通常使用 let 关键字来定义常量。

 

let name = “John”

​
上面的代码创建了名为 name 的新常量，并为该常量赋值 “John”。如果希望在稍后的代码行中访问该值，可以使用 name 直接引用该常量。当你在程序中需要多次使用某个值时，这一快速引用尤其实用。

 

let name = “John”

print(name)

​
此代码会向控制台打印 “John”。

name 是常量，因此不能在赋值后再为其分配一个新值。例如，以下代码将无法运行：

 

let name = “John”

name = “James”

变量

如果想要为 app 生命周期内可能会改变的值命名，可以使用变量。

使用 var 关键字定义变量。​

var age = 29

print(age)

​
该代码会向控制台打印 29。

由于 age 是变量，因此可以稍后在代码行中向其赋新值。下面的代码编译不会出错。

 

var age = 29

age = 30

print(age)

 

此代码会向控制台打印 30。

你可以使用其他常量和变量为常量和变量赋值。当你想将一个值拷贝到一个或多个其他变量时，这个功能十分有用。

 

let defaultScore = 100

var playerOneScore = defaultScore

var playerTwoScore = defaultScore

 

print(playerOneScore)

print(playerTwoScore)

 

playerOneScore = 200

print(playerOneScore)

 

控制台输出：

100

100

200

 

常量还是变量？

你刚刚学习了，在值不会改变的情况下使用常量；在值可能会改变的情况下使用变量。

但还有一点需要注意：有些可能会改变的值也可以使用常量来表示。这是因为这些值在代码的整个执行周期内不会改变。

假设你正在计算关于旅程距离的信息。可以随着时间的推移重复使用程序来跟踪多趟旅行，但是一次只能跟踪一趟旅行。如何在代码中表示以下数据呢?

  • 起始位置：这是旅行的出发位置的 GPS 坐标。该位置在程序开始跟踪旅行后就不会再改变了。因此，可以使用常量来表示这个值。
  • 目的地：这个 GPS 坐标就是希望到达的位置。由于你的 app 可用于多个目的地，因此你可能会认为应该使用变量来表示这个值。但是，一旦程序开始跟踪旅行，目的地就不会改变。因此，可以使用常量来表示这个值。
  • 当前位置：当前位置的 GPS 坐标将随着你的移动而改变。因此，需要使用变量来表示这个值。
  • 旅行距离：你从起点出发之后走了多远? 这个值会随着你的移动而改变。因此，需要使用另一个变量来表示这个值。
  • 剩余距离：你还需要走多远才能到达目的地? 剩余距离会随着你的移动而改变。因此，需要使用变量来表示这个值。

常量和变量执行的任务非常类似。你可能认为全部使用变量会更容易，而彻底忽略了常量。从技术上来讲，你的代码也能够运转。那么，究竟为什么要使用常量呢？

第一，如果将值设置为常量，那么编译器会理解为该值永远不可更改。这意味着，更改常量的值将无法构建或运行程序。编译器会通过这种方式强制实现安全性。

第二，编译器可以对常量值进行特殊优化。当你将常量用于不会改变的值时，编译器可以对如何储存这些值进行底层假设。这些调整可以让程序执行得更快。

第三，这是 Swift 编程中惯用的一种做法。

随着你对构建软件的了解越来越多，你会逐渐体会到这是确保代码始终简洁、易读的最佳做法。在与团队中的其他开发者一起工作时，这种做法尤其重要。你还会发现，遵循最佳做法可以更轻松地记住代码的功能，而且对于日后更改代码特别有帮助。

如果你希望成为优秀的开发者，就需要遵循共用模式和惯例。本课程将引导你遵循 Swift 编程的最佳做法。

为常量和变量命名

为常量或变量命名时需要遵循许多规则。编译器会强制应用这些规则。不按规则命名将无法构建和运行程序。

  • 名称不能包含数学符号。
  • 名称不能包含空格。
  • 名称不能以数字开头。

除了这三条规则以外，你可以按自己喜欢的方式为常量或变量命名。

 

let π = 3.14159

let 一百 = 100

let 🎲 = 6

let mañana = “Tomorrow”

let anzahlDerBücher = 15 //numberOfBooks

 

除了这些规则以外，为常量和变量命名还有一些最佳做法：

  • 常量和变量名称应该清晰并具有描述性，可让你在稍后看到这些名称时能更轻松地理解代码。例如，firstName 要比 n 好，restaurantsNearCurrentCity 要比 nearby 好。
  • 要使名称清晰并具有描述性，通常需要在一个名称中使用多个单词。将两个或更多单词组合在一起时，应遵循驼峰式命名法惯例，也就是将名称的第一个字母小写，然后将后面每个新单词的首字母大写。你可能已经注意到了 defaultScore，这就是使用驼峰式命名法将 default 和 score 组合在一起的示例。驼峰式命名法比将所有单词直接串连在一起更容易读懂。例如，defaultScore 比 defaultscore 更清晰，restaurauntsNearCurrentCity 比 restaurantsnearcurrentcity 更清晰。驼峰式命名法还有助于辅助设备更好地区分单词边界。

注释

你是否注意到 anzahlDerBücher 的翻译写在了侧边? 随着代码愈加复杂，给你自己及其他可能阅读该代码的开发者留下一些注释，这样可能非常有用。可以通过在文本前面放置两个正斜线 (//) 来创建这些注释。编译代码时会忽略这些注释，因此可以尽量多写几段你认为有帮助的注释。

 

// 设置 pi 为一个粗略值

let π = 3.14

 

如果需要多行注释，还可以根据自己的需要在 /* 和 */ 之间添加更多文本。Swift 编译器会忽略这些文本。

 

/* pi 的位数是无限的，

所以我选择了一个近似值。*/

let π = 3.14

 

注释最常用于解释代码中不易理解的部分，或者在文件顶部提供版权信息以及日期信息（文件的创建或修改日期）。

类型

在 Swift 中，每个常量或变量都具有类型，用于描述值的种类。前面的示例中，29 是一个整数，由 Swift 类型 Int 来表示。同样，"John" 是一个字符串，由 Swift 类型 String 来表示。

Swift 包含许多预定义的类型，方便你更轻松地编写简洁的代码。无论程序是否需要包含数字、字符串或布尔值（true 或 false），都可以使用类型来代表特定类别的信息。

Swift 还支持集合类型，用于将多个值分组到单个常量或变量中。有一种集合类型叫做 Array，可用于储存值的有序列表。还有一种集合类型叫做 Dictionary，其具有可用于查找特定值的键。在后面的课程中你会进一步了解关于集合的知识。

Swift 标准库包含上述所有类型，供你随时使用。

此外，你还可以通过创建类型定义，在 Swift 中定义自己的类型。下面是一个简单的 Person 类型：

 

struct Person {

let firstName: String

let lastName: String

 

func sayHello() {

print(”Hello there! My name is \(firstName) \(lastName).”)

}

}

 

可以将类型定义视为一种蓝图。蓝图规定了如何建造大楼，但并不是大楼本身。你可以使用一张蓝图建造多栋大楼。

类型定义声明了类型所储存的信息（属性）及其功能或操作（方法）。在本例中，Person 在 firstName 和 lastName 这两个属性中储存 String 信息，并有一个操作，即 sayHello() 方法。你可能不熟悉上述所有这些语法，但不用担心。

 

上述示例描述了类型 Person 所存储的信息，以及要执行的操作。创建类型并将其赋给变量或常量，即是创建类型版本或类型实例。因此，一个实例就是一个值。下面是代码示例：

 

let aPerson = Person(firstName: “Jacob”, lastName: “Edwards”)

let anotherPerson = Person(firstName: “Candace”, lastName:

“Salinas”)

 

aPerson.sayHello()

anotherPerson.sayHello()

控制台输出：

Hello there! My name is Jacob Edwards.

Hello there! My name is Candace Salinas.

 

这段代码创建了两个 Person 类型的实例。其中一个实例代表名为 Jacob Edwards 的 Person，另一个实例代表名为 Candace Salinas 的 Person。

在后面的课程中你会进一步了解如何定义类型。

类型安全

Swift 是一种类型安全的语言。类型安全的语言鼓励或要求清晰地说明代码可以处理的值的类型。例如，如果某部分代码需要 Int，那么就不能向其传递 Double 或 String。

在编译代码时，Swift 会对所有常量和变量执行类型检查，并将不匹配的类型全部标记为错误。如果类型不匹配，程序就无法运行。

 

let playerName = “Julian”

var playerScore = 1000

var gameOver = false

 

playerScore = playerName

// 类型不匹配时，将无法编译。

 

类型安全会强制要求使用常量或变量可存储的类型，因此将 String 赋给只存储 Int 的变量是没有意义的。你很快就会了解 Swift 是如何识别类型的。

类型安全也同样适用于看似相似的变量、常量和字面量值所代表的数据类型。例如，尽管 Int 和 Double 都代表数字，但 Swift 仍会将它们视为完全不同的类型。

 

var wholeNumber = 30

var numberWithDecimals = 17.5

 

wholeNumber = numberWithDecimals

// 类型不匹配时，将无法编译。

 

在上述示例中，两个变量都是数字类型，但 wholeNumber 是 Int，而 numberWithDecimals 是 Double。Swift 不允许将一种类型的值赋给另一种类型的变量。

当你处理数字时，可能会发现需要将一个非常大的值赋给变量或常量。这个值阅读起来可能会很困难，就像很难在 1000000000 中看出有多少个 0。幸运的是，Swift 可以采用在数字中添加下划线这种格式，以便于阅读。

 

var largeUglyNumber = 1000000000

var largePrettyNumber = 1_000_000_000

 

类型推断

你可能已经注意到了，在声明常量或变量时无需指定值的类型。这称为类型推断。根据赋给常量和变量的值，Swift 将使用类型推断对类型进行假设。

 

let cityName = “San Francisco”

// “San Francisco”明显是 String 类型，因此编译器自动将 cityName 的类型分配给了 String。

 

let pi = 3.1415927

// 3.1415927 是一个带小数点的数字，因此编译器自动将 pi 的类型分配给了 Double。

 

在为常量或变量赋值之后，类型就已固定，不可更改。对于变量来说也是如此。变量的值可能会更改，但其类型则不会。

你可能会发现在某些情况下，显式指定常量或变量的类型会十分有用，甚至十分有必要。这称为类型标注。要指定类型，请在常量或变量名称之后添加冒号 (:)、空格和类型名称。

 

let cityName: String = “San Francisco”

 

let pi: Double = 3.1415927

 

你可以使用与数字类型不相同的类型标注。Swift 编译器将调整值来匹配类型。

 

let number: Double = 3

print(number)

控制台输出：

3.0

使用类型标注有三种常见的情况：

  1. 当你创建了常量或变量，但尚未赋值时。

 

let firstName: String

//...

firstName = “Layne”

 

  1. 当你创建了可以推断出多种类型的常量或变量时。

 

let middleInitial: Character = “J”

// “J”可能会被类型推断为 String 类型，但我们想要 Character 类型。

 

var remainingDistance: Double = 30

// 30 可能会被类型推断为 Int 类型，但这个变量能为我们提高精度。

 

  1. 当你编写自己的类型定义时。

 

struct Car {

var make: String

var model: String

var year: Int

}

 

必需值

无论何时定义常量或变量，都必须使用类型标注指定类型，或是向其赋值以便编译器能够推断类型。

 

var x

// 这将导致错误。

 

如果没有为常量或变量赋值，那么即使使用了类型标注，代码也无法运行。

 

var x: Int

print(x)

// 这将导致错误。

 

在赋值之后，就可以使用常量或变量了。

 

var x: Int

x = 10

print(x)
