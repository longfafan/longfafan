至于如何访问文档，每位开发者都有自己喜欢的方式。有些开发者喜欢在 Web 浏览器中进行查看，而有些开发者喜欢使用 Xcode 提供的文档浏览器。你可以了解多种查看文档的方式，然后选出最适合自己的一种。越快找到问题的答案，就能越快继续编写代码。

文档浏览器

在前一课中，你在 viewDidLoad() 函数中添加了几行代码。你知道该函数的作用是什么，或者何时在程序中调用该函数吗？你可以通过 Xcode 的“快速帮助”功能快速找到答案。按住 option 的同时点按 viewDidLoad() 方法名称，Xcode 会显示弹出窗口，其中包含该函数的简要说明。

在“快速帮助”弹出窗口中，点按“Open in Developer Documentation”以方便地访问该函数的 Xcode 开发者文档。文档包含该函数的更详细说明、支持函数的操作系统版本、函数所属的框架（在本例中为 UIKit），以及相关函数的参考。

 

在左侧导航中，可以快速跳转至开发者文档的不同部分。（也可以从“Window”菜单或使用快捷键 command-shift-0 访问开发者文档。）

 

文档窗口的顶部有一个搜索栏，可用于在文档中查找任何函数、类或框架的相关信息。使用此功能，可以在数千页的文档中快速搜索到所需答案，因此许多开发者都会在编程时选择让文档窗口一直保持打开状态，方便随时搜索 。

现在来浏览 Xcode 文档。首先，键入 UIViewController。当你键入时，Xcode 的自动填写功能会为你提供匹配的建议选项。在菜单中看到 UIViewController 后，将其选中即可显示 UIViewController 的文档。

在右栏的“On This Page”下方，点按“Topics”链接，可跳转到当前页面的后续内容。“Topics”部分会显示一列按主题分组的符号。符号是与特定类相关联的函数或变量。在文档中，这些符号按逻辑类别排序，而不是按字母顺序排序，这样非常便于用户查找感兴趣的项。

在构建新功能时，通过这些逻辑类别即可清楚地了解对象是否能够实现所需的功能。

浏览 UIViewController 的主题，尝试从中找到 view 属性的符号。点按此符号。

此时会显示所选属性的信息。大多数实例属性文档都遵循类似的模式：

  • 简短说明：属性定义的简短摘要
  • Declaration：用于访问属性的名称以及属性的关联类型
  • Discussion：深入讨论在使用属性时要考虑的一些重要细节
  • See Also：你可能感兴趣的与属性相关的其他符号

在左上角，有两个带有尖括号的按钮 。点按这些按钮，即可向后 和向前 查看你的浏览历史。点按后退按钮，返回到 UIViewController 的文档。找到函数 viewWillAppear(_:) 的符号。

​
大多数方法文档的模式与实例属性文档的模式基本类似，但可能会添加以下内容：

  • Parameters：如果函数具有参数（输入），则会列出用于访问参数的名称和简短的参数说明

随着你对本套课程的学习不断深入，你会更多地用到文档浏览器。

示例代码和主题

开发者文档中除了类型和方法的说明之外，还有其他很多内容。在你学习编程概念的过程中，深入阅读特定主题或试用一些示例代码会很有帮助。例如，UIKit 框架的顶层文档包含可帮助你入门的许多概述主题和文章。而 Views and Controls 子主题则包含 UIKit Catalog 示例代码项目。

要想成为一名成功的开发者，你需要了解如何阅读和理解文档。文档通常具有较强的技术性，第一次阅读时可能很难理解和消化。这需要多加练习。

本套课程的每节课中都设有“相关资源”部分，其中列出了相关参考指南和 API 文档的链接。请通读这些内容。对于所读的每个句子或概念，你未必都能理解，但这会锻炼你的学习能力，助你成为一名成功的开发者。
