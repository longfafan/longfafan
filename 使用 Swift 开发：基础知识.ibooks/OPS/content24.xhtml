到目前为止在本单元中，你已经学习了许多关于 Swift 的基础知识。现在是时候来运用这些知识了。

你的项目是编写一款名为“苹果派”的游戏。在这款简单的猜字游戏中，每位玩家都需要在有限的回合中猜出一个单词中的所有字母。每次猜测错误，树上都会掉下一颗苹果。如果玩家能够在所有苹果掉光之前正确猜出单词，则获胜。

作为一名新程序员，这个较大的项目可能让你望而生畏。请记住，你需要一步一个脚印才能构建一款好用的 app。如果你卡在某些步骤上，那么请反复复习相应的课程。你可以做到的！

在此版“苹果派”中，无论上一回合游戏胜负如何，都会立即开始新游戏。如果没有更多单词可供猜测，则会禁用所有按钮，并需要重新启动 app。

第 1 部分

构建界面

使用 iOS“App”模板创建新的项目，将项目命名为“苹果派”。这款游戏是专为 iPad 设计的，因此你将构建的界面不适用于小型 iOS 设备。要使 app 仅适用于 iPad，请在项目导航器中选择该项目文件，然后在“Deployment Info”下取消选择“Device for iPhone”下的复选框。

 

 

Storyboard 中的布局

打开 Main.storyboard 并使用“View as”按钮将设备和方向更改为 iPad 横屏模式。由于你将使用“自动布局”来构建界面，因此该界面同样适用于竖屏模式。回顾一下已完成 app 的样子。利用目前已经学到的关于 Interface Builder 工具的知识，你可以如何构造此界面？顶部有一张图像，下接排成网格状的按钮，然后是两行包含文本的标签。或许构建此界面的最简单方法是，使用垂直堆栈视图。请在“Object”库中找垂直堆栈视图并拖移到 iPad 画布上。

 

 

正如你在前面的课程中所学到的那样，可以通过在堆栈和超级视图之间添加约束，来确定堆栈视图的位置、宽度和高度。堆栈要从何处开始，又在何处结束？图像视图靠近顶部，最后一个标签位于屏幕底部附近。按钮网格从左边缘附近开始，在右边缘附近结束。因此，可以将堆栈视图约束为覆盖整个屏幕。选择堆栈视图，然后使用“Add New Constraints”工具添加四个约束。将弹出窗口顶部的四个字段全部设置为 0。红色指示线将变为实线，以表示哪些边距已设置了约束。完成后，点按“Add 4 Constraints”。

 

现在，可以开始向堆栈中添加视图和控件了。在“Object”库中搜索图像视图，并将一个图像视图拖移到堆栈视图中。由于此图像视图是堆栈内唯一的项，因此它将占据堆栈的整个空间。在“Attributes”检查器中将图像视图的“Content Mode”更改为“Aspect Fit”。此操作将保证图像的宽度和高度不会因图像视图的比例而变形。

 

将苹果树图像 (Tree 0.pdf...Tree 7.pdf) 从学生资源文件夹添加到 Assets.xcassets 文件夹中。图像添加完成之后，选中全部图像，然后从“Attributes”检查器的“Scales”菜单中选择“Single Scale”。

 

 

这些素材是矢量图像，这意味着图像可以动态缩放到任何大小，而无损画质。请尽可能使用包含矢量图像的 PDF 文件，这样就无需为各种设备提供多种比例的图像。

现在，可以返回 Storyboard 中的图像视图，并更新图像属性来使用其中一张苹果树图像。即使是在代码中的图像更新视图，你也可以直观地从 Storyboard 中查看界面。

 

按钮网格稍微有点复杂。你需要模拟标准 QWERTY 键盘的布局，每一行的按键数量都不尽相同。可以将每一行想象成单独的水平堆栈视图，并可以将这些行包含在一个垂直堆栈视图中。使用“Object”库在图像视图下添加一个垂直堆栈视图。你可以通过在文稿大纲中将项目拖移到任意位置，对堆栈内的项目重新排序。

你要添加的所有水平堆栈视图（或行）的大小都将相同，并且都会居中。选择新添加的垂直堆栈视图，然后打开“Attributes”检查器，并将“Alignment”更改为“Center”，“Distribution”更改为“Fill Equally”。此外，通过将“Spacing”设置为 5，在各行之间保持一些间距。

 

 

现在，在垂直堆栈中添加一个水平堆栈视图。你需要各列之间的间距与行间距相同，因此将“Spacing”设置为 5。将“Alignment”和“Distribution”都设置为“Fill”。

 

 

每行的按钮数量不同，第一行 10 个，第二行 9 个，第三行 7 个。使用“Object”库向水平堆栈视图添加一个按钮。键盘上的字母键通常是正方形的。要实现此外观，请选中按钮，并使用“Add New Constraints”按钮来添加“Aspect Ratio”约束。添加好约束后，但宽高比不正确。

 

使用按钮的“Size”检查器编辑“Aspect Ratio”约束，将比例系数设置为 1。这可确保按钮是正方形。

为了使按钮易于阅读，请使用“Attributes”检查器将字体大小设置为 30。

将按钮的文本更改为字母 Q，然后选择该按钮并将其拷贝 (command-C) 到剪贴板中。将按钮的新副本粘贴 (command-V) 到堆栈视图中，并重复此步骤，直到堆栈视图中包含 10 个按钮为止。参考你自己的键盘，使用“Attributes”检查器将每个按钮的文本更新为单个大写字母。

 

 

现在选中水平堆栈视图，并将其拷贝到剪贴板中。粘贴堆栈视图的两个新副本。参考你自己的键盘，删除多余的按钮，再次将每个按钮更新为合适的字母。在 Interface Builder 中，可以双击按钮来快速编辑按钮文本。

 

 

在第二个垂直堆栈视图中的按钮网格下方，从“Object”库添加两个标签。使用“Attributes”检查器将第一个标签的字体大小更改为 30.0，第二个更改为 20.0。按钮和标签都具有所谓的固有尺寸，由其中的文本大小决定。“自动布局”会使用其固有尺寸以及堆栈视图属性来正确地设置按钮网格的大小，同时树的图像视图可以根据需要进行收缩和扩展。

 

请确认 Interface Builder 中没有任何警告或错误，然后再继续构建。你可以构建并运行 app，以确保布局在不同的 iPad 模拟器上都正确显示，或使用 Interface Builder 中的“View as”功能来确保这一点。

 

创建插座变量和响应动作

在游戏过程中，当你正确猜出字母或新一回合开始时，标签的文本都会发生更改。当你猜错字母时，系统需要更改图像。这些字母按钮被按下后都需要被禁用，直到新一回合开始才会重新启用。要更新这些视图，请创建插座变量，以便在代码中引用这些视图。

打开助理编辑器，这样 ViewController 类定义就会出现在 Storyboard 的右侧。接下来，选中左侧的图像视图，按住 control 键并将其拖移到类定义区域内，然后松开鼠标按键。在出现的弹出菜单中，将该插座变量命名为 treeImageView。当你按下“Connect”按钮时，将会创建该插座变量的代码。

 

@IBOutlet var treeImageView: UIImageView!

 

对两个标签重复此过程。将顶部标签命名为 correctWordLabel，底部标签命名为 scoreLabel。

 

@IBOutlet var correctWordLabel: UILabel!

@IBOutlet var scoreLabel: UILabel!

 

为每个 UIButton 创建单独的插座变量会很繁琐。你可以改为创建一个包含按钮集合的插座变量。首先，选中标题为字母“Q”的第一个按钮。按住 control 键并将其拖移到类定义中以创建一个插座变量，然后松开鼠标按键。在出现的弹出菜单中，将“Connection”类型从“Outlet”更改为“Outlet Collection”。然后将插座变量集合的名称设置为 letterButtons。当你按下“Connect”时，系统将会创建一个引用按钮集合的插座变量。

 

@IBOutlet var letterButtons: [UIButton]!

 

现在点按插座变量集合旁边的圆圈，并拖移到另一个按钮。将会出现蓝色矩形，指示连接有效。松开鼠标按键，然后为场景中的每个按钮重复此步骤。

 

每个按钮还需要绑定到一个响应动作。同样，为每个按钮创建单独的响应动作会很繁琐。你可以改为创建一个相应动作，用于所有按钮在按下时的调用。首先，选中标题为字母“Q”的第一个按钮。按住 control 键并将其拖移到类定义中，然后松开鼠标按键。在出现的弹出菜单中，将“Connection”类型更改为“Action”。将响应动作的名称设置为 letterButtonPressed，并将自变量的类型更改为 UIButton。当你按下“Connect”时，即会创建以下方法：无论何时轻点字母按钮，都会将其禁用（玩家在同一回合中不可多次选择同一字母）。

 

@IBAction func letterButtonPressed(_ sender: UIButton) {

sender.isEnabled = false

}

 

为什么要在弹出菜单中将自变量类型从“Any”更改为“UIButton”？由于有 26 个按钮连接到同一响应动作，因此你将需要使用 sender 来确定具体哪个按钮触发了方法。如果 sender 的类型为 Any，你将无法访问 title 属性（我们稍后将需要该属性）。

按住 control 键并将其余的按钮拖移到现有的响应动作中。当你将鼠标悬停在方法附近时，如果出现一个蓝色矩形，就表示连接有效。

 

构建并运行 app，以验证在轻点每个按钮时是否会将其禁用。当你构建完 app 的其他部分之后，就会更加明确在此方法中还需要进行哪些操作。

第 2 部分

开始一轮游戏

现在可以开始处理“苹果派”游戏的逻辑了。

定义单词和回合数

你的第一项任务是提供一个供玩家猜测的单词列表。在 ViewController 的顶部，定义一个名为 listOfWords 的变量。使用单词填满此数组，例如食物名称、爱好、动物、家居用品或任何其他事物。为了简单起见，请仅使用小写字母。

var listOfWords = [”buccaneer”, “swift”, “glorious”,

“incandescent”, “bug”, “program”]

 

在 listOfWords 下面，定义了名为 incorrectMovesAllowed 的常量，其中确定了每轮所允许的猜错次数。数字越小，玩家越难获胜。由于一共有七张不同的苹果树图像，此值应该介于 1 到 7 之间。

 

let incorrectMovesAllowed = 7

定义获胜和失败的次数

在每一回合结束之后，底部标签将显示更新的获胜次数和失败次数。创建两个变量来分别容纳每个值，并将初始值都设置为 0。

 

var totalWins = 0

var totalLosses = 0

 

开始第一回合

启动 app 时将会调用 ViewController 的 viewDidLoad() 方法。此处是开始新一回合的绝佳场所。定义一个名为 newRound 的方法。

 

override func viewDidLoad() {

super.viewDidLoad()

newRound()

}

 

func newRound() {

 

}

 

开始一轮新回合意味着什么？每一回合都需要先选择新单词，并重置 incorrectMovesAllowed 中的玩家可操作次数。将游戏状态包含在 Game 结构体之内将会十分有用。通过在 Xcode 菜单栏中选择“File”>“New”>“File” (command-N)，在项目中创建一个新文件。选择“Swift File”作为模板，然后再选择“Next”。将文件命名为 Game.swift。

在 Game.swift 中定义一个名为 Game 的结构体。现在，你知道 Game 实例具有两个属性：单词，以及剩下的猜字次数。

 

import Foundation

 

struct Game {

var word: String

var incorrectMovesRemaining: Int

}

 

回到 newRound() 方法中，你可以创建一个 Game 的新实例。你应该创建一个属性来容纳当前游戏值，以便在整个视图控制器代码内对其进行更新。你可以在构造器中向 Game 分配一个新单词，具体方法是从 listOfWords 集合中移除第一个值，并将 incorrectMovesRemaining 设置为 incorrectMovesAllowed 中储存的可操作次数。

 

var currentGame: Game!

 

func newRound() {

let newWord = listOfWords.removeFirst()

currentGame = Game(word: newWord, incorrectMovesRemaining:

incorrectMovesAllowed)

}

 

为何 currentGame 变量末尾有感叹号？因为在 app 启动后到第一回合开始之前的短暂瞬间，currentGame 并没有值。你将在下一个单元中学习这个概念。目前你只需要知道，感叹号表示该属性短期没有值也是可以的。

既然你已经开始了新一回合，那么接下来你需要更新界面才能反映新游戏。另行创建一个名为 updateUI() 的方法来处理界面更新，然后在 newRound 的末尾调用此方法。在此方法中，得分标签和图像视图这两部分界面可以使用到目前为止所编写的代码来更新。得分标签使用简单的字符串插值将 totalWins 和 totalLosses 合并为一个字符串。由于每张树图像都命名为“Tree X”（“X”指剩余可操作次数），因此你可以再次使用字符串插值来构造图像名称。

 

func newRound() {

let newWord = listOfWords.removeFirst()

currentGame = Game(word: newWord, incorrectMovesRemaining:

incorrectMovesAllowed)

updateUI()

}

 

func updateUI() {

scoreLabel.text = “Wins: \(totalWins), Losses: \

(totalLosses)”

treeImageView.image = UIImage(named: “Tree \

(currentGame.incorrectMovesRemaining)”)

}

 

构建并运行应用程序。得分标签和图像视图应进行更新来反映新一回合的开始。非常好！

第 3 部分

更新游戏状态

到目前为止，你已经构建了一个有效的界面，并成功开始了新一回合。现在你需要添加一些代码来在游戏中实现获胜或失败。

提取按钮标题

目前，letterButtonPressed(_:) 方法会禁用玩家猜词用过的按钮，却并未更新游戏状态。当点按按钮时，应该读取此按钮的标题，并判断玩家正尝试猜测的单词中是否包含该字母。首先，读取按钮处于“normal”状态时所使用的标题。并非所有按钮都有标题，因此你需要添加一个感叹号来告诉 Swift 编译器：你的按钮具有标题。（你将在下一个单元中了解有关该感叹号的更多信息。）你应该将字母小写，因为小写形式的内容在比较时将更加容易，然后将其从 String 转换为 Character。

 

@IBAction func letterButtonPressed(_ sender: UIButton) {

sender.isEnabled = false

let letterString = sender.title(for: .normal)!

let letter = Character(letterString.lowercased())

}

 

猜测字母

获取玩家猜测的字母之后，接下来应该如何处理呢？Game 可管理剩余可操作次数，但不知道在该回合中选择过哪些字母。你应该向 Game 添加一个名为 guessedLetters 的字符集合来跟踪所选字母。然后在 Game 中添加一个方法，来接收 Character 并将其添加到集合，以及根据需要更新 incorrectMovesRemaining。（添加 guessedLetters 属性之后，你将需要更新初始的 currentGame。）

 

struct Game {

var word: String

var incorrectMovesRemaining: Int

var guessedLetters: [Character]

 

mutating func playerGuessed(letter: Character) {

guessedLetters.append(letter)

if !word.contains(letter) {

incorrectMovesRemaining -= 1

}

}

}

 

func newRound() {

let newWord = listOfWords.removeFirst()

currentGame = Game(word: newWord, incorrectMovesRemaining:

incorrectMovesAllowed, guessedLetters: [])

updateUI()

}

 

@IBAction func letterButtonPressed(_ sender: UIButton) {

sender.isEnabled = false

let letterString = sender.title(for: .normal)!

let letter = Character(letterString.lowercased())

currentGame.playerGuessed(letter: letter)

updateUI()

}

 

构建并运行应用程序。当你按下每个按钮时，系统会将对应的字符添加到玩家的已猜字母列表中，并且每猜错一次，incorrectMovesRemaining 就会减 1。

第 4 部分

创建提示单词

通过使用 word 和 guessedLetters，现在可以计算出一个隐藏缺失字母的单词。例如，如果这一回合的单词为“buccaneer”并且用户已经猜出了字母“c”、“e”、“b”和“j”，则玩家应该可以在屏幕底部看到“b_cc__ee_”。

首先，在 Game 定义中创建一个名为 formattedWord 的计算型属性。以下是 formattedWord 的一种计算方法：

  • 先使用一个空的字符串变量。
  • 循环遍历 word 中的每个字符。
  • 如果 guessedLetters 中存在该字符，则将其转换为字符串，然后再附加到变量。
  • 否则，将 _ 附加到变量。

 

var formattedWord: String {

var guessedWord = “”

for letter in word {

if guessedLetters.contains(letter) {

guessedWord += “\(letter)”

} else {

guessedWord += “_”

}

}

return guessedWord

}

 

既然 formattedWord 是 UI 可以显示的属性，那么请尝试将其用于 updateUI() 函数里面的 currentWordLabel 的文本。

 

func updateUI() {

correctWordLabel.text = currentGame.formattedWord

scoreLabel.text = “Wins: \(totalWins), Losses: \

(totalLosses)”

treeImageView.image = UIImage(named: “Tree \

(currentGame.incorrectMovesRemaining)”)

}

 

构建并运行应用程序。当字母被选中之后，会被添加到 guessedLetters 集合中，并且如果在 updateUI() 中访问该字母，则会重新计算 formattedWord。

你可能注意到一个新问题：由于多条下划线在界面中显示为一条实线，因此很难判断一个单词中究竟有多少个字母。一种解决方案是在计算 formattedWord 时添加空格。但这纯粹是界面改善问题，而不影响计算的数据。因此，更好的解决方案是在更新 correctWordLabel 的文本时添加空格。

要正确设置 correctWordLabel 文本，可以使用对字符串数组进行操作的 Swift 方法，称为 joined(separator:)。此函数会将字符串集合串联成一个字符串，中间以给定的值作为分隔符。下面是一个示例：

 

let cast = [”Vivien”, “Marlon”, “Kim”, “Karl”]

let list = cast.joined(separator: “, “)

print(list) // “Vivien, Marlon, Kim, Karl”

 

设想一下，如何使用 joined(separator:) 方法来设置 correctWordLabel？首先，将 formattedWord 中的字符数组转换为字符串数组。使用 for 循环将所有新创建的字符串储存到 [String] 数组中。然后，可以调用 joined(separator:) 方法将新集合串联在一起，并以空格分隔。

 

func updateUI() {

var letters = [String]()

for letter in currentGame.formattedWord {

letters.append(String(letter))

}

let wordWithSpacing = letters.joined(separator: “ “)

correctWordLabel.text = wordWithSpacing

scoreLabel.text = “Wins: \(totalWins), Losses:

\(totalLosses)”

treeImageView.image = UIImage(named: “Tree

\(currentGame.incorrectMovesRemaining)”)

}

 

构建并运行应用程序，可以看到字母与下划线之间出现了清晰间隔。

第 5 部分

处理获胜或失败

“苹果派”开始像一款真正的游戏了。随着每个按钮被按下，游戏的回合继续向前推进，但是却存在两个明显的问题。第一个问题是 incorrectMovesRemaining 可能会小于 0，第二个问题是玩家即使猜中所有字母也无法获胜。

sender.isEnabled = false

let letterString = sender.title(for: .normal)!

currentGame.playerGuessed(letter: letter)

}

如何确定游戏的状态是获胜、失败，还是玩家继续？如果 incorrectMovesRemaining 达到 0，则游戏失败。当游戏失败时，应递增 totalLosses。如果玩家尚未失败，并且当前游戏的 word 属性等于 formattedWord（如果已成功猜出所有字母，则 formattedWord 不会包含任何下划线），则可以确定游戏获胜。当游戏获胜时，应递增 totalWins。如果游戏尚未获胜或失败，则应该允许玩家继续猜字，并应更新界面。

 

func updateGameState() {

if currentGame.incorrectMovesRemaining == 0 {

totalLosses += 1

} else if currentGame.word == currentGame.formattedWord {

totalWins += 1

} else {

updateUI()

}

}

 

构建并运行应用程序。游戏能否正常运行？已经很接近成功了，但是在玩家获胜或失败之后并未开始新一回合。当 totalWins 或 totalLosses 发生更改时，可以开始新一回合。所以，此时是向 totalWins 和 totalLosses 添加 didSet 属性观察器的绝佳时机。

 

var totalWins = 0 {

}

重新启用按钮和修复崩溃

看起来就快要完成了！你已经能够顺利完成“苹果派”的一个回合，但是新一回合尚未重新启用字母按钮。此外，如果你尝试开始新一回合，但没有更多单词可供选择，游戏将崩溃。

所以，newRound 的逻辑需要再智能一点。如果 listOfWords 不为空，应该执行与先前一样的操作，而且还要重新启用所有按钮。如果没有其他单词可供游戏使用，请禁用所有按钮，这样玩家就无法继续游戏了。

 

func newRound() {

if !listOfWords.isEmpty {

let newWord = listOfWords.removeFirst()

currentGame = Game(word: newWord,

incorrectMovesRemaining: incorrectMovesAllowed,

guessedLetters: [])

enableLetterButtons(true)

updateUI()

} else {

enableLetterButtons(false)

}

}

 

enableLetterButtons(:) 方法相当简单。它采用 Bool 作为自变量，并通过循环遍历按钮集合来使用参数对按钮进行启用或禁用。

 

func enableLetterButtons(_ enable: Bool) {

for button in letterButtons {

button.isEnabled = enable

}

}

如果你构建并运行应用程序，应该可以遍历“苹果派”的所有回合，直到猜词列表用完为止，并禁用所有按钮。
