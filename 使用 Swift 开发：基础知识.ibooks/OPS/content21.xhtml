规划 app

大多数 app 都是从简单的想法开始的，我们可以把这个想法总结为一句话。考虑一下每天都使用的 app。它们也许就是从以下直接的想法开始的：

  • “发送和接收信息。”
  • “和我的朋友和家人分享照片。”
  • “发布短信息，并查看其他人撰写的短信息。”

作为一名 app 开发者，你将学习如何使用一句话来概述一个 app 或功能，并规划其实现方式。

在此项目中，你的任务是构建一款介绍自己的 app。本指南将为你提供一个框架，可以为你的 app 需要包含哪些内容做参考，但注意此 app 介绍的是你自己。请加入你自己的想法，让它成为你的 app。

考虑一下要如何向以前从未见过的人介绍自己。你希望确保别人了解关于你哪方面的信息？首先，写下你的名字，然后再花几分钟写下希望分享的关于自己的内容。使用以下提示可帮助你为 app 构想出足够的信息。

  • 你是谁？你的生活是什么样子的？
  • 你如何度过日常生活中的某一天？
  • 如果你有足够的时间和金钱，你会如何处理自己生活？
  • 你最喜欢的事情或活动有哪些？

请将你认为其他人将会希望了解的其他信息包含在内。

有时显示在 app 中的信息是持续的，并且永远不会更改。此类信息在编程中称为“静态数据”。你需要在规划 app 时头脑风暴一下，列出关于自己的静态数据。在后面的 app 中，你将处理可变数据，称为“动态数据”。

## About Me

My name is Alexis Kay. I’m a dedicated student, I play lacrosse,

I love animals, and I’m looking for an internship in Marine

Biology.

I want to sail around the world, spend time in new countries,

当你构想出独一无二的信息时，你可能会想到多种方式来显示这些信息并与他人分享。对于此项目，尝试考虑一下如何在一个屏幕中显示此信息。

由于仍处于规划阶段，建议你打开“Pages 文稿”或“Keynote 讲演”，创建一个粗略的大纲来列出想要 app 显示的数据。

你可能发现在一个 iOS 设备屏幕中无法显示希望包含的所有信息，因此需要启用滚动功能。在本节课中，你将创建项目的第一部分：添加姓名、个人资料图像，以及关于自己的几句话。在后面的课程中，你将完成第二部分：在 Interface Builder 中使用更高级的布局功能来向滚动视图中添加图像和标签。

创建项目

使用 iOS“App”模板创建新的 Xcode 项目。创建项目时，请确保将界面选项设置为“Storyboard”。将项目命名为“Hello”，并将其存储至项目文件夹中。

 

输入你的信息

打开 Main.storyboard 文件。你将使用初始场景来添加自己的信息。由于 app 将始终显示相同的数据，因此可以在 Interface Builder 中使用标签或图像将静态数据直接构建到屏幕中。在后面的课程中，你将学习编写用于显示动态数据的代码。

当你选择字体、颜色和其他显示设置时，演示样式应尽量整洁，并可在一个 iOS 设备屏幕中完全显示。避免使用难以辨认的字体、冲突的配色方案和令人感到困惑的文本布局。

 

 

当你放置视图时，请利用布局参考线让内容居中显示，并仔细留意边距以保证对象之间有足够的间距。还应避免将内容放置到屏幕顶部的状态栏中。

添加用于显示文本的标签

针对每个要包括的文本项，将“Label”对象从“Library”拖移到场景中。添加几个标签，至少能显示你的姓名，以及一两句自我介绍。当你在后面的课程中完成此项目时，可以添加剩余部分。

一次在 Storyboard 中选择一个标签，使用“Attributes”检查器设置标签文本，并选择合适的字体。如果希望标签显示为标题，请对字体进行设置，使其比默认文本更大、更粗。你想要使用自定字体吗？打开“Font”弹出式菜单，并选择“Custom”，以在 Mac 上已安装的全部字体中进行选择。

 

 

可在检查器中尝试调整不同的属性，以了解这些属性分别如何修改标签的外观。例如，你可以更改文本的对齐方式或标签显示的行数。默认情况下，标签仅显示一行文本，这可能会截断行末的剩余文本。

在使用静态信息时，你可以一直增加行数，直到标签能够显示所有文本为止。不过还有一种更好的方法可根据需要显示文本行数，以便显示所提供的完整文本。

查看 UILabel 文档，并找到 numberOfLines 符号。在“Description”部分，你将看到以下文本：

 

This property controls the maximum number of lines to use in

order to fit the label’s text into its bounding rectangle. pThe

default value for this property is 1.To remove any maximum

limit, and use as many lines as needed, set the value of this

property to 0.

 

当你希望标签中的文本显示为多行时，numberOfLines 属性十分有用。请随时参阅 UILabel 文档，尝试发现可以让文本适合标签的其他方法。

将你想要包含在“Hello”app 中的文本添加到的所有标签中。

添加图像

如果没有你的照片，你的个人介绍是不完整的。如果没有喜欢的照片，可以使用“Photo Booth”app 自拍。请确保将图像文件存放在桌面或“访达”中的便捷位置。如果有更多图像，你将有机会在后面的课程中将照片添加到 app 中。

现在继续并将图像文件添加到场景中，让 app 显示该图像。

向 Xcode 项目中添加图像的方式如下：

 

  1. 打开导航器区域。确保左侧的导航器区域已打开，并且项目导航器中显示你的文件列表。如果你是使用小屏幕设备进行操作，可能会希望关闭右侧的检查器区域。
  2. 选择 Assets.xcassets。Xcode 已经自动向你的项目添加了一个素材目录。这是存放将在 app 中使用的图像的最佳位置。
  1. 拖入图像文件。将图像添加到 Storyboard 中，并在文稿大纲中为其命名。该图像现已成为 app 的一部分。

既然你已经添加了自己的照片图像，接下来可以在 Interface Builder 中设置图像视图。返回 Main.storyboard。将“Image View”对象从“Library”拖移到场景中。

使用“Attributes”检查器从“Image”弹出式菜单中选择图像名称。这一操作将会向刚刚创建的图像视图中分配该图像。

在前一课中，你已经了解到图像视图就像是包覆在要展示的图像周围的边框。图像视图的大小决定了图像可显示的总大小。通过设置图像视图的内容模式，你可以控制：是否将在边框内显示图像，如此可能会留有空白区域；是否将会拉伸图像以适合框架，如此可能会导致图像变形；是否将拉伸图像以填满整个框架，如此可能会裁切掉部分图像。

在下面的表格中，你可以看到方形 UIImageView 在显示非方形图像时，有哪些最常见的 contentMode 选项。

 

请为添加到项目中的图像选择正确的内容模式。
