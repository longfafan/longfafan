到目前为止，你已在本节课中学习了 Xcode 和 Interface Builder 的基础知识。现在，你将运用所学的这些工具来构建一个项目。

在本指导项目结束时，你将会有一个创建好的“灯光”app，它可以将屏幕从黑色变为白色。当用户轻点按钮时，还可以变回原样。要成功构建该 app，需要使用 Xcode 文档、设置断点，以及创建插座变量和响应动作。

此项目将涉及一些代码的修改，而你可能对 Swift 语言还不太熟悉。如果项目中的代码相关部分让你觉得很难，请不要灰心。只要坚持下去就会好的！

第 1 部分

创建按钮和响应动作

使用你在前面课程中使用过的 iOS“App”模板来创建一个新的 Xcode 项目。创建项目时，请确保将界面选项设置为“Storyboard”。请将该项目命名为“Light”。当你构建并运行该项目时，你会发现还没有用户可与之交互的内容。这种状况很快就会改变。

在项目导航器中，选择 Main.storyboard，在 Interface Builder 中打开 Storyboard。

 

由 Xcode 创建的初始界面包含一个 ViewController 实例。ViewController 是 UIViewController 的子类，是“Single View App”模板中预定义的内容，会在项目导航器中列出。你可以点按“Show Document Outline”按钮 ，以显示 Main.storyboard 中定义的所有视图控制器 。

 

 

在文稿大纲中选择“视图控制器”，然后点按检查器区域顶部的“Identity”检查器按钮 ，以确认此特定视图控制器的类型为 ViewController。

如果你使用的是“Simulator”，可以使用画布底部的“View as”按钮来调整画布的大小。将画布大小设置为 iPhone 11 Pro 配置，然后从 Xcode 工具栏左端的菜单中选择 iPhone 11 Pro 模拟器。

如果使用的是你自己的设备来进行此项目，那么视图控制器的大小可能与你的屏幕大小不一样。要使之与你的设备相匹配，可以使用画布底部的“View as”按钮来调整画布的大小，但请始终保持竖屏模式。

 

 

选择视图控制器的视图，然后选择“Attributes”检查器 ，其可用于自定任何界面元素的属性。此视图的背景颜色已设置为白色。这是 app 启动时的理想状态，因此现在不需要更改任何内容。

接下来，添加一个用于更改视图背景颜色的按钮，模拟开灯和关灯时的情景。按工具栏右上角的 + 按钮以打开“Library”。前面课程已介绍过，此库包含一个可添加到 Storyboard 中的常见对象的列表。滚动浏览该列表，或使用库区域顶部的搜索过滤器。

 

在该库中找到“Button”对象，然后将其拖到视图的顶部。释放光标时，从文稿大纲应该可以看出，此按钮已被添加为子视图。

 

请 将该按钮移到视图的左上角。边距对齐参考线有助于将对象对准到正确的位置。

 

请 为按钮指定一个要在被点按或轻点时执行的响应动作。使用“Adjust Editor Options”按钮添加一个助理编辑器 ，然后选择“Assistant Editor”，Xcode 工作区会分成两个部分：Interface Builder 及其相应的代码。

 

 

现在按住 control 键并拖移（或右键点按）按钮到 ViewController 类定义中的可用区域内。当你拖移时，会从按钮延伸出一条蓝色的线，而你的光标下方会显示一个蓝色的横条区域，指示可创建连接的有效位置。

当你释放鼠标光标时，系统会提示你创建插座变量或定义响应动作。请将“Connection”更改为“Action”，然后将“Name”设置为“buttonPressed”。当你点按“Connect”按钮时，Xcode 会创建一个只要轻点按钮即会调用的新方法。你可能会注意到，buttonPressed(_:) 方法之前有一个 @IBAction 关键字。@IBAction 会通知 Xcode 可以在 Storyboard 中的可视控件和该函数之间建立关系。

构建并运行 app。当你按下按钮时，按钮没有反应，因为 buttonPressed(_:) 方法中没有可执行的代码。要验证是否调用了该方法，可以在方法定义中放入一个断点。现在按下按钮，会触发断点。

 

现在，你已确认响应动作一切正常，可以添加一些代码来更改背景颜色了。在继续之前，请先删除断点。

 

第 2 部分

更改背景

如果灯是打开的（背景为白色），那么应该关灯。否则，如果灯是关闭的（背景为黑色），那么应该开灯。“灯是打开的”这句话既可以是真命题，也可以是假命题。因此，可以使用布尔值来确定如何更改背景颜色。

在 ViewController 类定义的顶部附近，创建一个称为 lightOn 的变量，并将初始值设置为 true，因为屏幕一开始为白色背景。

 

var lightOn = true

 

每次轻点按钮时，该值都应该从 true 变为 false，或者从 false 变为 true。Swift 布尔值有一个 toggle() 方法，正好可以实现此功能。由于只要轻点按钮即会调用 buttonPressed(_:) 方法，所以应在此处进行更改。

 

@IBAction func buttonPressed(_ sender: Any) {

lightOn.toggle()

}

 

在更改值之后，可以使用新值来确定如何更改背景颜色。如果灯应该是关闭的，则将背景颜色更改为黑色；如果灯应该是打开的，则将背景颜色更改为白色。你可以使用一个简单的 if 语句来编写此逻辑。

 

@IBAction func buttonPressed(_ sender: Any) {

lightOn.toggle()

if lightOn {

view.backgroundColor = .white

} else {

view.backgroundColor = .black

}

}

 

构建并运行应用程序，背景应该能在每次按下按钮时正确地变换了。非常好！

为了避免应用程序变得臃肿，你应该始终确保代码井然有序。你可以将 if 语句放入到一个用于处理整个用户界面更新的新方法中，而不是将其直接放入到 buttonPressed(_:) 方法中。

 

选择整个 if 语句，然后从“Editor”菜单中选择“Refactor”>”Extract to Method”。

 

Xcode 会将代码移入到新方法中，然后进入特殊的编辑模式，以允许你设置新方法的名称，同时在方法的调用位置进行更新。请键入 updateUI，然后按下 return 键以设置名称。

Xcode 将所选代码移入到了名为 updateUI() 的新方法中，然后在代码的原来位置添加了对 updateUI() 的调用。

结果应该如下所示：

fileprivate func updateUI() {

if lightOn {

view.backgroundColor = .white

} else {

view.backgroundColor = .black

}

}

 

@IBAction func buttonPressed(_ sender: Any) {

lightOn.toggle()

updateUI()

}

 

注意 Xcode 在方法名称前面使用了 fileprivate 关键字。Swift 使用这个特殊的访问修饰符来指定可以从何处调用此方法。如果要使用 Xcode 重构功能，并需要在方法定义所在文件之外调用该方法，请移除其前面的 fileprivate 关键字。

这是小 app 的一个小改变。这种模式会贯穿于本书之中，而且你会发现，如果将更新视图的代码全部包含在一个方法中，调试界面问题就会容易得多。

第 3 部分

更新按钮文本

现在，你已成功设置了一个按钮来更改视图的背景颜色，但无论灯的状态如何，按钮文本的标题都不会改变。目前，还无法在代码中引用新创建的按钮。首先，必须为按钮创建一个插座变量。按住 control 键的同时，将 Interface Builder 中的按钮拖移到编辑器区域中位于视图控制器定义顶部的空白区域。在弹出窗口中，确认在“Connection”中选择了“Outlet”，然后在“Name”字段中输入“lightButton”。

弹出菜单应如下所示：

 

点按“Connect”。

 

根据前面课程的介绍，该插座变量的详细信息从左到右依次为：

  • 圆圈：实心圆圈表示已连接插座变量。如果属性没有任何连接的内容，圆圈是空心的。
  • @IBOutlet：此关键字会通知 Xcode 此行上的属性是一个插座变量。
  • var lightButton：这是对“lightButton”的属性的声明。
  • UIButton!：该属性的类型是 UIButton!。感叹号表示，在未连接插座变量的情况下尝试访问此属性会导致程序崩溃。在后面学习可选类型的单元中，你会了解到有关感叹号的更多信息。

插座变量已创建完毕，现在可以通过编程方式对按钮进行更改了。首先，确定应在何处更新按钮的标题，以及应执行什么样的响应动作。

 

 

请在 buttonPressed(_:) 方法的第一行添加一个断点，然后构建并运行 app。正常情况下，当你按下按钮更改颜色时，会执行相应的响应动作 buttonPressed(_:)。但是，由于添加了断点，代码会在执行有断点的代码行之前暂停。

点按“Step over”按钮 来运行第一行。现在，app 会在运行 updateUI() 方法之前暂停。点按“Step into”按钮 。现在，app 会在执行 updateUI 的第一行之前暂停。

 

 

花点时间来读一下 updateUI() 方法的主体。Swift 是一种可读性非常好的编程语言。对于上面的代码行，你可以按如下进行理解：“如果灯是打开的，视图的背景颜色应为白色。否则，视图的背景颜色应为黑色。”

现在思考一下应如何确定按钮的标题。如果灯是打开的，按钮的标题应显示为“Off”。否则，按钮的标题应显示为“On”。按照这样的逻辑，按钮标题的更新方式应与视图背景颜色的更新方式类似。

现在，你已知道应在何处更新按钮文本了，那就是在 updateUI() 方法中。但是，具体要使用哪个函数来实现这个功能呢？前面已介绍过，Xcode 文档始终可以为你提供帮助。只要有不确定的地方，就可以去文档中寻找答案。

 

 

要了解如何设置 lightButton 的标题，首先，阅读文档中有关其类型 UIButton 的相关信息。使用 Xcode 菜单中的“Window”>“Developer Documentation”可以直接访问文档，这会打开一个文档查看器窗口。

在文档搜索栏中开始输入 UIButton。自动填写功能会快速提供建议选项。在菜单中看到 UIButton 后，将其选中。

 

向下滚动到“Symbols”部分，找到一个称为“Configuring the Button”的列表。在那里，你会找到一个用于设置标题的函数，即 func setTitle(String?, for: UIControl.State)。

点按函数名称可查看其声明和参数列表。第一个参数是所需的文本，第二个参数是 UIControl.State。UIControl.State 代表按钮在不同情况下可能具有的状态，例如闲置时、被用户轻点时或被停用时。点按“Declaration”部分中的 UIControl.State。

 

在这个新的符号列表中，“Constants”部分包含一个 normal 常量 ，其对应于按钮在屏幕上被启用但处于闲置时的状态。这是设置按钮标题的正确状态。

 

 

在新学的 updateUI() 方法中，可以将之前添加的断点移除。经过调整后，你将得到类似于以下的内容：

 

func updateUI() {

if lightOn {

view.backgroundColor = .white

lightButton.setTitle(”Off”, for: .normal)

} else {

view.backgroundColor = .black

lightButton.setTitle(”On”, for: .normal)

}

}

 

构建并运行 app。现在，点按或轻点按钮就可以更改背景颜色和按钮文本了。但有一个程序错误：在点按按钮之前，按钮文本仍显示为“Button”，而不是“On”或“Off”。如何更新代码来解决此问题呢？

你的项目中的 updateUI() 代码可确保按钮的文本与灯的开关状态相匹配。只需在视图第一次载入时调用该函数即可。这样，按钮就能在视图显示时更新，而不是在用户第一次轻点时更新。你可以在 viewDidLoad() 函数中运行设置代码，该函数已在视图控制器子类中定义，当视图控制器准备显示在屏幕上时，会对其进行调用。

在此方法中调用 updateUI()，如下所示：

override func viewDidLoad() {

super.viewDidLoad()

updateUI()

}

 

再次构建并运行 app。现在，启动 app 时，按钮文本应会显示为“On”。

第 4 部分

改进用户体验

你目前有了一个可运行的 app。这是一个好的开始。现在，花点时间考虑一下该 app 的设计和用户体验。还有什么地方可以改进？

再看一看按钮，你可能会觉得文本有点多余。从背景颜色上即可对灯的当前状态（打开或关闭）一目了然。对于用户而言，按钮的文本是否为必需项？

按钮在屏幕上的位置重要吗？另外，大部分屏幕都不可轻点，是不是有点奇怪？

将按钮文本移除并使按钮填满整个屏幕可能会更好。这样，用户就可以轻点屏幕上的任何位置来开灯和关灯了。在构建任何 app 时，一定要抓住 app 设计中的微妙细节，让用户界面尽可能简单直观、功能强大。在本例中，用户可以轻点屏幕上的任何位置来切换背景颜色，这很合情合理。

首先，需要调整按钮大小，使其填满屏幕。在 Interface Builder 中将按钮选中，然后拖移按钮的左上角和右下角，使之覆盖整个白色视图区域。

 

 

移除按钮的标题文本。按钮的可轻点区域仍将按预期方式执行，虽然现在看起来好像只有白色视图存在于画布上。

 

移除按钮文本后，就可以将 updateUI() 中用于更改按钮标题的代码行移除。

func updateUI() {

if lightOn {

view.backgroundColor = .white

} else {

view.backgroundColor = .black

}

}

 

在这个更新的设计中，当用户在使用 app 时，按钮的属性不需要改变。这意味着，你在项目之初创建的插座变量不再需要了。作为开发者，你会希望代码尽可能简洁明了，将 IBOutlet 删去就是个好方法。

首先，断开按钮与其插座变量之间的连接。要执行此操作，请在 Interface Builder 中将按钮选中，此时会显示连接检查器 。在这里，可以查看与特定对象连接的所有事件和插座变量。在本例中，可以看到按钮已与 buttonPressed: 响应动作和 lightButton 插座变量绑定。​
点按该插座变量旁边的 X，将其移除。

由于不再有任何对象与 lightButton 相连接，所以 @IBOutlet 旁边的小圆圈不再是实心的。

 

现在可以安全地删除 lightButton 的声明。请最后一次构建并运行 app，测试一下改进后的新设计。你现在应该可以轻点或点按屏幕上的任何位置来更改背景颜色了。

if-else 语句中的两行代码所执行的操作相同（都是设置 backgroundColor），因此如果使用三元运算符而不是 if-else 语句，updateUI() 方法可能会看起来更简洁。

 

func updateUI() {

view.backgroundColor = lightOn ? .white : .black

}

 

再次构建并运行你的应用程序，验证一下三元运算符是否能按预期方式运行。

总结

你已成功创建了一个 app，当用户轻点按钮时，屏幕即会在黑色与白色之间进行切换，就像手电筒一样。此外，你还简化了该 app 的设计，并合理地通过删除按钮标题以及将整个屏幕都变成可轻点的区域来改进了用户体验。尽管你对 Swift 的了解有限，但你可以逐行调试代码，参考文档来完成此项目。

你会觉得浏览 Xcode 或使用 Interface Builder 很难吗？如果觉得很难，请复习前面详细讨论这些内容的课程。你会发现，构建得越多，就会越熟练。

当你要开始下一个单元时，手边请备好开发者文档，以便随时翻阅。只有使用好的工具和资源，开发者才能开发出优秀的产品。
