Xcode 有一个称为 Interface Builder 的内置工具，可用于轻松而直观地创建界面。在本节课中，你将学习如何浏览 Interface Builder、如何在画布上添加元素，以及如何使用代码与这些元素进行交互。

  • 如何在不编译 app 的情况下预览用户界面

学习 Interface Builder 基础知识的最好方法是深入了解 Xcode 并钻研其中的某些功能。开始时，使用 iOS“App”模板创建新的 Xcode 项目。创建项目时，请确保将界面选项设置为“Storyboard”。将该项目命名为“IBBasics”。

Storyboard

当你从项目导航器中选择某个 XIB 文件 (.xib) 或 Storyboard 文件 (.storyboard) 时，Interface Builder 即会打开。

XIB 文件包含单个可视控件的用户界面，如全屏视图、表格视图单元格或自定 UI 控件。在引入 Storyboard 之前，使用 XIB 的情况更多一些。你可能听到过一些经验丰富的 macOS 或 iOS 开发者将 XIB 文件称为“Nib”文件。XIB 格式对于某些情况依旧很有用，但本节课将着重介绍 Storyboard。

与 XIB 不同，Storyboard 文件包含界面的多个部分，还定义了一个或多个屏幕的布局，以及如何在屏幕之间进行切换。作为开发者，你会发现同时查看多个屏幕可以帮助你理解 app 中的流程。

 

在项目导航器中，点按 Main.storyboard，在 Interface Builder 中打开该文件。在屏幕的中央，你会看到一个场景，即空画布上有一个普通的白色视图。这个场景称为“初始视图控制器”，它是你打开 app 时看到的第一个屏幕。当你向 Storyboard 添加更多的场景时，可以将它们拖移到画布上的任意位置。要同时查看多个视图控制器，请在多点触控板上通过双指开合来缩小画布或放大某个特定视图。如果没有触控板，也可以使用画布底部附近的缩放按钮来达到同样的效果。

构建并运行项目。“Simulator”会显示你在 Storyboard 中看到的那个白色屏幕。App 是如何知道要显示此屏幕的？要了解此问题的答案，请在项目导航器中选择最顶部的文件 (IBBasics)，然后在“General”标题下找到“Deployment Info”部分。“Main Interface”中的输入内容决定了 app 在启动时要先载入哪个 Storyboard 文件。在创建项目时，如果使用的是“Single View App”模板，那么预配置为使用 Main.storyboard。

要更改初始视图控制器，可以将入口点（向右箭头）移到所需视图控制器的左侧。目前，此项目的 Main.storyboard 只有一个视图控制器，因此无法更改。

Interface Builder 布局

主画布的左侧是文稿大纲视图。要显示此视图，请点按画布左下角的“Show Document Outline”按钮 。文稿大纲显示场景中的每个视图控制器，并以层次结构形式列出了每个视图控制器中的各个元素。点按每个项目左侧的灰色三角形可查看相应内容。你可能会注意到，点按大纲中的视图时，画布上的相应元素就会高亮显示出来。

 

选择 Xcode 工具栏中的“Library”按钮 ，以显示“Object”库。“Object”库包含按钮、文本字段、标签以及可添加到场景中的其他视图控制器。

滚动浏览“Object”库以找到“Button”，将其选中并从库中拖移到视图上。当你将按钮拖向视图的左上角时，请尝试使用布局参考线。当你在 Interface Builder 中放置视图对象或调整其大小时，布局参考线会显示为一条蓝色虚线。布局参考线可帮助你居中显示内容、在对象之间使用适当的边距和间距，以及避免将内容置于屏幕顶部的状态栏中。

然后，松开鼠标或触控板，将按钮插入视图中。请注意，此时按钮对象也会显示在文稿大纲中。

 

 

在屏幕右侧，你会看到检查器区域，如果看不到，请按下工具栏左上角的“Hide or show the inspectors”按钮，或使用键盘快捷键 option-command-0。

当你位于 Interface Builder 中时，除了“File”、“History”和“快速帮助”检查器（始终显示）之外，检查器区域的顶部还会显示四个上下文相关的检查器。要了解这些不同的检查器，以及如何通过它们来自定视图中的对象，请在文稿大纲或场景本身中选择你刚刚添加的按钮。

插座变量和响应动作

通常，在代码中引用可视控件时，你会希望能够在运行时或当 app 已运行时调整这些元素。从 Interface Builder 到代码之间的这种引用称为插座变量。如果你有一个想让用户与之进行交互的对象，那么需要创建一个响应动作。响应动作是对一段代码的引用，当用户与对象交互时即会执行该段代码。

在大纲视图中选择视图控制器，然后选择“Identity”检查器。你在创建项目时所选择的模板已将“Custom Class”设置为 ViewController。通过点按“Adjust Editor Options”按钮 并选择“Assistant”来打开助理编辑器。

 

ViewController 的源代码与 Storyboard 一起显示（因为它对应于“Identity”检查器中的“Custom Class”字段）。

此时，ViewController 类还无法访问你所添加的按钮。要使代码能够访问对象，需要创建插座变量。

创建插座变量

按住 control 键并点按（或右键点按）Storyboard 中的按钮，然后开始向包含 ViewController 类定义的助理编辑器面板中拖移。将指针向代码中拖移时，会看到一条蓝色线 。

 

 

松开指针时，将显示“Outlets and Actions”对话框。确保“Connection”设置为“Outlet”，“Storage”设置为“Strong”。在“Name”字段中，为按钮指定一个变量名称：“myButton”。点按“Connect”按钮以完成插座变量的创建，此时会生成一行定义插座变量的代码。

 

现在，代码可以访问按钮了。接下来，请在 viewDidLoad() 函数中添加以下代码行：

 

myButton.setTitleColor(.red, for: .normal)

 

此代码行会将按钮标题的颜色从蓝色更改为红色。构建并运行 app，看一看更改生效后的效果。

效果不错吧，但你可能会发现，点按按钮时，按钮没有任何反应。要为按钮添加功能，需要创建一个与按钮绑定的响应动作。

在显示“Outlets and Actions”对话框时，将“Connection”设置为“Action”。在这种情况下，你在“Name”字段中输入的内容并不是定义变量，而是定义与轻点按钮绑定的响应动作。将响应动作名称指定为“buttonPressed”。点按“Connect”按钮以完成响应动作的创建。

关于 Interface Builder 的说明

Interface Builder 中的每个属性 (attribute) 都代表一个属性 (property)，后者也可通过编程方式或在代码中进行配置。Interface Builder 只是一个图形界面，用于配置和设置要在 app 中显示的 UIKit 类的属性。

将标签添加到场景中，然后查看标签的“Attributes”检查器。

许多可在 Interface Builder 中配置的对象都具有只能以编程方式进行设置的属性。例如，UIScrollView 有 contentSize 属性，它在“Attributes”检查器中没有匹配的选项。

当你需要调整其中一个设置时，可通过编程方式来实现，方法是设置滚动视图的 IBOutlet，然后使用点语法更新属性。

 

scrollView.contentSize = CGSize(width: 100, height: 100)

 

实际上，你在 Interface Builder 中所做的一切都可通过编程方式来完成，包括设置所有子视图并将它们添加到屏幕上。

 

let label = UILabel(frame: CGRect(x: 16, y: 16, width: 200,

height: 44))

view.addSubview(label) // 将 label 作为子视图添加到 view 当中

 

此类设置通常在视图控制器的 viewDidLoad() 方法中完成，这样可确保在视图显示在屏幕上之前可以访问视图控制器的主 view 属性。

虽然可以通过编程方式完成所有操作，但你会发现，在设置复杂视图时，Interface Builder 可以帮你节省大量的时间。随着项目的不断变大，你会发现使用 Storyboard 能够更轻松地维护界面设置。此外，Interface Builder 只需要在一个地方即可构建支持多种设备大小的复杂视图。
