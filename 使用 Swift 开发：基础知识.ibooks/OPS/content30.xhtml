在编写更大、更复杂的程序时，需要注意声明常量和变量的位置。代码中的最佳位置在哪儿？如果在顶端声明所有变量，代码就会变得难以阅读，更难调试。

在本节课中，你将学习编写结构清晰且易于阅读的代码。这可通过正确定义常量和变量的作用域来实现。

内容提要

  • 如何区分全局和本地作用域
  • 如何在全局和本地作用域中创建变量和函数
  • 如何借助变量隐蔽来重复使用变量名称

词汇

  • Global scope (全局作用域)
  • Local scope (本地作用域)
  • Scope (作用域)
  • Variable shadowing (变量隐蔽)

每个常量和变量都位于某种作用域内，也就是使其可见并可访问的某个位置。有两个不同级别的作用域：全局和本地。在全局作用域中声明的变量称为全局变量，在本地作用域中声明的变量称为本地变量。

全局作用域指的是可从程序中任意位置访问的代码。例如，当你开始在 Xcode playground 中声明变量时，就是在全局作用域内声明变量。在某一行中定义变量之后，该变量在该行后的每一行中都可用。无论何时完成在 playground 中输入内容，代码都会从这一全局作用域开始逐行执行。

每当你添加一对花括号 ({ }) 时，无论是针对结构体、类、函数、if 语句还是 for 循环等等，括号内的区域就定义了新的本地作用域。在花括号中声明的任何常量或变量都是在该本地作用域内定义的，无法从任何其他作用域对其进行访问。

例如以下代码块：

 

var age = 55

 

func printMyAge() {

print(”My age: \(age)”)

}

 

print(age)

printMyAge()

控制台输出：

55

My age: 55

 

请注意，变量 age 是在代码的顶部定义的，不是在控制流程结构体或函数内定义的。这意味着它位于全局作用域，在整个程序中都可以访问。虽然 age 并未作为参数传入，但是函数 printMyAge 可以引用该变量。类似地，函数 printMyAge 并未定义在结构体或类之内，所以它位于全局作用域，并因此可由代码中的最后一行访问。

现在看一下另一个代码块：

 

func printBottleCount() {

let bottleCount = 99

print(bottleCount)

}

 

printBottleCount()

print(bottleCount)

 

变量 bottleCount 是在函数 printBottleCount 之内定义的，该函数使用花括号来定义自己的本地作用域。所以 bottleCount 位于本地作用域，只有花括号中的函数内容可以访问该变量。代码的最后一行将抛出错误，因为找不到名为 bottleCount 的变量。

再看一个示例：

 

func printTenNames() {

var name = “Grey”

for index in 1...10 {

print(”\(index): \(name)”)

}

print(index)

print(name)

}

 

printTenNames()

 

在上面的代码中，name 是本地变量，在相同作用域内定义的任何内容都可以使用该变量。甚至在更小的本地作用域中也可以使用该变量：在下一行的 for 循环中。变量 index 虽然是在函数中定义的，但它定义在循环之内，这可以视为函数作用域中更狭窄定义的子部分。因此，只能在该循环之内访问 index。因为 print(index) 发生在循环外部，所以会生成错误。

变量隐蔽

接下来这个示例在两个不同的位置定义了名为 points 的变量：函数的本地作用域内和 for 循环的本地作用域内。这称为“变量隐蔽”。这是有效的 Swift 代码，但可能不太容易看出在执行代码时会发生什么。

 

func printComplexScope() {

let points = 100

print(points)

 

for index in 1...3 {

let points = 200

print(”Loop \(index): \(points+index)”)

}

 

print(points)

}

 

printComplexScope()

控制台输出：

100

Loop 1: 201

Loop 2: 202

Loop 3: 203

100

 

首先，声明了 points 并将其设置为 100。该值在下一行中打印。在 for 循环内，还声明了另一个 points，它的值是 200。第二个 points 完全遮蔽了函数作用域的变量，意思是它致使第一个 points 根本无法访问。对 points 的所有引用都将访问距离同一作用域最近的那个。所以当调用循环中的 print 语句时，会将 200 这个值打印五次。在循环结束后，print 语句仅会打印可访问的 points 变量，即值为 100 的那个。

在此特定示例中，为了避免不必要的困惑，你可能更倾向于更改里边的 points 变量的名称。你的想法可能是对的。但在某些情况下，变量隐蔽却很有用。假设有一个可选类型的 name 字符串，你想使用 if let 语法对其值进行操作。此时无需使用一个新变量名（如 unwrappedName），你可以在 if let 花括号的作用域内重复使用 name：

 

var name: String? = “Brady”

 

if let name = name {

// 此处的 name 为本地 String，隐蔽了全局的同名 String?

print(”My name is \(name)”)

}

 

使用变量隐蔽还可以简化从 guard 语句解包可选类型的命名。

 

func exclaim(name: String?) {

if let name = name {

// 大括号内，name 为解包的 String 值

print(”Exclaim function was passed: \(name)”)

}

}

 

func exclaim(name: String?) {

guard let name = name else { return }

// String? 类型的 name 不再可用，只有 String 类型的 name

print(”Exclaim function was passed: \(name)”)

}

 

在 Swift 代码中，隐蔽带有解包值的可选类型，是很常见的。请确保你能够阅读和理解这一常见模式。

隐蔽和构造器

你可以利用有关变量隐蔽的知识来创建简洁、易于阅读的构造器。假定你要通过传入 name 和 age 两个参数来创建一个 Person 实例。另外再假定每个 Person 实例都有 name 和 age 两个属性：

 

struct Person {

var name: String

var age: Int

}

 

let tim = Person(name: “Tim”, age: 35)

print(tim.name)

print(tim.age)

控制台输出：

Tim

35

 

在你编写构造器时，你希望尽量简单并合乎逻辑：将 name 参数指定给 name 属性，将 age 参数指定给 age 属性。

 

init(name: String, age: Int) {

self.name = name

self.age = age

}

 

因为 name 和 age 是函数作用域内的参数名称，所以它们遮蔽了在 Person 作用域内定义的 name 和 age 属性。你可以将关键字 self 放在属性名称前边来专门引用该属性，避免变量隐蔽可能会给编译器和读者造成的困惑。此语法清晰地表达了将 name 和 age 属性设置为传入构造器的 name 和 age 参数。
